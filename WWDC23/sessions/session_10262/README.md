# Session 10262 - 重新探索 Safari 开发者功能

> 作者：Style 月月， iOS 程序媛，简书/掘金文章贡献者，目前任职于小米，侧重于海外相关业务；
> 审核：；

本文是根据 WWDC23 中的 [Rediscover Safari developer features](https://developer.apple.com/videos/play/wwdc2023/10262/) 撰写，主要是探索 Safari 为开发、设计准备的调试工具集。

## 引言

Safari 是苹果公司开发的一款网络浏览器，是一个功能强大、易于使用且注重隐私和安全的网络浏览器，适用于苹果设备用户在互联网上浏览网页、搜索信息和进行在线活动等。

除此之外，Safari 还可以为开发者、设计人员提供丰富的工具集来检查网页、Web App、内嵌 Web 的 App 等。当设备与 Mac 配对时，这些工具集可以在 iOS、iPadOS、tvOS 和 xrOS 上无缝工作。

![image-20230618220706546](./wwdc23_10262/image-20230618220706546.png)

### 常用 Web 调试工具

Web 常用的开发者调试工具有两种，都可以用于调试、测试和分析 Web 应用程序，分别是：

- **Safari 调试工具**：是苹果公司为 Safari 浏览器提供的一套内置开发者工具，以下是 Safari 所具备的常用调试功能说明

| 调试功能                    | 说明                                                         |
| --------------------------- | ------------------------------------------------------------ |
| 元素检查器（**Elements**）  | 允许开发者检查和编辑 DOM 元素（HTML、CSS 、布局等）          |
| 控制台（**Console**)        | 用于测试、执行 JavaScript 代码（查看日志，错误排查、输出调试信息等） |
| 网络面板（**Network**）     | 显示 Web 加载的所有网络请求（HTML、CSS、JavaScript 文件、图片和 API 请求等），同时提供详细的请求、响应信息（HTTP 头部、请求时间、大小），以及模拟不同网络 |
| 资源检查器（**Resources**） | 用于查看和管理网络资源（图像、样式表、脚本文件、本地存储等），也可以查看资源的详细信息（文件大小、URL、缓存状态等） |
| 审查器（**Auditor**）       | 用于分析 Web 的性能和可访问性（性能瓶颈、安全问题、可访问性违规等） |
| 存储空间（**Storage**）     | 用于管理和调试 Web 的本地存储、缓存等（查看和修改存储数据、清除缓存、模拟离线状态等） |

![image-20230627092809488](./wwdc23_10262/image-20230627092809488.png)

- **Chrome 调试工具**：是由 Google Chrome 浏览器内置的一套开发者工具，以下是 Chrome 所具备的常用调试功能说明

| 调试功能                   | 说明                                                         |
| -------------------------- | ------------------------------------------------------------ |
| 元素检查器（**Elements**） | 允许开发者检查和编辑 DOM 元素（HTML、CSS、布局等）           |
| 控制台（**Console**)       | 用于测试、执行 JavaScript 代码（查看日志，错误排查、输出调试信息等） |
| 资源（**Network**）        | 显示 Web 加载的所有网络请求（HTML、CSS、JavaScript 文件、图片和 API 请求等），同时提供详细的请求、响应信息（HTTP 头部、请求时间、大小），以及模拟不同网络 |
| 性能（**Performance**）    | 用于分析 Web 的性能瓶颈和优化建议（页面加载时间、CPU、内存使用情况等） |
| 应用（**Application**）    | 用于管理和调试 Web 的本地存储、缓存等（查看和修改存储数据、清除缓存、模拟离线状态等） |
| 安全（**Security**）       | 提供有关网页安全性的信息和警告（SSL 证书、混合内容、跨站脚本等） |

![image-20230627092904352](./wwdc23_10262/image-20230627092904352.png)

**对比**

以下是两款调试工具的一些对比

| 对比功能        | Chrome                                                       | Safari                                                |
| --------------- | ------------------------------------------------------------ | ----------------------------------------------------- |
| 用户界面        | 调试界面复杂且全面，具备更多功能和选项                       | 调试界面简化，更注重 JavaScript 样式和 DOM            |
| 远程调试        | 支持远程调试功能，支持多个平台设备调试（如 Android、iOS 等） | 支持远程调试，局限于 Apple 设备，且需要具备一定的条件 |
| JavaScript 调试 | JS 调试非常强大（调试器、断电设置、变量监视等）              | 也支持，不如 Chrome 全面                              |
| 网络分析        | 提供了详细的网络请求信息、性能分析和缓存分析等，对前端性能分析和网络调试非常有用 | 也支持，不如 Chrome 全面                              |

### 建议 & 使用技巧

两种调试工具各有优劣，至于如何选择，可以从以下几方面进行考虑：

- **浏览器目标平台**：
  - 如果开发者主要是针对苹果设备的网站和 App，Safari 更适合，因为他对 iOS 设备具有更好的集成和调试；
  - 如果开发者需要跨平台开发，选择 Chrome 更合适；
- **功能需求**：如果开发者需要特定的功能，可以根据需求来选择合适的调试工具
- **习惯和熟悉度**：开发者根据自己的使用习惯选用调试工具，因为适合自己的就是最好的

当然，对于 iOS 开发者来说，Safari 就是最好的选择，以下是 Safari 的一些常用使用技巧：

- 打开 Safari 调试工具：**Option + Command + I**，或者**开发 -> 显示 Web 检查器**

- 模拟不同设备：**Option + Command + R**，或者**开发 -> 响应式设计模式**

  ![image-20230627092948125](./wwdc23_10262/image-20230627092948125.png)

- Cookie 增删改查：当我们想校验一个 cookie 时，可以通过调试工具来检查其效果

  ![image-20230627093452757](./wwdc23_10262/image-20230627093452757.png)

- 断点调试：类似于 Xcode 的断点调试，可以方便开发者实时排查问题

  ![image-20230627093651401](./wwdc23_10262/image-20230627093651401.png)

### Web 新 API

AppStore 有超百万 App 使用网页和 JavaScript。在这种情况下，苹果在 macOS 13.3 +、iOS 和 iPadOS 16.4+ 以及 xrOS 中提供了新 API 用于检查 App 中的内容。

**isInspectable 属性**

为了开发者更好的使用 Web Inspector，苹果在 iOS 16.4 SDK 中的 WKWebVIew 与 JSContext 新增了 **isInspectable** 属性。这个属性默认是 false，对于高版本的设备（ >= iOS 16.4）来说，只有开启了这个属性才允许进行 Web Inspector，且这个属性可以在任意时间修改（不需要重启 APP 或者重新创建 Webview 实例），设置代码如下所示。

- WKWebView

```swift
let webConfiguration = WKWebViewConfiguration()
let webView = WKWebView(frame: .zero, configuration: webConfiguration)
webView.isInspectable = true
```

- JSContext

```swift
let jsContext = JSContext()
jsContext?.isInspectable = true
```

设置完成后，我们就可以像往常一样进行 Web Inspector 了。

**JSContext 设置 name**

同时开发者可以将网页内容作为 App 用户界面的一部分，或者使用 JavaScript 来控制 App。此 API 可用于 WKWebView 和 JSContext，苹果建议开发者在使每个 JSContext 可检查时其提供一个名称，该名称将显示在 Safari 的“开发”菜单中，用于帮助开发者区分多个 JSContext。

![image-20230618230255968](./wwdc23_10262/image-20230618230255968.png)

一旦 App 启用了对某些内容的检查，开发者就可以从 Safari 的开发菜单中进行调试，就像调试其他 Web 内容一样。如图所示是对 Example App 进行的 JavaScript 调试。

![image-20230618230511896](./wwdc23_10262/image-20230618230511896.png)

如果还想要详细了解的，请参见[Enabling the Inspection of Web Content in Apps](https://webkit.org/blog/13936/enabling-the-inspection-of-web-content-in-apps/) 。

#### 合成字体警告

有时开发者会产生疑问，页面的字体真的是想要的吗，实际使用的字体是什么？为了解决这个问题，Web Inspector 提供了一系列的排版检查工具，如下所示

![image-20230618180845654](./wwdc23_10262/image-20230618180845654.png)

比如开发者想看元素的元素的字体样式，在调试工具中选择 **Elements -> Font** ，这里显示了元素使用的字体的属性和功能，此时可以检查元素的字体是否是开发者所期望的字体。

![image-20230618181008539](./wwdc23_10262/image-20230618181008539.png)

除字体的名称外，还展示了字体的大小、样式、粗细等，详情可参见 [Discover Web Inspector improvements](https://developer.apple.com/videos/play/wwdc2021/10031) 。

![image-20230618181049199](./wwdc23_10262/image-20230618181049199.png)

今年 WWDC23 在排版工具中新增了合成字体样式的警告。什么是**合成字体**？合成字体就是指在**常规字体样式上，通过一些算法运行后产生的字体样式**。

![image-20230618200107764](./wwdc23_10262/image-20230618200107764.png)

#### 可变字体配置支持轴调节

什么是**可变字体**？简单来说就是一种**字体格式**，可以在一个文件中包含一种字体生成多种样式所需的信息，如粗细、宽度等。

![image-20230618200150660](./wwdc23_10262/image-20230618200150660.png)

对于每种样式的支持，可变字体提供了对应维度的值。字体的各个维度均可配置，而这些可配置的维度都可以表示为可调节的轴。Web Insepector 今年新增了可调节的轴列表来展示这些可配置信息。当开发者需要检查使用了可变字体，调试工具的 Elements -> Font 显示了一个可调节的轴列表。

![image-20230618200859997](./wwdc23_10262/image-20230618200859997.png)

对于每个轴，均有以下展示

- 轴标识符（四字字符）、轴标签（可选）

- 轴的最小、最大值

  ![image-20230618201208778](./wwdc23_10262/image-20230618201208778.png)

  ![image-20230618201236355](./wwdc23_10262/image-20230618201236355.png)

#### 用户偏好模拟

Web Inspector 的另一个新特性是**用户偏好模拟**，用来确保正在构建的页面内容符合用户需求。在此之前，开发者想要测试一个用户偏好，需要去 macOS 的辅助功能设置，但这种设置更改将影响整个系统。而开发者只是想在测试的页面中设置，为了减少对开发者系统的影响，Web Inseoctor 新增了用户偏好模拟工具。

![image-20230618202550788](./wwdc23_10262/image-20230618202550788.png)

- **Color scheme**：页面主题颜色
- **Reduce motion**：减少动画
- **Increase contrast**：增加对比度

## Preview responsive layouts 预览响应式布局

![image-20230627112241977](./wwdc23_10262/image-20230627112241977.png)

今年的第二个更新点在于“响应式设计模式”工具，开发者可以从 ”**开发 -> 响应式设计模式**” 来预览 Web 页面在不同设备、尺寸的布局情况，下面是 WWDC23 之前的响应式设计模式图示。

![image-20230618223229267](./wwdc23_10262/image-20230618223229267.png)

开发者可以从三个方面来预览网页内容

- 指定页面大小预览
- 指定页面像素预览
- 指定设备预览

下面进行一一说明

### 指定页面大小预览

开发者可以尝试使网页视口比浏览器窗口大，甚至可以比电脑屏幕大。有两种方式

- **手势拖动**设定视口大小，网页内容会根据视口大小进行自适应。

  ![20230626235312](./wwdc23_10262/20230626235312.png)

- 在视口上方**输入任意宽高**，设定视口大小。

  ![image-20230618222933239](./wwdc23_10262/image-20230618222933239.png)

在视口的上方宽高的右边，可以看到比例因子，这个是视口被渲染的实际尺寸的百分比。如图所示，视口的比例不是 100%，因为当前视口高度是 800 像素，不足以显示整个视口，所以 Safari 将视口的显式缩小到实际尺寸的 78%。

![image-20230618223046767](./wwdc23_10262/image-20230618223046767.png)

### 指定页面像素预览

响应式设计模式除了大小可以改变，还允许开发者改变**像素比例**。针对不同显示器提供定制的图像资源和样式，开发者可以通过像素比例来测试不同设备的响应式布局。

![image-20230618223134496](./wwdc23_10262/image-20230618223134496.png)

同时允许开发者调整页面样式，如边框大小、响应显示的像素比等。有以下几种方式可以响应 Web 上显示的像素比例，分别是

- **img** 标签的 src
- 可以在任何地方使用的 CSS 的 **image-set** 图像集功能
- **media** 分辨率媒体查询

![image-20230618223422680](./wwdc23_10262/image-20230618223422680.png)

### 指定设备预览

除了需要在 macOS 上测试页面的响应式，还需要在 iOS、iPadOS 和 xrOS 上测试页面的响应式。在此之前，开发者在响应式设计模式中只能模拟不同设备的显示，如下所示

![image-20230627001650943](./wwdc23_10262/image-20230627001650943.png)

在今年 WWDC23 中，引入了从 Safari 快速跳转到模拟器的功能，相比在 Safari 中的模拟显示，在模拟器中测试 iOS、iPadOS 和 xrOS 上的网页内容是一个好的选择，因为不需要真机即可进行测试。

iOS 和 iPadOS 的模拟器包含在 Xcode 中，当开发者安装好 Xcode 并运行完成设置后，再次启动 Xcode，就可以在 Safari 中使用模拟器了。

使用步骤如下：

## Automate tests 自动化测试

![image-20230618230938503](./wwdc23_10262/image-20230618230938503.png)

随着网络的发展，需要不断测试网页来确保是按照开发者预期运行的。开发者可以通过 **WebDriver** 来检查和调试网页。WebDriver 是一个**跨浏览器的 API**，用于自动测试所有主流浏览器支持的 Web 内容，而不需要特定于浏览器的代码。他托管了一个本地 Web 服务器，可以接受 HTTP 请求，因此可以接受来自各种测试设置的自动化命令，同时也提供了很多命名用于网页的自动化测试。

![image-20230618231036004](./wwdc23_10262/image-20230618231036004.png)

Safari 的 WebDriver 支持是始于 iOS 10 系统，在 10 以前，Safari 浏览器没有原生支持 WebDriver。在 Safari 浏览器中，WebDriver 是一种用于自动化测试和控制浏览器行为的工具。它是 Web Driver 规范的一部分，提供了一组 API 和协议，允许开发人员通过编程方式与 Safari 浏览器进行交互。

WebDriver 可以让开发人员自动化执行各种浏览器操作，例如导航到特定的 URL、填写表单、模拟点击和键盘输入等。它还可以获取和操作页面元素，例如查找元素、执行 JavaScript 代码、获取元素属性和文本等。

使用 WebDriver，开发人员可以编写自动化测试脚本，以验证网页在不同条件下的行为和性能。这对于进行功能测试、回归测试和性能测试非常有用。此外，WebDriver 还可以用于执行屏幕截图、监控网络请求、处理弹出窗口等。

![image-20230618231112487](./wwdc23_10262/image-20230618231112487.png)

大多数时候，开发者将使用三方库与 WebDriver 进行交互，其中一个流行的库是 Selenium，它提供了 Python、Java、PHP、JavaScript 等语言的 API。

## Explore the future web 探索 Web 未来

前面已经介绍了今年的更新点，下面我们来探索下 Web 的未来发展。网络是在不断发展的，对于开发者来说看到潜在的工具和技术是兴奋的，更令人的振奋的是开发者可以在发布前进行使用。

在即将更新的 Safari 中，如果开发者想试用未发布的工具，可以通过新的**功能标志设置（Feature Flags）**来启用这些功能，Feature Flags 在此之前被称为**实验性功能** (Experimental Features)，如下所示

![image-20230618231757056](./wwdc23_10262/image-20230618231757056.png)

![image-20230618231843373](./wwdc23_10262/image-20230618231843373.png)

这些 **Feature Flags** 是按照主题划分的，如动画、CSS、JavaScript、媒体等，而且可以根据功能进行搜索，如果想启用，只需要在功能前的复选框打钩即可。

![image-20230618232220194](./wwdc23_10262/image-20230618232220194.png)

这些未发布的功能状态主要分四种：

![image-20230618232325955](./wwdc23_10262/image-20230618232325955.png)

- **stable**（已发布状态）：表示最近开发在 Safari 中发布的功能，默认是打开的。我们可以切换这些功能来帮助确定某个功能是否会引发问题等，而这些稳定功能最终会从 Feature Flags 列表中移除。

  ![image-20230618232400161](./wwdc23_10262/image-20230618232400161.png)

- **testable** （可测试状态）：表示可测试的功能，即还没有完全准备进入黄金时期（简单理解就是仍在集测，尚未全量的功能），但可能已经准备好进行早期反馈或正在实现过程中的功能，这些功能默认情况下**禁用**（即未启用）的。

  ![image-20230618232535059](./wwdc23_10262/image-20230618232535059.png)

- **preview** （预发布状态）：表示已经准备好让开发人员开始测试的功能，但仍然可能存在 bug，虽然在 Safari 中**默认禁用**，但在 Safari 技术预览（[Safari Technology Preview](https://developer.apple.com/safari/technology-preview/)）中默认启用的。

  ![image-20230618232731570](./wwdc23_10262/image-20230618232731570.png)

- **developer**（开发者状态）：专门为开发者提供的一种状态，可以让开发人员重新启用已经弃用的 API 进行测试。

  ![image-20230618232820686](./wwdc23_10262/image-20230618232820686.png)

>PS：当更新 Safari 时，这些 Feature Flags 会自动重置为默认状态。

## 总结

综上所述，Safari 开发者功能新增点如下

- **Inspect web pages**：介绍了常规网页调试，并结合  [What's new in Web Inspector](https://developer.apple.com/videos/play/wwdc2023/10118/) 简单介绍了下 Web Inspector 中新增的调试功能，如：用户偏好模拟、新增元素角标、新增符号断点调试等
- **Preview responsive layouts**：介绍了响应式设计模式的使用，并新增了模拟器的响应式调试
- **Debug beyond the Mac**：除了常规的 iOS、iPadOS 等设备调试，还新增了 xrOS 设备的调试，以及新增的 Web API，用于调试网页和 JSContext
- **Automate tests**：介绍了 WebDriver 的使用，并演示了如何实现自动化测试
- **Explore the future web**：开发菜单栏新增了 Feature Flags，开发者可以使用即将发布、测试中甚至开发中的 Web 功能，也可以在这里重新启用已经被启用的 API 进行调试
