# 【WWDC - 10147】 创造优秀的视频播放体验

本文基于 [Session 10147](https://developer.apple.com/videos/play/wwdc2022/10147/) 梳理。

> 作者：刘思源，就职于字节跳动音乐团队，现参与汽水音乐。

这篇 session 围绕着 Apple 新升级的视频播放器，描述了新的播放器的新界面、新功能，以及 Apple 自己对提高视频播放体验的建议和思考。

## 全新的视频播放器

系统播放器迎来了全新的设计和交互。我们将新版的播放器设计和旧版的做对比来看：

![jlVw59.png](https://s1.ax1x.com/2022/07/01/jlVw59.png)

新版播放器

![jlVaE4.jpg](https://s1.ax1x.com/2022/07/01/jlVaE4.jpg)

旧版播放器

不同之处可以体现在以下几点：

- **强调了播放控制**：暂停/播放、进度控制、进度条、音量。相较于之前的设计更突出，也更现代

- **突出了内容信息**：原本的视频播放器在根本看不到标题、描述之类的信息。Apple 这次的改动也把这些信息展示了出来

- **交互更现代**：优化了用户的交互手势，对操作的容错更强。在一些特定的交互，比如在进度条控制上增加了惯性，更符合用户感知，体验更顺滑

- **多平台统一**：在不同设备上：iOS、macOS、tvOS ，播放视频的体验将变得统一，支持各种外接设备，原生感更强

  我们接下来就视频强调的几点展开描述下

### 更方便的交互

- 播放/暂停

Apple 将播放/暂停和快进/快退控件放在前面和中间，使它们更容易交互。并且现在可以点击显示屏的中心，即使控件被隐藏以播放和暂停视频。

- 进度条控制

Apple 对时间线的可用性进行了一些重大改进。进度条的滑块按钮被取消了，现在可以从进度条的任何位置与时间线进行交互。并且拖动不再需要从当前时间标记开始，可以从视频的任何位置滚动时间轴。当用户开始滚动视频帧时，界面会消失，只留下最相关的 UI，让焦点停留在内容上。

- 缩放

Apple 添加了一种更改视频填充方面的新方法。用户现在可以使用捏合手势在可用缩放级别之间移动。捏合会将视频带到显示器的安全区域内。捏合将放大视频以完全填满显示屏。

### 支持展示视频元信息

你可能注意到，在新版视频播放器里面支持展示视频的元数据信息：比如视频的标题、副标题、描述。默认情况下，标题、副标题和内容描述将从媒体的元数据中提取。同时它提供了自定义的配置能力：可以通过 `AVPlayerItem` 上的 `externalMetadata` API 覆盖媒体中的值。

![视频元信息](https://s1.ax1x.com/2022/07/01/jlFpJx.png)

我们可以生成不同的 `AVMetadataItem` 去覆盖对应的信息，这有点像经常用在通知栏中的若干属性。

- `commonIdentifierTitle` 标题

标题应该简单并且具有描述性，过长的标题会导致 UI 错乱

- `commonIdentifierSubTitle` 副标题

副标题实际上会展示在标题的上方，内容应当是能描述内容的几个词

- `commonIdentifierDescription` 详细描述

如果视频有详细描述的话，标题和副标题的右侧将会显示一个箭头。点击箭头将显示包含内容描述的信息面板。描述应该是几句话，包含有关内容的更多信息。点击任何地方都会关闭信息面板。

### 多平台和系统支持

新版的播放器在各个端上的体验达到了统一。在 iOS、macOS、iPadOS、tvOS 上，播放器无缝集成到系统中，完全支持键盘、触控板、鼠标、游戏控制器等等。Apple 还添加了许多与控件交互的新方法，这些控件使导航内容和一些常见交互变得更加容易和直观。
在你的应用程序中使用 `AVPlayerViewController`，你将获得系统播放器的全面支持——支持画中画、SharePlay、可视化分析、原生 Catalyst 支持、新硬件和新功能支持等等。

![多平台支持](https://s1.ax1x.com/2022/07/01/jlEkOU.png)

>
> 我们想与你分享这个过程；**我们是如何设计新播放器的，为什么要按照我们的方式设计它们，以及我们认为定义了惊人的媒体体验的东西。**

我们在上一部分介绍了 Apple 新的播放器设计，这其中其实包含了 Apple 对于播放器体验的思考。现在，让我们谈谈如何在设计令人惊叹的播放体验。Apple 认为让媒体体验极致的三件事:

1. 直观的体验
2. 和系统深度集成
3. 内容优先

### 直观

体验应该是直观的。即使你从未使用过它，它也应该让人感觉轻松、熟悉、自然。你很难去用语言描述哪些体验是让你感到直观的，但当你看到他的时候，你就知道了。

这其实更像是一种设计指引，那到底怎么落实呢，究竟是什么让界面或者体验变得直观？Apple 认为这始于熟悉。你可以通过过去的经验，轻松的理解新事物，并且或许你自己都没有意识到这一过程时，我们就认为这是一个直观的行为。你不需要解释它是如何工作的，甚至不需要考虑它是如何工作的时，它就会完全按照你的预期工作。我们每个人都在和现实世界之间构建交互体验。这是体验熟悉度的重要来源，并且通常是我们在设计新系统媒体播放器时开始的地方。

有很多类型的经验可以从中汲取直觉，但我想重点关注两种——我们在设计系统媒体播放器时最常依赖的两个：**平台范式和现实世界**。

#### 平台范式

第一个来自我们每天使用技术的经验。多年使用电视遥控器告诉你，方向键左右移动焦点。类似地，点击第一个触摸设备上的音量按钮将使音频静音。这些交互感觉很直观，因为它们很熟悉。你可以在媒体体验中使用这些类型的熟悉交互，以帮助使你的应用程序感觉更直观、更有吸引力，甚至使用起来更自然。相反，发现不熟悉或意外的交互可能会令人困惑。

来看一些比较好的例子吧：iOS 拉起系统播放器。

这实际上使用的是模态的展示。播放器从底部推出，用户可以手动下滑来关闭这个页面。我们看到这个模型在大量场景中广泛使用。例如，Apple Music 的播放页面，播放页可以从底部的 minibar 中拉起，并且可以通过向下滑动来关闭。

#### 真实世界

但在某些情况下，我们可能不是从我们对技术的理解中汲取经验，而是从日常生活中汲取经验。这些类型的体验来自现实世界。数百万年的进化帮助我们对自然过程产生了深刻的本能理解。你可以利用这种理解来帮助在你的软件中构建熟悉和直观的体验。

一个很好的例子是我们在 iOS 播放器中的新滚动手势。类似于在桌子上滚动一辆玩具车，在视频上的每一次滑动都有惯性和阻尼，通过直接交互继续移动时间线，直到时间线慢慢停止。仅这里的动量就与现实世界的移动物体建立了联系。这种关联可以帮助你发现交互的微妙深度。就像玩具车一样，我可以用力推它，或者连续推它几次，让它跑得更快。如果我抓住它，它就会停止。感觉很自然，正因为它就是自然的。

构建直观的设计是使你的媒体体验出色的一个方面，但如果没有人们期望的所有功能和集成点，你的应用程序可能会无意中将焦点从内容上移开。

### 和系统的深度集成

创造令人惊叹的媒体体验的第二个关键方面——紧密集成。当体验紧密集成时，人们期望工作的所有功能、特性和设备都能正常工作。重要的是，他们的工作方式与他们的期望一致。

随着人们使用他们的设备，他们习惯于依赖平台的功能。例如，拉下控制中心并在“正在播放”媒体控件中看到你的内容，或者在观看电视节目时响应通知，并让视频顺利进入画中画。将这种紧密的系统集成构建到你的应用程序中是让你的体验感觉无缝的关键。

你的应用应该感觉像是系统的原生部分，我们努力为你提供实现这一目标所需的工具。这包括诸如 CoreSpotlight 集成以使你的内容可搜索、正在播放信息以便你的内容可以出现在系统媒体 UI 中、MediaRemote 命令允许你的应用程序响应诸如按下键盘或电视遥控器上的播放按钮之类的事情。我们甚至可以将你的媒体直接集成到电视应用程序中，从而将你的内容提供给更广泛的受众。

除了让你的应用程序感觉原生之外，提供人们喜爱的所有功能也很重要。 AirPlay、SharePlay 和画中画等功能。我们认为人们会期待这些功能，并且提供这些功能可以增强使用你的应用程序的体验。人们将在许多设备甚至更多输入格式上使用你的应用程序。为所有这些提供支持对于确保每个人都能获得你的体验至关重要。

这在 tvOS 上尤为重要，支持所有可用的遥控器对于确保每个人都可以使用你的应用程序至关重要。这是我们始终建议在 tvOS 上使用系统媒体播放器的原因之一。你的应用程序应确保所有电视遥控器、键盘、触控板、游戏控制器和耳机控件的流畅体验。此外，你应确保在屏幕的安全区域内绘制应用程序的 UI 元素，以避免与显示屏中的圆角或凹口发生碰撞。我们认识到为所有这些集成点、功能和硬件配置构建支持可能会令人生畏。这就是我们推荐开发者使用 `AVPlayerViewController` 的原因，只需几行代码，任何使用你的应用程序的人都可以拥有令人惊叹的媒体体验。

### 内容优先

最后，所有这些都将我们带到了设计媒体体验的最重要方面：内容优先。这应该是你设计的主要目标，也是我们考虑的优质媒体体验的定义方面。当你的应用程序感觉直观时，当人们期望的所有集成点和所有功能都正常工作时，你将你的内容变成焦点，而其他一切都消失在背景中。

但是，你应该记住一些特定于设计内容的事情。确保在界面和系统中提供所有适当的元数据。提供此信息有助于为你的媒体提供上下文，并允许系统在控制中心和锁定屏幕中提供更好的体验。这包括标题和副标题、描述、缩略图、季节和剧集信息，或者直播的开始和结束日期。始终保持媒体的原始纵横比。这允许系统将你的视频放置在屏幕上的正确位置。确保尽可能支持最新的媒体标准，例如，HDR 和杜比全景声。最后，确保包含多种语言的音频和字幕轨道，以便尽可能多的人使用你的媒体。

## AVPlayerViewController 的新功能

新的播放器不仅仅是 UI 层面的改动，在 API 和`AVPlayerViewController`支持的能力上也有了新的变化。我们主要阐述三个方面：

- 视觉只能
- 插页广告的官方实现
- 播放速度

### 视觉智能

![jlFiQO.png](https://s1.ax1x.com/2022/07/01/jlFiQO.png)

在此示例中，我们将视频暂停在其中包含代码片段的帧上。长按代码片段将选择它。然后，你可以将其复制并直接粘贴到 Playground 中进行试用。这也适用于 macOS。将光标悬停在相同的代码上将显示鼠标光标的样式时，指示文本是可选的。然后，你可以使用光标突出显示它或全选。

![jlFFyD.png](https://s1.ax1x.com/2022/07/01/jlFFyD.png)

Apple 引入一个新的 API 来让开发者使用这个功能。在 iOS 上的 `AVPlayerViewController` 和 macOS 上的 `AVPlayerView` 中可用。当 `allowVideoFrameAnalysis` 设置为 true 时，一旦媒体暂停，AVKit 将在设定的时间后开始分析当前视频帧。请注意，出于性能原因或受保护的内容，不建议在滚动时分析帧。
但是，在某些情况下，你可能希望在应用程序中禁用可视化分析。例如，在性能关键的应用程序中，或者在不期望与视频交互的情况下。有关如何将视觉智能功能集集成到你的应用程序的更多信息，请参阅关于 VisionKit 的相关讨论。

### 插页广告

接下来，让我们看看我们对插页式广告所做的一些改进。到目前为止，插页式广告仅在 tvOS 上的 `AVPlayerViewController` 中受支持。在本次 WWDC 中，Apple 也为 iOS 带来相同级别的支持。

插页式广告，无论是在流中还是通过 `AVPlayerInterstitialEvents` 在本地定义，现在都将沿时间线进行标记。当时间线到达某个标记时，将开始播放插页式广告。如果你的插页式广告已在你的 HLS 播放列表中完全定义，你将自动获得此行为。如果没有，或者你的应用需要更多自定义行为，一些新的 API 也能助你一臂之力。

![jlFkOe.png](https://s1.ax1x.com/2022/07/01/jlFkOe.png)

`AVInterstitialTimeRange` 目前在 iOS 上也是可用的。这个值将自动填充到 `AVPlayerItem` 属性的 `interstitialTimeRanges` 中。使用 HLS 流时，将为流中的每个插页式广告合成一个 `AVInterstitialTimeRange`。通过 `AVFoundation` API 在本地创建插页式事件时，将为每个 `AVPlayerInterstitialEvent` 合成一个 `AVInterstitialTimeRange`。

然而，与 tvOS 不同，`interstitialTimeRanges` 是一个只读属性。插页式广告要么需要在 HLS 流中定义，要么通过 `AVPlayerInterstitialEvents` 定义。

![jlFEeH.png](https://s1.ax1x.com/2022/07/01/jlFEeH.png)

本次更新还从 tvOS 引入了两个代理方法。这些可用于回调插页式广告开始或结束播放的时机。

我们来举个例子看看如何使用：

假设我们要为贴片广告添加跳过按钮。首先，我们将为播放器创建一个 `AVPlayerInterstitialEventController`。接下来，我们将创建一个插页式事件，并增加一些限制。这些限制可防止在插页式广告中搜索并防止跳过插页式广告。然后我们将插页式广告添加到事件控制器。最后，我们可以实现新的 `willPresentInterstitial` 代理回调，以在设定的时间间隔后显示一个广告跳过按钮。按下按钮后，我们将取消插页式广告。

![jlFZTA.png](https://s1.ax1x.com/2022/07/01/jlFZTA.png)

请注意，将任何自定义 UI 元素添加到 `AVPlayerViewController` 时，例如此广告跳过按钮，请始终确保将其添加为 `contentOverlayView` 的子视图。

了解更多的贴片广告的内容，可以查看 [Display ads and interstitials in SharePlay - WWDC22 - Videos - Apple Developer](https://developer.apple.com/videos/play/wwdc2022/110380)

### 播放速度

Apple 今年在所有平台上添加的新功能还有一项：支持播放速度控制。 AVPlayerView 和 `AVPlayerViewController` 现在可以选择使用新的 API 显示播放速度菜单。

![jlFufP.png](https://s1.ax1x.com/2022/07/01/jlFufP.png)

功能支持在 macOS、iOS 和 tvOS 上。让我们来看看这是什么样子的。在 tvOS 上，你会在传输栏中看到一个新控件。选择控件将显示可供选择的可用播放速度列表。在 iOS 上，此菜单将出现在控制拉起菜单中。同样，在 macOS 上，控件将出现在拉起菜单中。所有链接到新的 iOS、macOS 和 tvOS SDK 的应用程序都将自动获得此功能，无需额外更改。

但是，某些应用程序可能想要修改速度列表、以编程方式选择速度或完全禁用菜单。为了适应这些用例，`AVPlayerView` 和 `AVPlayerViewController` 添加了一些新的 API。

![jlFVwd.png](https://s1.ax1x.com/2022/07/01/jlFVwd.png)

首先 AVKit 中添加了一个新类：`AVPlaybackSpeed`。 `AVPlaybackSpeeds` 表示播放 UI 中用户可选择的速度选项，它们具有三个属性：

- `Rate`：表示快进的速率
- `localizedName`：用于表示无障碍系统中的播放速度提示。例如，2.5 的速度可能使用“两倍半速度”的本地化名称。
- `localizedNumericName`：该值是从 rate 属性合成的，将是播放速度菜单中显示的字符串。如果你的应用需要播放器外部的自定义播放速度菜单，请使用此字符串来表示速度。
- `AVPlaybackSpeed`：`AVPlaybackSpeed` 定义了应尽可能使用的系统默认速度列表。默认情况下，此属性将设置为 `systemDefaultSpeeds` 数组。

![jlFmFI.png](https://s1.ax1x.com/2022/07/01/jlFmFI.png)

让我们看一个例子。在这里，我们正在创建了 `AVPlayerViewController` 并展示它。默认情况下，这将使用系统提供的播放速度列表。你可以通过创建 `AVPlaybackSpeed` 并将其附加到 `AVPlayerViewController` 中的速度列表来向菜单添加新速度。

![jlFMSf.png](https://s1.ax1x.com/2022/07/01/jlFMSf.png)

我们还可以通过设置一个空的速度列表来禁用菜单。但请注意，你应该始终调用 `AVPlayer` 的`play()` 来开始播放。永远不要通过调用 `setRate:1.0` 开始播放，因为选择的速率可能不是 1.0。

## 总结

这篇 Session 的内容到这里基本就结束了，我们来总结一下内容：

- 我们看到了重新设计的新 iOS 系统播放器

- 我们了解了你如何设计惊人的播放体验

- 我们看到了一些很酷的新视觉智能功能，并了解了新插页式广告和播放速度 API

  本文相较于硬核的技术内容，更偏向于功能介绍和设计建议。近年来，Apple TV 在流媒体上也逐渐发力，我们可以看到更多来自于 Apple 独家制作的视频内容。与之而来的有本次升级的播放器功能，可以说是是内容推动了形式，但对于视频播放器来说，或许形式就是内容的一部分。加上苹果播放器天生对系统的深度集成，或者系统播放器会是更多独立开发者/小团队的首选。在视频播放器的设计上，Apple 的官方建议也颇具参考，或许能对我们在代码之外有所启发。
