## 简介

本文主要回顾了往年 Session 中的 Vision 条形码识别、文本识别（OCR），然后是本次新增加 DocumentSegmentation（文档分割）。

目前，Vision 已经被广泛应用于图像分析领域，包括但不限于以下场景：

![Vision Focus](https://images.xiaozhuanlan.com/photo/2021/a9be570995dd4752f35adfcb13115602.png)

1. 运动（动作分类，手势辨认、对象跟踪等）
2. 辅助功能（OCR 识别，帮助视觉障碍人士读取图片上的信息）
3. 脸部识别、身体分割
4. 健康（条形码识别，分析人的姿势以做出摔倒的概率预测等）
5. 拍照（比如系统相机里的人像模式，检测出脸部和环境，从而聚焦在脸部）
6. 安全
7. 文档
8. 等等...

而本文聚焦在第7种，即文档方面的新应用。



## 回顾及更新

### 1. 条形码

Vision 的条形码检测器特性：

- 支持识别一维、二维的条形码
- 支持一次性识别一张图片中的多个条形码
- 支持一次性识别多种类条形码

#### 它们是如何工作的？

##### 一维码

![一维码](https://images.xiaozhuanlan.com/photo/2021/128686b0eee5f7834d505d4cacee909c.png)

由于一维码是按行扫描的，而每行的信息是一样的，所以我们会得到多个一样的检测结果。但是，我们不需要人工去重，因为苹果已经帮我们做了，直接读取 **payload**，就是真正的数据了。

>  一般情况下，商品的包装袋或者图书馆的图书上都有这种码。

##### 二维码

![二维码](https://images.xiaozhuanlan.com/photo/2021/1e2552422da119604e8de7134f9344b1.png)

二维码是直接按单元扫描的，每个码有且仅有一个 boundingBox。 直接读取 **payload** 就可搞定了。另外如果码中内容是 URL，需要转义 % 符号的，可以用 **NSDataDetector**。

> 它是我们最常见的码，比如微信、支付宝等等的 `QR Code`。

#### Demo

![条形码Demo](https://images.xiaozhuanlan.com/photo/2021/f9ab4538d19f700b8afd7d238c8bc356.png)

图上大多是样板式请求代码。其中，这两个地方需要注意：

```swift
// 明确指名使用的算法版本
detectBarCodesRequest.revision = VNDectectBarcodesRequestRevision2 
// 需要检测哪种类型的条形码，可传入多个。 若传入[]，则代表需要检测所有类型
detectBarCodesRequest.symbologies = [.codabar]
```



#### 更新内容

此次苹果更新了算法，增加了 **VNDectectBarcodesRequestRevision2**，支持更多种类的条形码检测：

![新的条形码算法](https://images.xiaozhuanlan.com/photo/2021/3f565f86550a38e8cb5e1d9251f3f7a5.png)

其中，**microQR** 可能是对我们最有用的！如果你只想简单用一个 URL 生成二维码，又想生成的图片占更小的面积，那么就用它吧！

需要**注意**的是，这次算法**变更更大**的地方在于，它改变了一些行为：

![](https://images.xiaozhuanlan.com/photo/2021/13f2bb2f835781702f0b236747e126e4.png)

旧版本算法（Revision1）：不管你是否设置了 ROI ，得到的结果中的 BoundingBox 总是基于原图的左下角

新版本算法（Revision2）：得到的结果中的 BoundingBox，是基于 ROI 的左下角为坐标系的（当没有设置 ROI 的时候，和旧算法一致）

> ROI：Region Of Interest，感兴趣的区域，图上的黄色区域。通过它，你可以在检测某张图片时候，只聚焦在其中一部分。通常可以提高检测的速度，减少耗时。
>
> BoundingBox：检测到的物体的几个顶点组成的，最少能包围住顶点的矩形边框，图上的红色区域。

所以，在使用 Vision 的时候，我们最好**明确的、显式**写上当前使用的版本！如果不写，一旦以最新的 SDK 重新编译你的 app，将默认使用最新版本的算法。同样的代码，却可能得到莫名其妙的结果。 



### 2. 文本识别

Vision 的文本识别也可叫 OCR 识别，于2019年的 WWDC 中首次引入。它有 **Fast**(快速) 和 **Accurate**(精确) 两种工作模式：

![compare](https://images.xiaozhuanlan.com/photo/2021/a33dbed8289cf02eb95f8a0eeb554b46.png)

这两种模式，都需要经过 `识别器识别 -> 语言矫正 -> 得到识别的文字`。

那么，两种模式间有何区别？主要是识别器的不同。Fast 模式下使用的是拉丁字符识别器，Accurate 下使用的是基于机器学习的识别器。 而这一个区别也导致了，像**中文**这种复杂的非拉丁字符组成的文字，只能用 Accurate 模式！而**英语**则可以从两种模式中选择。

```swift
        let textRequest = VNRecognizeTextRequest()
        // 设置工作模式 .fast 或者 .accurate
        textRequest.recognitionLevel = .accurate
        // 是否启用语言矫正
        textRequest.usesLanguageCorrection = false
```

另外， 是否启用语言矫正，也对性能产生影响，根据你使用的情况，如果能**关闭**，识别速度可以更快。

举个例子，如果你有一个 crash 文件，里面的是被看门狗杀死的代码 `0x8badf00d`，把 `0x8badf00d` 截成图片拿去识别。如果你开启了语言矫正功能，则会被 Vision 识别成 0x8bad**food**，因为 0 像 o，food 是有意义的单词，而 f00d 是没有的！这个时候，如果要识别出 f00d，就需要关闭自动矫正。



##### 关于语言的设置

在识别中文时候，需要手动设置语言， 我们可以通过 `supportedRecognitionLanguages` 得到当前支持的所有语言，然后进行设置。

```swift
        let supportLanauages = try? textRequest.supportedRecognitionLanguages() // iOS 15才有的api
        // ["en-US", "fr-FR", "it-IT", "de-DE", "es-ES", "pt-BR", "zh-Hans", "zh-Hant"]
        textRequest.recognitionLanguages = ["zh-Hans", "zh-Hant"]
```

注意，语言的顺序非常重要！Vision 默认使用你指定的第一种语言的模型先进行识别，失败后才会用第二种、第三种等等。

### 3. 文档检测

在今天的新主题 **文档分割请求** `VNDetectDocumentSegmentationRequest` 出来之前，苹果已经有了一个便捷的文档工具 `VNDocumentCameraViewController`。 它是一个类似于 UIImagePickerController 一样的高度封装工具，都是 UIViewController。 通过它，你能进入到一个文档扫描拍摄的页面，可以连续扫描多张，系统为你进行自动矫正（有时候需要人工选择四个边角）。

```swift
// 需要导入这个
import VisionKit

if VNDocumentCameraViewController.isSupported {
    let vc = VNDocumentCameraViewController()
    vc.delegate = self
    present(vc, animated: true, completion: nil)
} else {
    print("current device doesn't support document camera")
}
```

之后也是通过 delegate 进行结果回调，和 `UIImagePickerController` 简直一模一样：

```swift
// MARK: - <VNDocumentCameraViewControllerDelegate>

extension VisionKitDemoVC: VNDocumentCameraViewControllerDelegate {
    
    // 成功的回调
    func documentCameraViewController(_ controller: VNDocumentCameraViewController, didFinishWith scan: VNDocumentCameraScan) {
        controller.dismiss(animated: true, completion: nil)
        
        // 可能扫描了多张
        for pageIndex in 0..<scan.pageCount {
            let image = scan.imageOfPage(at: pageIndex)
            // 对图片进行后续的处理...
        }
    }
    
    // 用户取消了
    func documentCameraViewControllerDidCancel(_ controller: VNDocumentCameraViewController) {
        controller.dismiss(animated: true, completion: nil)
    }
    
    // 错误处理
    func documentCameraViewController(_ controller: VNDocumentCameraViewController, didFailWithError error: Error) {
        controller.dismiss(animated: true, completion: nil)
    }
}
```

## 文档分割

> 注意：仅 iOS 15.0+ 可使用!!! 

`VNDetectDocumentSegmentationRequest`  文档分割，把图片中的文档部分切割出来。 

我们知道，从图片中切出文档的部分，用 RectangleDector 后再特殊判断也是可以的，那么为什么还要用新的 DocumentSegmentation 呢？

`VNDetectDocumentSegmentationRequest` 和 `VNDetectRectanglesRequest` 的不同点是，RectangleDetector 对于画面中能检测到的矩形框，都可能是识别结果（视你的最小检测物体比例和最大识别个数设置），而 DocumentSegmentation 只会去识别画面中的文档部分！ 这样就少了特殊处理的环节，且误判更少。

![Document Segment vs Rectangle Detector](https://images.xiaozhuanlan.com/photo/2021/75fb2c4ec96ed68e576456e91f03efb4.png)

如上图，同一张图片，左边的 DocumentSegmentation 只能得到一个结果，就是文档本身； 而右边的 RectangleDetector 得到了5个矩形框，它可以是嵌套的！

在底层技术以及细节上，对比如下：

![Details: Document Segment vs Rectangle Detector](https://images.xiaozhuanlan.com/photo/2021/17cd11d8a3e464c3dffdf796030fcbd8.png)

DocumentSegmentation 基于机器学习，运行在神经网络引擎，GPU 或者 CPU 上的；而 RectangleDetector 只能是在 CPU 上运行传统算法。

DocumentSegmentation 能识别出非矩形形状的文档、标签、符号等，而 RectangleDetector 只能基于边缘，查找交集。DocumentSegmentation 现在能够提供 mask 和顶角点坐标，而 RectangleDetector 只提供顶角点坐标。

另外，上面讲的 `VNDocumentCameraController` 现在已经把内部的核心检测换成了 `DocumentSegmentation`！



#### Demo

1. 获取图片中的文档

```swift
func recognize(image: UIImage) -> VNRectangleObservation? {
    guard let ciImage = image.ciImage ?? CIImage(image: image) else {
        print("创建ciimage失败!")
        return nil
    }

    let documentSegmentationHandler = VNImageRequestHandler(ciImage: ciImage, orientation: CGImagePropertyOrientation(image.imageOrientation), options: [:])
    let documentSegmentationRequest = VNDetectDocumentSegmentationRequest()
    documentSegmentationRequest.revision = VNDetectDocumentSegmentationRequestRevision1
    try? documentSegmentationHandler.perform([documentSegmentationRequest])

    guard let result = documentSegmentationRequest.results?.first, result.confidence > 0.7 else {
        print("未检测到文档")
        return nil
    }
    return result
}
```

之后通过得到的 observation 中的四个点（其实是 **VNRectangleObservation** 的属性，VNDetectDocumentSegmentationRequest 执行返回的结果和 RectangleDetector 得到的结果是同一个类），进行线的绘制

```objc
API_AVAILABLE(macos(10.13), ios(11.0), tvos(11.0))
@interface VNRectangleObservation : VNDetectedObjectObservation
  
@property (readonly, nonatomic, assign) CGPoint topLeft;
@property (readonly, nonatomic, assign) CGPoint topRight;
@property (readonly, nonatomic, assign) CGPoint bottomLeft;
@property (readonly, nonatomic, assign) CGPoint bottomRight;

@end
```

2. 裁剪、矫正

   我们可以把这部分区域或者 boundingBox 的区域裁剪出来，然后进行矫正。一般我们是把 boundingBox 的区域用 CIImage 的 `crop(to:)` 方法直接裁剪，然后再用 CIFilter 进行视觉矫正，核心代码如下：

```swift
let imageSize = ciImage.extent.size
let boundingBox = result.boundingBox.scaled(to: imageSize)
let correctedImage = ciImage
    .cropped(to: boundingBox)
    .perspectiveCorrect(by: result, imageSize: imageSize)


extension CIImage {
  
    /// 视觉矫正， 把 distort 的弄正
    /// - Parameter observation: Vision返回的结果
    /// - Returns: 新的CIImage
    func perspectiveCorrect(by observation: VNRectangleObservation, imageSize: CGSize?) -> CIImage {
        let imageSize = imageSize ?? self.extent.size
        let newImage =
        applyingFilter("CIPerspectiveCorrection", parameters: [
            "inputTopLeft": CIVector(cgPoint: observation.topLeft.scaled(to: imageSize)),
            "inputTopRight": CIVector(cgPoint: observation.topRight.scaled(to: imageSize)),
            "inputBottomLeft": CIVector(cgPoint: observation.bottomLeft.scaled(to: imageSize)),
            "inputBottomRight": CIVector(cgPoint: observation.bottomRight.scaled(to: imageSize))
        ])
        return newImage
    }
}
```

3. 后续处理

如果图片中有你需要的二维码或者文字，则继续把上一步取得的结果 correctedImage，作为下一步的输入：

```swift
// 二维码检测识别
let barcodesRequest = VNDetectBarcodesRequest()
barcodesRequest.revision = VNDetectBarcodesRequestRevision2
barcodesRequest.symbologies = [.qr]

// ocr
let textRequest = VNRecognizeTextRequest()
textRequest.revision = VNRecognizeTextRequestRevision2
textRequest.recognitionLevel = .accurate
textRequest.recognitionLanguages = ["zh-Hans", "zh-Hant"]

// 使用correctedImage创建新的handler
let requestHandler = VNImageRequestHandler(ciImage: correctedImage, options: [:])
// 开始识别、检测
try? requestHandler.perform([barcodesRequest, textRequest])

// 结果处理 - 二维码检测识别 
if let observation = barcodesRequest.results?.first,
   let payload = observation.payloadStringValue {
    print(payload)
}

// 结果处理 - ocr
if let observation = textRequest.results?.first,
   let canditate = observation.topCandidates(1).first {
    print(canditate.string)
}

// 控制台输出：
// http://weixin.qq.com/r/jHWvt6XEQPg-reHU9yCe
// 功能介绍
```

这里，我们一次性用 requestHandler 执行了两个请求，如果图片中有你需要的其他信息，你可以执行更多的请求。

> 实际应用中，记得不要放在主线程处理。Vision 的每一次检测都是比较耗时的。特别是 ocr！
