## SwiftUI on the Mac: The finishing touches


SwiftUI 能够让我们在很短的时间内，构建具有多功能的应用程序。但是用户倾向各种各样的方式使用我们的应用程序。一个优秀 Mac App 能够轻松应对。让我们来看看我们可以在保持原则的同时为每个人构建应用程序的一些方法。


首先，我们将通过处理来自操作系统和应用程序中内部的更改，来了解成为完全可定制的 Macs 应用程序意味着什么。通过处理用户拖拽对应的数据，来添加额外的工作流，这是提供灵活用户体验的一种方式。然后我们会探索如何把 Mac App 中的数据导出到操作系统。最后我们会支持 `continuity camera`(摄像头接力) ，将图片无缝倒入到 Mac App。

另一种符合 MacOS 用户直觉到交互方式提供添加新植物的方法——拖拽，在模版工程中已经提供了展示植物的 `Table`，可以直接做一些调整就能支持拖拽添加数据了。
在 `GardenDetail` 文件中 修改 `table` 计算属性中创建 `Table` 的方式如代码所示:

```Swift
    var table: some View {
        Table(selection: $selection, sortOrder: $sortOrder) {
            TableColumn("Variety", value: \.variety)

            TableColumn("Days to Maturity", value: \.daysToMaturity) { plant in
                Text(plant.daysToMaturity.formatted())
            }

            TableColumn("Date Planted", value: \.datePlanted) { plant in
                Text(plant.datePlanted.formatted(date: .abbreviated, time: .omitted))
            }

            TableColumn("Harvest Date", value: \.harvestDate) { plant in
                Text(plant.harvestDate.formatted(date: .abbreviated, time: .omitted))
            }

            TableColumn("Last Watered", value: \.lastWateredOn) { plant in
                Text(plant.lastWateredOn.formatted(date: .abbreviated, time: .omitted))
            }

            TableColumn("Favorite", value: \.favorite, comparator: BoolComparator()) { plant in
                Toggle("Favorite", isOn: $garden[plant.id].favorite)
                    .labelsHidden()
            }
            .width(50)
        } rows: {
            ForEach(plants) { plant in
                TableRow(plant)
                    .itemProvider { plant.itemProvider }
            }
            .onInsert(of: [Plant.draggableType]) { index, providers in
                Plant.fromItemProviders(providers) { plants in
                    garden.plants.insert(contentsOf: plants, at: index)
                }
            }
        }
    }
```

现在，我们已经为 `Table` 中的每一行都添加了 `.itemProvider` 修饰，这些行支持作为拖动源。但是如果没有人接受它，这就没有意义了。添加 `.onInsert` 来决定接受拖拽过来的内容的类型，在闭包内处理接受拖拽后的数据。现在打开一个新的窗口，在当前窗口侧边栏 中选中 `Backyard Flower Bed`,选择一些花朵拖拽到另外一个窗口中去。可以看到数据被添加到了另外一个窗口中的 `Indoor Plants` 中。拖拽是处理 App 内部数据的优秀方式。

![](https://otakumate.cn/wwdc2021/16238497248689/16241091171717.jpg)


那么要如何处理 App 和 操作系统间转移数据呢？我们的用户希望能够导出所有这些数据，也许是出于备份目的或导入另一个应用程序。添加一个主菜单项，用于以通用文件格式导出我们的数据库。它们可以与其他应用程序共享。在工程 `Main Menu` 路径下 `ImportExportCommands` 文件中，已经创建了用于支持导出数据的 `Commands` 组件：

```Swift
struct ImportExportCommands: Commands {
    var store: Store
    @State private var isShowingExportDialog = false

    var body: some Commands {
        CommandGroup(replacing: .importExport) {
            Section {
                Button("Export…") {
                    isShowingExportDialog = true
                }
                .fileExporter(
                    isPresented: $isShowingExportDialog, document: store,
                    contentType: Store.readableContentTypes.first!) { result in
                }
            }
        }
    }
}
```

`CommandGroup` 里 `Section` 中 `Button` 文字带有省略号暗示了用户，点击按钮会弹出对话窗口。用 `.fileExporter` 修饰初始化方法中 `document` 需要实现 `ReferenceFileDocument` 协议，示例工程中 `Store` 类型已经实现。最后在 `GardenApp` 中添加 `ImportExportCommands` 并传入 `store`变量。现在可以通过文件菜单中 `Export...` 选项导出数据到操作系统。

最后让我们看下如何用 iOS 设备拍照并直接导入到应用程序中。

`GardenDetail` 文件中 `body` 最后一行 添加 `.importsItemProviders` 如下：

```Swift
.importsItemProviders(selection.isEmpty ? [] : Plant.importImageTypes) { providers in
            Plant.importImageFromProviders(providers) { url in
                for plantID in selection {
                    garden[plantID].imageURL = url
                }
            }
        }
```
当用户选中某个植物时候，支持导入图片，否则不支持。 `Plant.importImageFromProviders` 闭包中会返回 iOS 设备拍照之后存储在 Mac App 沙盒内的图片文件地址。 更新对应的 Garden 实例的 `imageURL` 属性就会触发 UI 刷新。


在 `GardenApp` 文件中添加 `ImportFromDevicesCommands()`，使用 USB 连接 iPhone 到 Mac，运行程序，在工具栏中切换展示 植物列表的展示方式为画廊，此时选中需要添加图片的植物，在文件菜单中选择 `Import from iPhone -> take photo`, iPhone 就会进入拍照模式，拍照之后图片就会成为选中植物的封面。
