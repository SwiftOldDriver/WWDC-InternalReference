##  【WWDC21 10062/10289】通过 SwiftUI 构建优秀的 Mac App

SwiftUI 是一个多平台框架，其初衷是为了让开发者轻松地开发出能运行在所有 Apple 平台上的 APP 。当接触一个新概念或 API 的时候，应该停下来想想每个平台能从中得到什么。SwiftUI 优秀之处在于它的 API 和 相关设计理念适用于所有 Apple 平台。构建优秀 Mac APP 主要原则为—— **灵活**、**易于上手**、**可扩展**、**精细**

![](https://otakumate.cn/wwdc2021/16238497248689/16240780795877.jpg)

### 基本原则

优秀的 Mac App 应当是很灵活的，我们每个人能够根据使用 Mac App 的习惯进行做相应的调整。我们可以根据个人习惯通过键盘、鼠标、触控板、甚至 iPad 来使用 Mac App。Mac App 本身也可以按个人需求调整。比如可以调整侧边栏的项目、详细面板、显示模式、窗口的大小位置来自定义最合适自己的工作流。

![](https://otakumate.cn/wwdc2021/16238497248689/16240787994832.jpg)

Mac APP 使用与系统一致的控件和设计模式使得其非常容易上手。将创建新事物命令放在菜单中的 `File` 选项中、搜索栏在 Mac App 中保持外观一致、允许用户缩放窗口、在左边侧边栏中保持导航和层次结构、主体内容区域放中间、顶部和右侧摆放用户功能。自定义控件的时候也应当设计得与系统控件相似。这些都能让 Mac App 更符合用户直觉，易于上手。

![](https://otakumate.cn/wwdc2021/16238497248689/16240788560672.jpg)
 
Mac App 的 UI 可扩展性强。在大屏幕下且通常有多窗口显示，意味着可以在屏幕上展示更多信息，而无需将信息分层组织在很多导航堆栈中。使用带有大纲视图和缩略图预览的侧边栏、弹出窗口、在控件窗格之间切换的选项卡以及右侧使用 `DisclosureGroup` 控件收起、展示列表。

![](https://otakumate.cn/wwdc2021/16238497248689/16240788827733.jpg)

Mac App 很精细，他们不仅有大的窗口，而且他们的视图用紧凑布局，呈现出高密度的内容和控件。这些控件是设计给鼠标指针用的。随着内容密度的增加不会导致 Mac App 的复杂度增加。一个服务于简单任务 Mac App 也能成为一个优秀的 Mac App。

![](https://otakumate.cn/wwdc2021/16238497248689/16240789209043.jpg)
 
### 工程实践

接下来我们会在实践中实现这些想法。首先下载官方提供的[骨架工程](https://docs-assets.developer.apple.com/published/396ec33d22/BuildingAGreatMacAppWithSwiftUI.zip)

![代码示例1](https://otakumate.cn/wwdc2021/16238497248689/16240319708443.jpg)

打开 `BuildingAGreatMacAppWithSwiftUI` 目录下 `Session1/Part1-Start/GardenApp.xcodeproj` 工程。骨架工程有一些已经写好的 `View`。点击 `ContentView.swift` 文件。我们把`TextView` 嵌套在 `NavigationView` 中。去掉 `Padding Modifier` 然后把 `TextView` 中的文字改为 `Sidebar`，再加一行 `TextView` 文字为 `Plant Table`。

按住 `command` 点击 `Text("Side bar")`，选中 `Extract SubView`，Xcode会帮助生成一段代码，将当前选中的 `View` 封装在一个叫 `ExtractedView`的 `View` 中，然后重命名为 `SideBar`。
声明 `@EnvironmentObject var store: Store` 用于访问状态管理实例。使用 `List` 构建列表每项为有系统内置的叶子图片的 `Label` 控件。
![示例代码2](https://otakumate.cn/wwdc2021/16238497248689/16240335027295.jpg)

将 `SideBar` `body` 方法改为:

```Swift
 List {
    DisclosureGroup {
        ForEach(store.gardens(in: store.currentYear)) { garden in
            Label(garden.name, systemImage: "leaf")
        }
    } label: {
        Label("Current", systemImage: "chart.bar.doc.horizontal")
    }
 }
```

![](https://otakumate.cn/wwdc2021/16238497248689/16240337412320.jpg)

目前为止每次 App 启动的时候，`DisclosureGroup` 默认是未展开的，如果希望默认展开并且每次打开 App 都对 List 中每项的展开状态、选中状态有所记录。需要用到 `@SceneStorage` 这个 `Property Wrapper`。它会在每次 App 打开关闭的时候去读去存储对应 `key` 的值。
将 `ContentView` 内容调整如下：

```Swift

struct ContentView: View {
    
    @SceneStorage("selection") private var selectedGardenID: Garden.ID?

    var body: some View {
        NavigationView {
            SideBar(selection: $selectedGardenID)
            Text("Plant Table")
        }
    }
}

struct Sidebar: View {

    @EnvironmentObject var store: Store

    @SceneStorage("expansionState") var expansionState = ExpansionState()

    @Binding var selection: Garden.ID?

    var body: some View {
        List(selection: $selection) {
            DisclosureGroup(isExpanded: $expansionState[store.currentYear]) {
                ForEach(store.gardens(in: store.currentYear)) { garden in
                    Label(garden.name, systemImage: "leaf")
                        .badge(garden.numberOfPlantsNeedingWater)
                }
            } label: {
                Label("Current", systemImage: "chart.bar.doc.horizontal")
            }

            Section("History") {
                GardenHistoryOutline(range: store.previousYears, expansionState: $expansionState)
            }
        }
        .frame(minWidth: 250)
    }
}
```

现在每次打开 App 的时候，都会读取关闭前 `History` 的展开状态、以及 `List` 中的选中状态。

骨架工程中事先准备了 `GardenDetail` 来展示花园详细信息，`SwiftUI` 目前提供 `Table` 控件，可以展示列表数据，同时按某项数据来排序的功能。在 `GardenDetail` 新增 `table` 计算属性用于展示植物列表，并且将 `body` 内容替换为 `table`

为了能让 `Table` 按某一列的属性排序，需要额外传入 `selection` `sortOrder`，工程 `Views/TableColumns.swift`中提供了其他 `TableColumn`,将剩下的复制到 `table` 计算属性内。

运行 App 现在可以点选 `Table` 顶部某一列，来决定按其对应属性升降序排列了。

`Table` 支持搜索很简单， 添加 `searchable` 这一 `Modifier` 即可，骨架工程提供了展示模式切换的控件，在 `GardenDetail` 的 `body` 中 补全代码：

```Swift
    var body: some View {
        table
            .searchable(text: $searchText)
            .toolbar {
                DisplayModePicker(mode: $mode)
                Button(action: addPlant) {
                    Label("Add Plant", systemImage: "plus")
                }
            }
            .navigationTitle(garden.name)
            .navigationSubtitle("\(garden.displayYear)")
    }
```


![](https://otakumate.cn/wwdc2021/16238497248689/16240725975455.jpg) 

在 `GardenApp.swift` 文件中，向 `WindowGroup` 添加 `commands` Modifier，并创建 `SideBarCommand()`。 `SideBarCommand()` 功能为收起/展开 `SideBar`。除此之前，还需要添加植物、将植物标记浇水两个命令。因为 Mac App 是多 `Window` 的，在处理命令的时候，需要借助 `focusedSceneValue`、`@FocusedBinding`，来达到最后一次响应用户的 `Window` 才接收执行命令。


回到 `GardenDetail` 中向 `body` 中的 `table` 追加两个 `focusedSceneValue` 来传递给当前响应用户 `Window` 处理 添加植物、标记浇水命令所需参数：

```Swift
var body: some View {
        table
            .focusedSceneValue(\.garden, gardenBinding)
            .focusedSceneValue(\.selection, $selection)
            .searchable(text: $searchText)
            .toolbar {
                DisplayModePicker(mode: $mode)
                Button(action: addPlant) {
                    Label("Add Plant", systemImage: "plus")
                }
            }
            .navigationTitle(garden.name)
            .navigationSubtitle("\(garden.displayYear)")
    }
```

在 `Main Menu/PlantCommands.swift` 中，修改代码接收 `focusedSceneValue` 传递的参数:

```Swift
struct PlantCommands: Commands {
    
    @FocusedBinding(\.garden) private var garden: Garden?
    @FocusedBinding(\.selection) private var selection: Set<Plant.ID>?
    
    var body: some Commands {
        CommandGroup(before: .newItem) {
            AddPlantButton(garden: $garden)
        }
        
        CommandMenu("Plants") {
            WaterPlantsButton(garden: $garden, plants: $selection)
        }
    }
}
```

`AddPlantButton` `WaterPlantsButton` 都支持了 `keyboardShortcut` 这个 `Modifier`， 可以通过键盘直接执行命令：

```Swift
struct AddPlantButton: View {
    @Binding var garden: Garden?

    var body: some View {
        Button {
            garden?.plants.append(Plant())
        } label: {
            Label("Add Plant", systemImage: "plus")
        }
        .keyboardShortcut("N", modifiers: [.command, .shift])
        .disabled(garden == nil)
    }
}
```

最后回到 `GardenApp.swift` 文件，加上 `PlantCommands()`, 编译运行 App。

SwiftUI 构建的 App 会自动响应系统偏好设置的外观、字体等。但这还不够，使用 Mac 的人各有各的习惯，最好 App 自己提供一个设置面板来让用户设置。

打开 `BuildingAGreatMacAppWithSwiftUI/Session2/Part2-Start/GardenApp.xcodeproj` 工程，在 `GardenApp` 的 `WindowGroup` 内添加 `Settings`:

```Swift
@main
struct GardenApp: App {

    @StateObject private var store = Store()

    var body: some Scene {
        WindowGroup {
            ContentView()
                .environmentObject(store)
        }
        .commands {
            SidebarCommands()
            PlantCommands()
        }
        
        Settings {
            SettingsView()
                .environmentObject(store)
        }
    }
}
```

在 MacOS 上，应用程序通常会提供设置界面，工具栏图标允许在不同的面板之间切换，特别是如果应用程序提供了很多设置选项，这样可以分为不同的类别，方便用户查找设置。
工程中提供了空白的 `SettingsView`，先设置整体和 `GeneralSettings`，`Picker` 控件会根据内部组件提供 `tag` 来匹配对应数据。在构建 App 时，要能让 `Picker` 选中状态的每次 App 启动或操作系统更新都能保存下来。在 SwiftUI 中，使用 `@AppStorage` 能够做到这一点。`@AppStorage` 修饰的变量，会通过 `UserDefaults` 系统持久化。

```Swift
struct SettingsView: View {
    var body: some View {
        
        
        TabView {
            GeneralSettings()
                .tabItem {
                    Label("General", systemImage: "gear")
                }
            ViewingSettings()
                .tabItem {
                    Label("Viewing", systemImage: "eyeglasses")
                }
        }
        .frame(width: 400, height: 200, alignment: .top)
    }

    private struct GeneralSettings: View {
        
        @EnvironmentObject var store: Store
        @AppStorage("defaultGraden") var selection: Garden.ID?

        var body: some View {
            Form {
                Picker("Default Garden", selection: $selection) {
                    Text("None")
                        .tag(Garden.ID?.none)
                    ForEach(store.gardens)  { garden in
                        Text("\(garden.name), \(garden.displayYear)")
                            .tag(Garden.ID?.some(garden.id))
                    }
                }
                .fixedSize()
            }.padding()
        }
    }
    ...
}
```

切换到 `ContentView` 文件，添加 `@AppStorage("defaultGarden") private var defaultGardenID: Garden.ID?` 属性。修改计算属性 `selection`，通过 get 方法保证默认选中逻辑。

```Swift
private var selection: Binding<Garden.ID?> {
    Binding(get: { selectedGardenID ?? defaultGardenID }, set: { selectedGardenID = $0 })
} 
```

重新编译运行 Mac App, 打开主菜单选择 `preferences` 选项，选 `indoor plants 2021` 作为默认选项，新建 Window，验证逻辑。

Mac App 通常都是支持通过拖拽来添加数据的，在工程中已经提供了展示植物的 `Table`，简单做一些调整就能支持。

在 `GardenDetail` 文件中 修改 `table` 计算属性代码，通过 `itemProvider` 向外提供当前拖拽项 的信息，通过 `onInsert` 检测是否支持当前拖拽数据源并处理：

```Swift
    var table: some View {
        ...
        } rows: {
            ForEach(plants) { plant in
                TableRow(plant)
                    .itemProvider { plant.itemProvider }
            }
            .onInsert(of: [Plant.draggableType]) { index, providers in
                Plant.fromItemProviders(providers) { plants in
                    garden.plants.insert(contentsOf: plants, at: index)
                }
            }
        }
    }
```

出于备份或导入另一个应用程序的目的，用户有时候会需要导出数据。添加一个主菜单项，用于以通用文件格式导出 App 数据。在工程 `Main Menu` 路径下 `ImportExportCommands` 文件中，已经创建了用于支持导出数据的 `Commands` 组件：

```Swift
struct ImportExportCommands: Commands {
    var store: Store
    @State private var isShowingExportDialog = false

    var body: some Commands {
        CommandGroup(replacing: .importExport) {
            Section {
                Button("Export…") {
                    isShowingExportDialog = true
                }
                .fileExporter(
                    isPresented: $isShowingExportDialog, document: store,
                    contentType: Store.readableContentTypes.first!) { result in
                }
            }
        }
    }
}
```

`CommandGroup` 里 `Section` 中 `Button` 文字带有省略号暗示了用户，点击按钮会弹出对话窗口。用 `fileExporter` 修饰初始化方法中 `document` 需要实现 `ReferenceFileDocument` 协议，示例工程中 `Store` 类型已经实现。最后在 `GardenApp` 中添加 `ImportExportCommands` 并传入 `store`变量。现在可以通过文件菜单中 `Export...` 选项导出数据到操作系统。

![](https://otakumate.cn/wwdc2021/16238497248689/16241104356011.jpg)

最后让我们看下如何利用 `Continuity Camera` 特性，用 iPhone 拍照并照片直接导入到应用程序中。

在 `GardenDetail` 文件中 `body` 最后一行 添加 `importsItemProviders`：

```Swift
.importsItemProviders(selection.isEmpty ? [] : Plant.importImageTypes) { providers in
            Plant.importImageFromProviders(providers) { url in
                for plantID in selection {
                    garden[plantID].imageURL = url
                }
            }
        }
```

当用户选中某个植物时候，支持导入图片，否则不支持。`Plant.importImageFromProviders` 闭包中会返回 iOS 设备拍照之后存储在 Mac App 沙盒内的图片文件地址。 更新对应的 Garden 实例的 `imageURL` 属性就会触发 UI 刷新。

在 `GardenApp` 中添加 `ImportFromDevicesCommands()`，使用 USB 连接 iPhone 到 Mac，运行程序，在工具栏中切换展示 植物列表的展示方式为画廊，此时选中需要添加图片的植物，在文件菜单中选择 `Import from iPhone -> take photo`, iPhone 就会进入拍照模式，拍照之后图片就会成为选中植物的封面。
