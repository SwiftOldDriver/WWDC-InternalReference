## 概述  

![](https://images.xiaozhuanlan.com/photo/2021/4877d97e8483569348e2fad419ebd515.png)  

[Clips recording](https://developer.apple.com/videos/play/wwdc2021/10101/) 是 WWDC21 [ReplayKit](https://developer.apple.com/documentation/replaykit)  引入的新特性，用于在 iOS 以及 macOS 设备上记录游戏的精彩瞬间或是高光时刻（Clips recording 不支持 tvOS）。 

早在 WWDC15 上，iOS9 引入了 ReplayKit 。其目的是，提供给游戏开发者录制游戏的视频功能，用于社交分享以及推广。但早期提供的 ReplayKit 视频录制功能比较鸡肋且难用，官方也意识到ReplayKit的易用性较差。在接下来的 [WWDC16](https://developer.apple.com/videos/play/wwdc2016/601)、[WWDC17](https://developer.apple.com/videos/play/wwdc2017/606)、[WWDC18](https://developer.apple.com/videos/play/wwdc2018/601)、[WWDC20](https://developer.apple.com/videos/play/wwdc2020/10633)（[《WWDC20 10633 - 在 Mac 上使用 ReplayKit 录制屏幕内容》](https://xiaozhuanlan.com/topic/5127839640)）、[WWDC21](https://developer.apple.com/videos/play/wwdc2021/10101/) 大会上，都能看到对 ReplayKit 功能的扩展以及完善。目前 ReplayKit 已经具有屏幕录制、屏幕捕获、屏幕直播、高光时刻录制等功能, 已从“食之无味”进化到“真香”阶段。 

 In-App Screen Recording 一般用于长时间本地屏幕录制，系统内部会将音频和视频合成，然后通过系统封装的 RPPreviewViewController 保存到本地，对于开发者来说整个过程是一个黑盒；In-App Screen Capture 和 Screen Recording 一样用于长时间屏幕录制，在开启录屏后，开发者可以通过 handler 获得实时的 CMSampleBufferRef ，开发者拥有了一定的自主权，可以对其进行自定义的缩放、剪裁操作；In-App Screen Broadcast 适合用于在线直播；In-App Clips Recording 用于屏幕的短暂回放录制 。 

> 本文基于 WWDC21 [Session 10101](https://developer.apple.com/videos/play/wwdc2021/10101/) 梳理。  

## Clips recording 特性  

![](https://images.xiaozhuanlan.com/photo/2021/c925db28eba01db83be8076bd8cbd46b.png)  

Clips recording 具有的特性：   

 - 高清画质  
 - 对性能影响低  
 - 隐私保护  

对性能影响低以及隐私保护两个特性，在下文的 “Clips recording 流程” 和 “Clips recording 方法” 里会具体分析。  
  
在实际的游戏场景中，由于我们并不知道何时会出现高光时刻，我们不得不录制整个游戏过程（或是自己实现 Clip recording 的逻辑）。这样有可能会影响设备的性能以及内存问题，后期剪辑也非常麻烦，还得找到那个片段。有了官方提供的 Clip recording 功能，我们不用考虑何时出现高光时刻，只需要在高光时刻结束后，导出并保存到本地相册即可。

## Clips recording 应用  

### Clips recording 流程  

![](https://images.xiaozhuanlan.com/photo/2021/8b8c000898537eef8c907b3460e18256.png)  

Clips recording的几个主要步骤：  

 - 启动 app ，获取 RPScreenRecorder 对象，并初始化环境  
 - 系统通过 ReplayKit 开启音视频缓存  
 - 周而复始缓存最近 15 秒时长的音视频  
 - 导出并保存 Clip 视频文件  
 - 关闭 ReplayKit 音视频缓存   

 因为 Rolling Buffer 只缓存最新的 15 秒，过期的音视频 buffer 将会被遗弃掉，最新的 buffer 将会被加入缓存区域。这样，对内存的占用是相对稳定的一个值。若录制期间屏幕的场景频繁变更，内存也将会有一定的波动。在导出 clip 的时候，将会根据开发者设置的 duration 与15 秒的最小值，从 Rolling Buffer 区域取出最新的音视频流。

### Clips recording 方法  

![](https://images.xiaozhuanlan.com/photo/2021/c01171046888b9879f956393d436d4f7.png)  

Clips recording 提供的 api 非常简单，只有三个：开启 clip 缓存、关闭 clip 缓存、导出 clip 音视频，支持 [objc](https://developer.apple.com/documentation/replaykit/rpscreenrecorder?language=objc) 、[swift](https://developer.apple.com/documentation/replaykit/rpscreenrecorder) 语言。  

```  
// 获取 RPScreenRecorder 单例
+ (RPScreenRecorder *)sharedRecorder;

// 在使用前，先判断 ReplayKit 在当前设备上是否可用，也可通过 KVO 侦听该属性的状态变化
@property (nonatomic, readonly, getter = isAvailable) BOOL available;

// 是否录制麦克风声音
@property (nonatomic, getter = isMicrophoneEnabled) BOOL microphoneEnabled API_UNAVAILABLE(tvos);

// 是否将相机视频录制
@property (nonatomic, getter = isCameraEnabled) BOOL cameraEnabled API_AVAILABLE(ios(10.0), macos(11.0)) API_UNAVAILABLE(tvos);

/*!
@abstract 开启 Clip 音视频流缓存. 在开启录制前，系统将会弹出一个授权弹窗让用户确认是否录制。
@discussion handler 将在 clip recording 开始后被调用. 录制开启过程若遇到问题将会返回一个 NSError 。
*/
- (void)startClipBufferingWithCompletionHandler:(nullable void (^)(NSError *_Nullable error))completionHandler API_AVAILABLE(ios(15.0), macos(12.0)) API_UNAVAILABLE(tvos);

/*!
@abstract 关闭 Clip 音视频流缓存
@discussion handler 将会在 clip recording 关闭后被调用. clip recording 关闭若遇到问题将会返回一个 NSError 。
*/
- (void)stopClipBufferingWithCompletionHandler:(nullable void (^)(NSError *_Nullable error))completionHandler API_AVAILABLE(ios(15.0), macos(12.0)) API_UNAVAILABLE(tvos);

/*!
@abstract 导出 clip 音视频流 
@param url 包含 clip 绝对地址的 URL
@param duration 可设置不长于 15 秒的时间
@discussion 必须在 startClipBufferingWithCompletionHandler: 之后调用, 否则将会返回一个错误. 导出 duration 时长的最新 clip . 回调将会在导出成功后被调用. 
*/
- (void)exportClipToURL:(NSURL *)url duration:(NSTimeInterval)duration completionHandler:(nullable void (^)(NSError *_Nullable error))completionHandler API_AVAILABLE(ios(15.0), macos(12.0)) API_UNAVAILABLE(tvos);

```  
在实际使用 Clip 录制过程中，我们需要用到 RPScreenRecorder 类的以上属性和方法。使用过 ReplayKit 的同学应该了解，我们不需要去主动实例化 RPScreenRecorder 对象。而是直接通过 `+ (RPScreenRecorder *)sharedRecorder;` 单例方法获取实例。且需要先判断 ReplayKit 在当前设备上是否可用，是否录制麦克风音频，是否采集摄像头视频流。做完这些处理以后，再通过  `startClipBufferingWithCompletionHandler:` 方法开启 Clip 缓存。剩下的，我们要做的就是在合适的时机，让用户去触发 `exportClipToURL: duration: completionHandler:` 方法导出 Clip 短视频。导出的 Clip 视频不是以视频流格式，而是系统合成后的音频和视频 mp4 URL ，需要我们使用 PHPhotoLibrary 存储到相册里。 

以下为具体的代码调用：     

```  
// 开启 Clip 缓存
func startClipBuffering() {
    RPScreenRecorder.shared().startClipBuffering { (error) in
        if error != nil {
            print("Error attempting to start Clip Buffering")
            
            self.setClipState(active: false)
            
        } else {
            // There's no error when attempting to start a clip session. Update the clip state.
		    // do something
        }
    }
}

// 关闭 Clip 缓存    
func stopClipBuffering() {
    RPScreenRecorder.shared().stopClipBuffering { (error) in
        if error != nil {
            print("Error attempting to stop Clip Buffering")
        }
        // The sample calls this handler when stopClipBuffering finishes. Update the clip state.
        // do something
    }
}

// 导出 Clip 视频     
func exportClip() {
    let clipURL = getDirectory()
    // 设置 5 秒时长
    let interval = TimeInterval(5)
    
    print("Generating clip at URL: ", clipURL)
    RPScreenRecorder.shared().exportClip(to: clipURL, duration: interval) { error in
        if error != nil {
            print("Error attempting to start Clip Buffering")
        } else {
            // There isn't an error, so save the clip at the URL to Photos.
            self.saveToPhotos(tempURL: clipURL)
        }
    }
}

// 获取临时存储路径    
func getDirectory() -> URL {
    var tempPath = URL(fileURLWithPath: NSTemporaryDirectory())
    let formatter = DateFormatter()
    formatter.dateFormat = "yyyy-MM-dd-hh-mm-ss"
    let stringDate = formatter.string(from: Date())
    print(stringDate)
    tempPath.appendPathComponent(String.localizedStringWithFormat("output-%@.mp4", stringDate))
    return tempPath
}

// 存储到相册        
func saveToPhotos(tempURL: URL) {
    print("saveToPhotos clip at URL: ", tempURL)
    PHPhotoLibrary.shared().performChanges {
        PHAssetChangeRequest.creationRequestForAssetFromVideo(atFileURL: tempURL)
    } completionHandler: { success, error in
        if success == true {
            print("Saved to photos")
        } else {
            print("Error exporting clip to Photos")
        }
    }
}
```  

Apple 出于对用户隐私的保护，近几年升级了很多隐私权限策略。所以，我们在 Clip 录制中使用到麦克风、相机、相册等，都需要在 Info.plist 配置对应的权限使用说明。  

由于 Clip 录制功能最多只缓存 15 秒时长的音视频流，所以我们不用担心内存问题。另外，音频和视频的合成由系统层帮我们做好了，避免了开发者使用不当导致的视频泄露等相关问题，一定程度上提升了安全性。另外通过保存到相册这个操作，也一定程度保护了用户的隐私性。 

为了验证 Clip 录制对设备性能的影响，用了官方的 demo 在 mac 上进行测试。使用系统提供的 Instruments 工具，在不开启 Clip 缓存的时候，内存占用 8.2M 左右，开启缓存 15 秒后稳定在 10.4M 左右。每隔 30 秒左右，会有短暂的波动，瞬间拉到 16M ，又回到 10.4M 。除了偶尔的波动，总体来讲 Clip 缓存占用了 2M 左右，当然这跟设备界面的复杂度有关。若是王者荣耀的高帧率团战场景，远不止 2M 的缓存占用。另外，对 CPU 的影响，也取决于应用或游戏实际的运行场景。复杂的界面、光影粒子效果、帧率等因素，都会对实际的内存和 CPU 占用产生影响。  

#### 注意点：  
 - 必须要在 `startClipBuffering()` 成功开启的情况下才可以执行 `exportClip()` 高光时刻导出（ tips : 可以多次导出 ）；执行完 `stopClipBuffering()` 后，将无法导出高光时刻；
 - 高光时刻 clip 时长取 `startClipBuffering()` 开启时长、自定义 duration 间隔、15 秒，三个维度的最小值；所以 clip 时长最多 15 秒；  
 - clip 录制是与屏幕录制/直播互斥的，当你需要长时间屏幕录制或是直播互斥，需要执行 `stopClipBuffering()` ；
 - 支持 app 内的音频、视频以及麦克风的音频录制； 
 - 存储到相册，需要在 Info.plist 里配置权限申明；
 - 测试需要使用 mac 或是 iPhone、iPad真机，不支持模拟器上运行。

### 游戏内使用场景  

![](https://images.xiaozhuanlan.com/photo/2021/b526538145936580b9a464f04346e96a.png)  

图示案例是官方示例游戏 Fox2 游戏内的高光时刻记录。  

考虑到 Clip 功能为屏幕以及音频录制，所以使用场景更适合游戏；而对于应用，可能使用的场景是记录视频通话片刻、录制 app 内正在播放的电影某个精彩片段、记录 bug 复现流程。

#### 实际使用的场景：  

 - 解锁某个成就；
 - 游戏内的精彩瞬间，例如：王者荣耀超神或五杀高光时刻、和平精英单人灭队或吃鸡瞬间、灌篮高手投入制胜一球、原神通关岩副本等等；  
 - 抽到 SSR 卡，或是礼盒宝箱开出极品装备瞬间；  
 - 当然也可以用来记录 bug 场景。  

#### 触发方式：   
  
 - 一些显性的 UI ，比如悬浮半透明按钮，为的是不影响应用或游戏的主体功能展示；
 - 特殊的手势操作，比如长按 3 秒触发；
 - 摇一摇 或 语音指令触发；
 - 通过一些预设的逻辑触发，例如王者荣耀完成 5 杀；
 - 通过游戏输入控制器，例如游戏手柄上的按键。 

### 配合游戏手柄  

![](https://images.xiaozhuanlan.com/photo/2021/a2d302893e69b4583647f1bdfe8a1f87.png)  

今年苹果官网对 iPhone、iPad、Mac 和 Apple TV 上的游戏优化了对 [物理输入控制器](https://developer.apple.com/videos/play/wwdc2021/10081/) 的支持，比如：游戏手柄。可以通过游戏手柄上的分享按键，快速的对游戏内的高光片段进行录制、分享。另外，[Game Controller](https://developer.apple.com/documentation/gamecontroller)    框架已经内置了 Clips recording 功能。
