## 概述  

本次 [Session](https://developer.apple.com/videos/play/wwdc2021/10102/?time=694) 基于 Session 10102 整理。

其中，主要介绍了两个关于 Location 的新特性：

1. 在 watchOS 上新增了基于地理位置的用户通知功能
2. 引入 CoreLocation Button，提供全新的一次性地理位置授权能力。在保障用户隐私的前提下，进一步优化了用户体验

## 基于地理位置的用户通知

### 介绍

watchOS 的 App，可以在用户达到指定地点后，委托系统进行推送提醒。比如当用户回到家时，App 提醒用户该给狗狗喂狗粮。

之前，App 需要获取 “Always” 地理位置授权之后，才可以提供类似功能；watchOS 8 中，App 无需获得 “Always” 授权，系统可以代为判断地理位置，并发出推送提醒。

可以考虑在以下的需求情境中，使用这项新特性：

用户可以选择非 “Always” 的地理位置授权，防止地理位置功能被滥用，并在此基础上，仍允许 App 为用户提供基于地理位置的个性化功能。

想要了解更多详细内容，请关注 Session [What's New in watchOS](https://developer.apple.com/videos/play/wwdc2021/10002)。

## Location Button

### iOS 14

在正式介绍 Location Button 前，先谈谈在 iOS 14 上，Location 授权的基本情况。

#### 介绍

在 iOS 中，授权状态主要包含以下六种：

```objc
typedef NS_ENUM(int, CLAuthorizationStatus) {
	// 用户暂未做出选择，是否使用地理位置
	kCLAuthorizationStatusNotDetermined = 0,

	// 因系统原因导致无法使用，用户无法修改
	kCLAuthorizationStatusRestricted,

	// 用户选择拒绝授权 App 使用地理位置
	kCLAuthorizationStatusDenied,

	// 用户允许 App 一直使用地理位置
	kCLAuthorizationStatusAuthorizedAlways API_AVAILABLE(macos(10.12), ios(8.0)),

	// 在 App 在被用户使用期间，使用地理位置
    // 在 iOS, tvOS and watchOS 被允许使用，在 MacOS 上被禁用
	kCLAuthorizationStatusAuthorizedWhenInUse API_AVAILABLE(ios(8.0)) API_UNAVAILABLE(macos),

    // 用户授权应用使用地理服务，
    // 在 iOS, tvOS and watchOS 禁用，只允许在 MacOS 上使用
	kCLAuthorizationStatusAuthorized 
};
```
众所周知，App 必须获取到用户授权，才可以使用地理位置功能。

在 iOS 14 中，当 App 请求获取地理位置授权时，系统会弹出系统授权弹窗。用户可以选择允许和不允许。

如果用户希望允许授权，系统还可以提供授权次数和精度两个维度，让用户进行选择：

1. 授权次数：
   1. Allow Once：一次性授权，授权只在 App 当前生命周期内生效，冷启后需要重新授权
   2. Allow While Using App：授权在 App 使用期间内一直生效，即使冷启，也不需要重新授权
2. 获取的地理位置精度：
   1. 精确（Precise）：获取用户当前的精确位置
   2. 粗略（Coarse）：获取用户当前粗略的地理位置，可以很好地保护用户隐私

![](https://images.xiaozhuanlan.com/photo/2021/dd8fc7167cdc516c7a5ce8b31e5fe2dd.png)

#### 存在问题

用户出于隐私上的考虑，不希望授权给 App 永久的地理位置权限，选择了 Allow Once 选项。

每次冷启动后，当 App 需要使用 Location 时，系统授权弹窗都会重新弹出，提示用户授权。当用户厌倦了无穷尽的系统弹窗后，就有可能干脆利落地选择了 Don't Allow 选项。App 虽然没有能力让系统授权弹窗出现了。但 App 仍会弹出提示，提醒用户去到设置页中修改授权开关。

啊，多么糟糕的用户体验啊。

### iOS 15 推出 Location Button

为了解决上面存在的问题， Location Button 在 iOS 15 闪亮登场。

在 iOS 15 中，UIKit 引入 [CLLocationButton](https://developer.apple.com/documentation/corelocationui)，在 SwiftUI 引入 LocationButton。后文主要以介绍 CLLocationButton 为主。

#### 介绍

那么当用户点击了 CLLocationButton，会发生什么？

1. 系统会主动检查 App 是否具备地理位置权限，如果状态是 Not Determined，则弹出如下图的一次性授权弹窗（这里的一次性概念与 iOS 14 已有的 Allow Once 概念相同）
2. 如果上一步的授权允许，或者已经具备授权能力，则触发 button 本身的按钮点击事件（比如地理位置请求）

![](https://images.xiaozhuanlan.com/photo/2021/a25c351e494ee46255085c572b46603a.png)

当用户点击下方蓝色的 Current Location 按钮，并点击 OK 授权成功后，可以看到左上角出现蓝色箭头按钮，代表 App 正在收集 Location 信息。

![](https://images.xiaozhuanlan.com/photo/2021/5279bca34585e7104f6e015f211ffa3a.png)

#### 适用平台

1. watchOS
2. iOS
3. macOS with Catalyst
4. iPad apps on Mac

#### API 介绍

CLLocationButton 隶属于 CoreLocationUI 框架。与 UIButton 类似，CLLocationButton 继承自 UIControl，因此具备了点击事件的能力（addTarget:action:forControlEvents:）。

CLLocationButton 拥有 4 个特有的属性：

```objc
/// icon 样式
@property (nonatomic, readwrite) CLLocationButtonIcon icon;
typedef NS_ENUM(NSInteger, CLLocationButtonIcon) {
	CLLocationButtonIconNone = 0,
	CLLocationButtonIconArrowFilled,
	CLLocationButtonIconArrowOutline,
};

/// label 内容
@property (nonatomic, readwrite) CLLocationButtonLabel label;

typedef NS_ENUM(NSInteger, CLLocationButtonLabel) {
	CLLocationButtonLabelNone = 0,
	CLLocationButtonLabelCurrentLocation,
	CLLocationButtonLabelSendCurrentLocation,
	CLLocationButtonLabelSendMyCurrentLocation,
	CLLocationButtonLabelShareCurrentLocation,
	CLLocationButtonLabelShareMyCurrentLocation,
};

/// 圆角大小
@property (nonatomic, readwrite) CGFloat cornerRadius;

/// 字体大小
@property (nonatomic, readwrite) CGFloat fontSize;

```

由上可以看出，CLLocationButton 的 icon 和 label 均只提供了有限的枚举类型选择。留给 App 自定义样式的空间会比较小。

以下定制化 UI 的示例代码：

```objc
float BUTTON_HEIGHT = 54;

CLLocationButton *locationButton = [[CLLocationButton alloc] initWithFrame:CGRectMake(0, 0, BUTTON_WIDTH, BUTTON_HEIGHT)];
locationButton.icon = CLLocationButtonIconArrowFilled;
locationButton.label = CLLocationButtonLabelCurrentLocation;
locationButton.backgroundColor = [UIColor whiteColor];
locationButton.tintColor = UIColor.blueColor;
locationButton.cornerRadius = BUTTON_HEIGHT / 2;
```

#### 如何调试 button

请注意，如果 Location Button 没有正确渲染的话，将会影响其正常功能的使用，系统将无法为 App 获取授权。

这里罗列了几种可能产生问题的场景：

1. tintColor 和 backgroundColor 对比度太低的情况
2. alpha 值过低，以致用户无法看清楚 button
3. 文案不能完整展示，文案宽度小于 button 宽度

那作为开发者来说，如何确保能发现所有这一类问题呢？别担心，Xcode 会给你提供帮助。

##### 对比度过低

比如有一个 backgroundColor 是 white、tintColor 是 black 的 Location Button，如下图，可以正常渲染。

![](https://images.xiaozhuanlan.com/photo/2021/f3920ddf9f94b488a81134450fdfc09b.png)

将 tintColor 修改为 white 之后，会发现箭头和背景融为一体，作为用户来说是无法分别出这个按钮里的 icon 样式的，这也就违背了对比度太低的原则。

如果是通过 **storyboard** 或 **xib** 创建 Location Button，程序无需运行，就可以在 Xcode 右侧导航栏中的 Issue Tab 下 Buildtime issue 下看到提示。

![](https://images.xiaozhuanlan.com/photo/2021/d380afe1575900f3ce297c2ae48fcc65.png)

笔者补充：

如果开发者是通过**纯代码**创建了有问题的 Location Button。在运行到这段代码时，控制台会输出 **Error: #button failed due to insufficient contrastRatio between tintColor and backgroundColor**。

##### alpha 值过小

那如果我们把按钮的 alpha 或 opacity 属性值设置得过小，比如下图中的 Location Button 是不够清晰的。

Xcode 同样会给出提示进行修改。

![](https://images.xiaozhuanlan.com/photo/2021/b8cc4b21858a8b5a150b59d4bd0f00a5.png)

##### 尺寸不合适

这种情况，一般会发生在国际化多语言 App 中。

在下图中，英文文案可以完整展示，没有问题。

![](https://images.xiaozhuanlan.com/photo/2021/fab36fb7f124c3b40b0667e13da3c055.png)

但如果把语言换成德文，文案就不能完整展示了。

![](https://images.xiaozhuanlan.com/photo/2021/35366cccca89471d315d5973a2bd5e69.png)

对此，Apple 的建议：尽量覆盖测试各种极端情况，以确保功能的正确性。

这部分内容可参考老司机周报分享过的一篇文章：[国际化适配：使用 Xcode 构建有助于本地化的布局](https://mp.weixin.qq.com/s/9iUNek72jZoqNJVLbPViVw)。

#### 对授权状态的影响

在 iOS 14 中设置中授权选项，包括“从不”、“下次询问”、“当使用 App 时”三种。

![](https://images.xiaozhuanlan.com/photo/2021/cbfdeec97e0afd0e44b2e5bdc173ba8e.png)

在 iOS 15 中，“下次询问”修改为“下次询问或当我分享”。这里的“当我分享”就是代表 Location Button 的授权，从这里，也可以看出 Location Button 带来的是一种一次性授权。

![](https://images.xiaozhuanlan.com/photo/2021/c9d5f19d18fcfaf1bd28c59d5c626774.png)

接下来讲讲在引入 Location Button 新特性之后，会对 App 已有的授权状态产生什么影响。

##### status 为 Not Determined

原有 status 为 Not Determined，当用户点击 Location Button 后，会弹出下图的系统弹窗，提示用户授权“当我分享”。授权成功后，status 会变更为 When In Use。当 App 冷启后，状态又会被重置为 Not Determined。

![](https://images.xiaozhuanlan.com/photo/2021/a25c351e494ee46255085c572b46603a.png)

##### status 为 When In Use

当 App 已经获取 While Using the App 权限时，Location Button 不会改变权限状态，而只是像普通 UIButton 一样，去响应点击事件。

##### status 为 Denied

Denied 状态表示用户选择了不允许授权。在 iOS 14 中，也就意味着 App 再没有机会唤起系统弹窗去让用户授权，而是只能提示前往设置中去手动设置。

在 status 为 Denied 的前提，使用 Location Button，可以让授权状态从 Denied 重置为 Not Determined。App 又重新拥有了唤起系统授权弹窗的能力。这也正好解决了前文提到 iOS 14 存在的问题，感谢 iOS 15 给了 App 新的机会~

### 对于开发者的建议

在开发 Location 功能，开发者需要好好考虑一下，是否仍然保留原有的授权方式，还是说可以使用 Location Button 来满足需求。

想要了解更多关于 Location 的设计原则，推荐可以看一下去年的 session，[WWDC20：Design for location privacy](https://developer.apple.com/videos/play/wwdc2020/10162/)。

里面给出了三条考虑的设计原则：

1. 把用户的控制权放在首位
2. 确保 App 的透明度来获取用户的信任
3. 以 App 提供的功能和价值来交换用户的授权

#### 补充

在 iOS 14 中，如果开发者需要使用 Location 功能，必须在 plist 对 Location 进行声明后，才可以使用。

在学习官方 demo [Parker Finder demo](https://developer.apple.com/documentation/corelocationui/sharing_your_location_to_find_a_park) 的过程中，却发现在 plist 并没有对 Location 权限进行声明，便可以使用 Location Button 的功能。目前尚不确定这个是苹果有意为之，还是属于 beta 版本的 bug，笔者也会持续关注。

## 总结

本次 session 主要介绍了两个新特性：

1. 在 watchOS 8 上，不依赖于 App 自身地理位置授权状态的前提下，新增了基于地理位置的用户通知功能。
2. 引入 CoreLocation Button，在保护用户隐私的前提下，进一步提升用户体验，对于开发者而言，是一个不错的新选择。


最后想要感慨一下，苹果对用户隐私的保护和关注，可谓是永无止境。从 iOS 13 新增了允许一次访问位置的临时授权，到 iOS 14 位置精度的控制，再到今年 iOS 15 中 Location Button 的推出，无不体现苹果对于用户隐私和用户体验的重视程度。
