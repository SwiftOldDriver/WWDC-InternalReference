# WWDC21 10278 - 使用Core Haptics为App增加优秀的音频触觉体验
## 前言
2016年，苹果发布了iPhone6s/6s plus, 其首次搭载了线性震动马达---Tapic Engine, 之后的几年里，这颗体积不小的硬件一直在新iPhone寸土寸金的主板上占据一席之地。
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624512933487-27cf3488-9b00-4694-a74f-bf2fe4eac5bc.png#clientId=uec7a2c07-69a2-4&from=paste&height=500&id=eP11l&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1000&originWidth=1656&originalType=binary&ratio=2&size=978240&status=done&style=none&taskId=u52df9bc3-f6d1-4ce6-93d5-fa9526dda6d&width=828)
​

基于Tapic Engine, iOS系统实现了丰富的触觉体验：比如在iPhone7上仿佛可以按压下去的虚拟Home键、3D Touch / Haptic Touch的真实触觉反馈、以及系统闹钟拨动时间滚轮那短促有力的咔哒声等...
​

事实上，除了iOS自身的应用外，苹果还为我们开发者提供了多种方式来丰富App内的触觉体验

1. 使用自带触觉反馈的UIKit组件：比如UISwitch、UISlider、UIPickerView
1. iOS10开放了UIFeedbackGenerator系列类，可以非常方便的实现简单的振动反馈效果
1. iOS13开放了Core Haptics， 可以实现自定义的触感振动。

![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624709679793-44a11abd-b9a2-4def-9514-3aaa4addf0de.png#clientId=u7b7561dd-8eed-4&from=paste&height=578&id=u237c16a2&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1156&originWidth=1760&originalType=binary&ratio=2&size=125507&status=done&style=none&taskId=ue32141a0-1aab-4e8c-a684-68fd275536a&width=880)
​

前面两种使用方式都非常简单，相信不少同学都已经在开发中用过了，在此不再赘述。
今天这篇文章主要结合WWDC19 <Introducing Core Haptics> 和 WWDC21 <Practice audio haptic design>会分享一下Core Haptics的关键概念和使用要点，希望能给大家在实现相关效果时提供参考。




## 1.Core Haptics 关键概念
### 1.1  持续触觉和瞬时触觉事件
Core Haptics里的触觉事件可以分为两种：

1. 持续(Continuous)事件：持续的振动，可以设置持续时间
1. 瞬时(Transient)事件：类似轻敲或者撞击的短暂事件

![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624710194276-5fa6cb6e-963d-478d-94b2-9de76a0f74ba.png#clientId=u7b7561dd-8eed-4&from=paste&height=536&id=u49138ff7&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1072&originWidth=1852&originalType=binary&ratio=2&size=220515&status=done&style=none&taskId=u1d2557aa-f039-4779-9f1e-4323a980c0d&width=926)
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624711073150-c94c9181-1090-4ccb-b445-fc6410b2a237.png#clientId=u6b4cc879-5e39-4&from=paste&height=536&id=u28cf0768&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1072&originWidth=1872&originalType=binary&ratio=2&size=78287&status=done&style=none&taskId=u7d6ad05c-4256-42b6-bca0-4c1922756f9&width=936)
### 1.2  振动的锐度和强度
无论是持续事件还是瞬时事件，我们都可以控制事件的锐度(sharpness)和强度(intensity)。
锐度：可以用来传达一种柔软、圆滑、有机的体验，或者是一种松脆、清晰、机器驱动的体验
强度：触觉的强度，值越大，感受到的振动越强
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624858146414-e5d4d952-4e90-43dc-a670-e7c7faf5a5f0.png#clientId=uec7a2c07-69a2-4&from=paste&height=400&id=ue103d7d5&margin=%5Bobject%20Object%5D&name=image.png&originHeight=736&originWidth=747&originalType=binary&ratio=2&size=98762&status=done&style=none&taskId=u3585cb47-e5fd-4024-8e24-28a1a747dd2&width=405.5)


## 2.如何使用Core Haptics构建自定义的触觉反馈


### 2.1. 检查设备兼容性
不是所有的设备都支持Core Haptics, 比如iPad、iPod Touch就不支持。
我们可以通过以下代码来检测当前设备的支持情况：
```swift
var supportsHaptics: Bool = false
...
// 检测设备是否支持触觉反馈
let hapticCapability = CHHapticEngine.capabilitiesForHardware()
supportsHaptics = hapticCapability.supportsHaptics
```
如果用户的设备不支持触觉反馈，我们可以选择使用具有更强的音频或其他多媒体替代触觉反馈。
​

### 2.2. 创建并设置触觉引擎(CHHapticEngine)
#### 2.2.1 创建一个触觉引擎
触觉引擎(CHHapticEngine)是应用程序与触觉硬件的交互接口，我们需要在播放触觉反馈前创建好一个触觉引擎，并且需要管理它的声明周期，以保证其不会播放触觉反馈过程中被释放。
我们可以通过如下代码创建一个触觉引擎：
```swift
// ViewController.swift
var engine: CHHapticEngine!
...
override func viewDidLoad() {
    ...
    // Create and configure a haptic engine.
    do {
        engine = try CHHapticEngine()
    } catch let error {
        fatalError("Engine Creation Error: \(error)")
    }
    ...
}
```
以上代码在ViewController类里声明了一个engine属性以保存CHHapticEngine实例，并且在viewDidLoad方法里创建了CHHapticEngine实例并赋值给engine。
​

**注意：**
CHHapticEngine不是一个单例，你可以应用内创建多个实例，这些实例都是独立运行的，彼此互不影响；
Core Haptics 是线程安全的，这意味着您可以在单独的线程上执行操作。
​

#### 2.2.2 设置Reset Handler
当Core Haptics 从错误中恢复时会调用CHHapticEngine的resetHandler，在这里我们应该做以下事情：

1. 重启引擎
1. 重新注册之前注册过的自定义音频资源
1. 重新创建之前创建过的CHHapticPatternPlayer
```swift
// The reset handler provides an opportunity to restart the engine.
engine.resetHandler = {
    print("Reset Handler: Restarting the engine.")
    do {
        // Try restarting the engine.
        try self.engine.start()
        // Register any custom resources you had registered, using registerAudioResource.
        // Recreate all haptic pattern players you had created, using createPlayer.
    } catch {
        fatalError("Failed to restart the engine: \(error)")
    }
}
```
#### 2.2.3 设置stoppedHandler
当外部因素导致触觉引擎停止时，例如来电导致的音频终端或者应用程序退到后台，Core Haptics 会调用stoppedHandler。
```swift
// The stopped handler alerts engine stoppage.
engine.stoppedHandler = { reason in
    print("Stop Handler: The engine stopped for reason: \(reason.rawValue)")
    switch reason {
    case .audioSessionInterrupt: print("Audio session interrupt")
    case .applicationSuspended: print("Application suspended")
    case .idleTimeout: print("Idle timeout")
    case .systemError: print("System error")
    @unknown default:
        print("Unknown error")
    }
}
```
注意：
stoppedHandler仅仅当外部原因导致引擎停止才会调用，通过stop(completionHandler:)方法手动停止引擎是不会调用的。
​

### 2.3 创建CHHapticPattern
CHHapticEvent：一个独立事件，可以指定相对时间、类型、时长等参数
CHHapticPattern: 一个播放模式, 由多个CHHapticEvent组成
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624815859730-b15bbc17-8cc9-40f6-863e-8b4c8f4eadc6.png#clientId=u6b4cc879-5e39-4&from=paste&height=469&id=uae5e5de8&margin=%5Bobject%20Object%5D&name=image.png&originHeight=938&originWidth=2282&originalType=binary&ratio=2&size=120311&status=done&style=none&taskId=u19716994-9fdc-46a5-bcd8-06b38cbf335&width=1141)
如下代码初始化了一个持续振动事件event1，并且指定其intensity为0.5，相对开始时间为0(即立即开始)，时长为1秒；
初始化了一个瞬时振动事件event2, 并设置其intensity和sharpness都为1，相对开始时间为1(即在最末尾开始)
最后根据event1和event2创建了pattern。
```swift
let event1 = CHHapticEvent(
            eventType: .hapticContinuous,
            parameters: [CHHapticEventParameter(parameterID: .hapticIntensity, value: 0.5)],
            relativeTime: 0,
            duration: 1)
        
let event2 = CHHapticEvent(
            eventType: .hapticTransient,
            parameters: [CHHapticEventParameter(parameterID: .hapticIntensity, value: 1),
                         CHHapticEventParameter(parameterID: .hapticSharpness, value: 1)],
            relativeTime: 1)

let pattern = try CHHapticPattern(events: [event1, event2], parameters: [])
```


### 2.4 创建CHHapticPatternPlayer并启动播放
通过如下代码，我们通过引擎传入了pattern创建了一个CHHapticPatternPlayer实例, 然后调用该实例的start(atTime:)方法立刻启动了播放。
```swift
try self.engine.start()
let player = try engine.makePlayer(with: pattern)
try player.start(atTime: CHHapticTimeImmediate)
```
### 2.5 通过AHAP文件来启动播放
对于简单的振动，我们可以通过如2.3中的代码所示创建pattern, 但是对于比较复杂的，有很多阶段的振动，这种方式未免太过繁琐。
对此苹果给出的解决方案是使用AHAP(Apple Haptic and Audio Pattern )文件, 是类似于JSON的文件格式，用于定义触觉模式。我们可以使用以下结构定义模式：
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624900174171-cc3419a2-d712-4ea7-bd42-4d9172965de5.png#clientId=uc9f5664e-89ed-4&from=paste&height=539&id=uaba785ab&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1278&originWidth=1422&originalType=binary&ratio=2&size=135481&status=done&style=none&taskId=u2639e811-0365-42c9-81f9-95de51de0e6&width=600)
在Mac OS 12上可以图形化预览AHAP文件：
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624902150050-133fe102-c0b4-4fa0-9a12-652688483fcd.png#clientId=uc9f5664e-89ed-4&from=paste&height=621&id=u8e266224&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1242&originWidth=2234&originalType=binary&ratio=2&size=580294&status=done&style=none&taskId=u9a0dd8fa-ff72-4534-8641-8a4bd3a6731&width=1117)
通过AHAP文件启动播放的代码非常简单：
```swift
// ahap文件路径
let path = Bundle.main.path(forResource: filename, ofType: "ahap")
// 启动播放
try engine.playPattern(from: URL(fileURLWithPath: path))
```
