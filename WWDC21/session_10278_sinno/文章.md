## 1. Core Haptics 关键概念


### 1.1 持续触觉和瞬时触觉事件


Core Haptics 里的触觉事件可以分为两种：


1. 持续( Continuous )事件：持续的振动，可以设置持续时间
1. 瞬时( Transient )事件：类似轻敲或者撞击的短暂事件



![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1626659725158-88426671-1ac4-4efd-a206-1c9b38a51c37.png#clientId=u072c19bb-82f0-4&from=paste&height=282&id=u223375f3&margin=%5Bobject%20Object%5D&name=image.png&originHeight=564&originWidth=1768&originalType=binary&ratio=1&size=179487&status=done&style=none&taskId=u762a6bc0-bf1e-453c-a14b-042e978c68c&width=884)




### 1.2 振动的锐度和强度


无论是持续事件还是瞬时事件，我们都可以控制事件的锐度( sharpness )和强度( intensity )。
​


- 锐度：可以用来传达一种柔软、圆滑、有机的体验，或者是一种松脆、清晰、机器驱动的体验
- 强度：触觉的强度，值越大，感受到的振动越强

### 1.3 Core Haptics 基本元素：触觉引擎、播放器、模式、事件


在 Core Haptics 里有4个最基本的元素，了解了这些元素对掌握触觉反馈很有帮助：

- **CHHapticEngine**：



CHHapticEngine代表着触觉引擎， 是应用程序与 Tapic Engine 硬件交互的接口。
需要注意的是，CHHapticEngine 不是一个单例，你可以应用内创建多个实例，这些实例都是独立运行的，彼此互不影响。
​


- **CHHapticPlayer**



CHHapticPlayer 代表播放器对象，用于播放控制，比如开始、停止、暂停等。
在 Core Haptics 中以协议的方式出现，如CHHapticPatternPlayer 和 CHHapticAdvancedPatternPlayer。
​


- **CHHapticPattern**： 



CHHapticPattern 可以理解为触觉播放的波形，其由多个 CHHapticEvent 以时间维度排列组成，各个 	CHHapticEvent 可以在时间上可以重叠，也可以有一段时间完全没有事件，当 pattern 被播放时，会按时间顺序触发各个事件，如下图所示：
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1626883018453-b62de013-36bc-42ac-aea8-3521fe70ea4b.png#clientId=u1237c635-12d3-4&from=paste&height=308&id=u6e7bf5ca&margin=%5Bobject%20Object%5D&name=image.png&originHeight=615&originWidth=1496&originalType=binary&ratio=1&size=89995&status=done&style=none&taskId=u2a3b9d5a-0ccf-4284-8b2c-538ecc65864&width=748)​
​


- **CHHapticEvent**：



     CHHapticEvent 代表单个触觉或声音事件，其中触觉事件就包含持续事件和瞬时事件。
​

通过设置事件的 relativeTime 属性来指定事件的开始时间。 通过设置事件的持续时间属性来指定事件的长度。 	设置可选参数以自定义事件属性， 例如触摸事件的强度和锐度。

## 2. Core Haptics 实践


### 2.1 一个实例
上面的概念看起来好像很复杂，实际上实现起来非常简单！
​

下面这不到40行代码，就已经实现了一个典型的 Core Haptics 应用实例：
```swift
import UIKit
import CoreHaptics

class ViewController: UIViewController {
  var engine: CHHapticEngine?
   
  override func viewDidLoad() {
    super.viewDidLoad()
    // 1. 初始化engine并启动，注意启动引擎应尽可能提前以降低延迟
    engine = try? CHHapticEngine()
    engine?.start(completionHandler:nil)
  }

  @IBAction func playButtonClick(_ sender: Any) {
    // 在播放前尝试启动引擎，以避免引擎被停止(比如退到后台)
    engine?.start(completionHandler: nil)
    // 2. 初始化event
    let event1 = CHHapticEvent(
      eventType: .hapticContinuous,
      parameters: [CHHapticEventParameter(parameterID: .hapticIntensity, value: 0.5)],
      relativeTime: 0,
      duration: 0.5)
     
    let event2 = CHHapticEvent(
      eventType: .hapticTransient,
      parameters: [CHHapticEventParameter(parameterID: .hapticIntensity, value: 1),
             CHHapticEventParameter(parameterID: .hapticSharpness, value: 1)],
      relativeTime: 1)
    // 3. 构造pattern
    let pattern = try! CHHapticPattern(events: [event1, event2], parameters: [])
    // 4. 构造player
    let player = try! engine?.makePlayer(with: pattern)
    // 5. 启动播放
    try? player?.start(atTime: CHHapticTimeImmediate)
  }
}
```
上面的代码构造了一个 ViewController，其包含一个按钮，点击即会播放我们设置好的震动模式。
​

整个震动模式总时长 1 秒，前面 0.5 秒是持续触觉事件，最后在 1 秒处有一个瞬时事件。

4. 构造一个player

### 2.2 需要注意的问题


在实际使用中，我们还需要注意以下几个问题：
​


1. **判断设备是否支持 Core Haptics**



并不是所有的 iOS 设备都支持 Core Haptics 的功能（比如 iPad 就不支持），在使用前我们应该进行判断，对不支持的设备应该以其它合适的方式进行反馈。
​

我们可以通过以下方式去判断当前设备是否支持 Core Haptics：
```swift
import CoreHaptics
···
// 硬件是否支持 Core Haptics
let supportsHaptics: Bool = CHHapticEngine.capabilitiesForHardware().supportsHaptics
```

2. **注意引擎的启动耗时**



CHHapticEngine启动需要时间，作者用 iPhone xs 真机测试，首次启动耗时大概在 100 ms 级别。
​

当触觉反馈伴随着声音、动画时，这种延迟很容易被用户感知到，所以为了用户体验，我们应该在真正使用触觉反馈前就启动引擎。
​

在上面的示例代码中，我们在 viewDidLoad 方法里提前启动了引擎，就是为了在点击按钮时能够尽可能快的提供触觉反馈。
​


3. **引擎可能随时会停止**



有很多因素会导致引擎停止，比如来电、退到后台等。
​

当引擎被停止时，我们必须重新启动引擎才可以播放触觉反馈。
​

在上面的示例代码中，在按钮的点击事件中，我们都会尝试调用 start(completionHandler:) ，就是为了处理引擎被停止的情况。当引擎已经在启动状态，再次启动不会有什么副作用，所以推荐在播放触觉反馈前直接尝试进行启动。
​

你也可以设置 stoppedHandler 来感知到引擎停止事件：
```swift
engine?.stoppedHandler = { reason in
      print("Stop Handler: The engine stopped for reason: \(reason.rawValue)")
      switch reason {
        case .audioSessionInterrupt: print("Audio session interrupt")
        case .applicationSuspended: print("Application suspended")
        case .idleTimeout: print("Idle timeout")
        case .systemError: print("System error")
        default: print("Other error")
      }
}
```
有时引擎会发生错误，当其从错误中恢复时会调用 CHHapticEngine 的 resetHandler，苹果推荐我们在这里将引擎进行重启：
```swift
engine.resetHandler = {
	// 重启引擎
    try? self.engine.start()
}
```


### 2.3 动态改变触觉参数
在某些场景下，我们可能需要对事件的属性进调整。对此 CHHapticPatternPlayer 提供了sendParameters(parameters:, atTime:）方法：
```swift
let dynamicParameter = CHHapticDynamicParameter(parameterID: .hapticIntensityControl, value: sender.value, relativeTime: 0.5)
try? player.sendParameters([dynamicParameter], atTime: CHHapticTimeImmediate)
```

### 2.4 更高级的player


CHHapticPatternPlayer 是一个协议，可以通过 CHHapticEngine 的 makePlayer(with:) 获得遵循该协议的播放器对象， 其有 start、stop、cancel 等方法，我们可以用其来控制触觉反馈播放过程。
​

除此之外，Core Haptics 里还有一个功能更加丰富的 CHHapticAdvancedPatternPlayer 协议，其除了继承了 CHHapticPatternPlayer 的所有方法外，还新增了 pause、resume、seek 等控制方法，还可以设置播放速度 （playbackRate），是否循环播放 （loopEnabled）等属性。
​

可以通过 CHHapticEngine 的 makeAdvancedPlayer(with:) 获得遵循 CHHapticAdvancedPatternPlayer 协议的播放器对象：
```swift
// 获取 CHHapticAdvancedPatternPlayer 播放器对象
let player = try engine.makeAdvancedPlayer(with: pattern)
// 设置相关属性，此处设置了循环播放、播放速度为 0.5
player.loopEnabled = true
player.playbackRate = 0.5
```
​

### 2.5 通过 AHAP 文件来启动播放
对于简单的振动，我们可以直接通过代码创建 event 和 pattern， 但是对于比较复杂的，有很多阶段的振动，这种方式未免太过繁琐。


对此苹果给出的解决方案是使用 AHAP ( Apple Haptic and Audio Pattern )文件， 这是类似于 JSON 的文件格式，用于定义触觉模式， 其结构如下图所示：
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1626855698843-2a44b0b5-e54c-4345-bf15-1092752efaf7.png#clientId=u1237c635-12d3-4&from=paste&height=448&id=ubc5e4658&margin=%5Bobject%20Object%5D&name=image.png&originHeight=895&originWidth=1496&originalType=binary&ratio=1&size=133967&status=done&style=none&taskId=u47baeb98-58ed-4d96-8fa6-a2f8d9a55e6&width=748)
​

通过 AHAP 文件启动播放有两种方式：
​


1. engine直接播放

这种方式非常简单，但是缺点是启动播放后无法进行暂停等播放过程控制。
```swift
// AHAP 文件路径
let path = Bundle.main.path(forResource： filename， ofType： "ahap")
// 启动播放
try engine.playPattern(from： URL(fileURLWithPath： path))
```

2. 通过 AHAP 文件创建 pattern & player
```swift
// 通过 AHAP 文件创建 pattern
let patternJSONData = try Data(contentsOf: URL(fileURLWithPath: ahapPath), options: [])
let patternDict = try JSONSerialization.jsonObject(with: patternJSONData, options: []) as? [CHHapticPattern.Key: Any]
let pattern = try CHHapticPattern(dictionary: patternDict!)

// 通过 pattern 创建player
let player = try engine?.makeAdvancedPlayer(with: pattern)
// 启动播放
try player?.start(atTime: CHHapticTimeImmediate)
```
可以看到这种方式先使用 AHAP 文件创建了 pattern， 然后通过 pattern 创建了 player。
既利用了 AHAP 文件避免了硬编码 event，同时兼有灵活控制能力，所以更加推荐使用这种方式。

## 3. 声音触觉设计原则


现在你应该已经初步掌握了 Core Haptics 的使用了，在将这个技术应用到项目中之前，你需要了解三个基本设计原则：


### 3.1 因果性( Causality )


理想情况下，用户应该知道为什么应用会播放触觉模式。如果触觉不能加强因果关系，它可能会令人困惑并且看起来毫无意义。
比如在我们的认知里，踢球的时候，“砰”的一声是正常的，是脚碰球这一动作的效果。如果我们给配一个玻璃破碎的音效，则无法将两者联系起来，这种体验将会非常怪异。
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1626870523645-3c572788-7256-4c42-aad9-29f35ba85b5e.png#clientId=u1237c635-12d3-4&from=paste&height=318&id=ue6c2c6fd&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1464&originWidth=2596&originalType=binary&ratio=1&size=1630200&status=done&style=none&taskId=uf21ebb29-0f97-4484-af63-0e495e16ca1&width=563)
​

### 3.2 协调性( Harmony )


协调性是指触觉给人的感受，应当与其外观和声音保持一致。
在现实生活中，音效、触觉和视觉能很自然地保持协调，因为三者间有很自然的联系。然而在数字世界，我们就需要手动创造了。
当应用程序的视觉、听觉和触觉反馈协调一致时——就像它们在物理世界中一样——用户体验会更加连贯且看起来更自然。
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1626871173052-ae5e1ef3-0e36-4377-b27f-de2e3da5bbc7.png#clientId=u1237c635-12d3-4&from=paste&height=597&id=u2036ac0d&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1194&originWidth=2118&originalType=binary&ratio=1&size=180777&status=done&style=none&taskId=ua910f68b-3850-4f1a-8196-c3b141709b7&width=1059)


### 3.3 实用性( Utility ) 
实用性是指仅在足以给你的 App 的用户体验带来明确价值和成效的时候，加入音效和触感体验。
不要仅仅因为你会使用而添加声音触感，这将会使用户觉得突兀和不悦。

## 总结


整体来说，Core Haptics 的关键概念并不复杂，实现步骤也比较清晰易懂。如果你的 APP 里有合适的场景，不妨现在就可以将 Core Haptics 应用进去提升用户体验。但是一定要记住在实践触觉反馈设计时的三个原则：因果性、协调性和实用性。苹果提供的三个示例程序基本包含了所有使用方式，你可以结合自身需求参考官方示例的实现，相信一定会有所收获。
