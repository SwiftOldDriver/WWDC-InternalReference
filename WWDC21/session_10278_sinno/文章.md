## 前言
2016 年，苹果发布了 iPhone 6s / 6s Plus，其首次搭载了线性震动马达---Tapic Engine，之后的几年里，这颗体积不小的硬件一直在新 iPhone 寸土寸金的主板上占据一席之地。

​

基于 Tapic Engine， iOS 系统实现了丰富的触觉体验：比如在 iPhone 7 上仿佛可以按压下去的虚拟Home 键、3D Touch / Haptic Touch的真实触觉反馈、以及系统闹钟拨动时间滚轮那短促有力的咔哒声等...
​

事实上，除了 iOS 自身的应用外，苹果还为我们开发者提供了多种方式来丰富 App 内的触觉体验

1. 使用自带触觉反馈的 UIKit 组件：比如 UISwitch、UISlider、UIPickerView
1. iOS 10 开放了 UIFeedbackGenerator 系列类，可以非常方便的实现简单的振动反馈效果
1. iOS 13 开放了 Core Haptics， 可以实现自定义的触感振动。

## 1. Core Haptics 关键概念
### 1.1  持续触觉和瞬时触觉事件
Core Haptics 里的触觉事件可以分为两种：

1. 持续( Continuous )事件：持续的振动，可以设置持续时间
1. 瞬时( Transient )事件：类似轻敲或者撞击的短暂事件

![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624710194276-5fa6cb6e-963d-478d-94b2-9de76a0f74ba.png#clientId=u7b7561dd-8eed-4&from=paste&height=192&id=u49138ff7&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1072&originWidth=1852&originalType=binary&ratio=2&size=220515&status=done&style=none&taskId=u1d2557aa-f039-4779-9f1e-4323a980c0d&width=331)![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624711073150-c94c9181-1090-4ccb-b445-fc6410b2a237.png#clientId=u6b4cc879-5e39-4&from=paste&height=191&id=u28cf0768&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1072&originWidth=1872&originalType=binary&ratio=2&size=78287&status=done&style=none&taskId=u7d6ad05c-4256-42b6-bca0-4c1922756f9&width=333)
### 1.2  振动的锐度和强度
无论是持续事件还是瞬时事件，我们都可以控制事件的锐度( sharpness )和强度( intensity )。
​

锐度：可以用来传达一种柔软、圆滑、有机的体验，或者是一种松脆、清晰、机器驱动的体验
强度：触觉的强度，值越大，感受到的振动越强
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624982288815-d2ffcacb-0ee4-47a6-bc40-41f23dc93ec9.png#clientId=u9c94e9be-391a-4&from=paste&height=236&id=u808364ca&margin=%5Bobject%20Object%5D&name=image.png&originHeight=472&originWidth=1456&originalType=binary&ratio=2&size=150789&status=done&style=none&taskId=u1af76067-9e17-4ddb-ad40-1bab2faa6f1&width=728)


### 1.3 Core Haptics 基本元素：触觉引擎、播放器、模式、事件
在 Core Haptics 里有4个最基本的元素，了解了这些元素对掌握触觉反馈很有帮助：
​

**CHHapticEngine**：触觉引擎， 是应用程序与 Tapic Engine 硬件交互的接口
**CHHapticPlayer**： 播放器，用于播放控制，比如开始、停止、暂停等
**CHHapticPattern**： 播放模式，表示触觉波形，是一组以时间维度排列的事件的组合
**CHHapticEvent**： 单个触觉或声音事件的描述，其中触觉事件就如 1.1 里所述包含持续事件和瞬时事件
​

## 2.使用 Core Haptics 构建自定义的触觉反馈


### 2.1. 检查设备兼容性
不是所有的设备都支持 Core Haptics， 比如 iPad、iPod Touch 就不支持。
我们可以通过以下代码来检测当前设备的支持情况：
```swift
var supportsHaptics： Bool = false
...
// 检测设备是否支持触觉反馈
let hapticCapability = CHHapticEngine.capabilitiesForHardware()
supportsHaptics = hapticCapability.supportsHaptics
```
如果用户的设备不支持触觉反馈，我们可以选择使用具有更强的声音或其他多媒体替代触觉反馈。
​

### 2.2. 创建并设置触觉引擎( CHHapticEngine )
#### 2.2.1 创建一个触觉引擎
触觉引擎( CHHapticEngine )是应用程序与触觉硬件的交互接口，我们需要在播放触觉反馈前创建好一个触觉引擎，并且需要管理它的生命周期，以保证其不会在播放触觉反馈过程中被释放。
可以通过如下代码创建一个触觉引擎：
```swift
// ViewController.swift
var engine： CHHapticEngine!
...
override func viewDidLoad() {
    ...
    // Create and configure a haptic engine.
    do {
        engine = try CHHapticEngine()
    } catch let error {
        fatalError("Engine Creation Error： \(error)")
    }
    ...
}
```
以上代码在 ViewController 类里声明了一个 engine 属性以保存 CHHapticEngine 实例，并且在 viewDidLoad 方法里创建了 CHHapticEngine 实例并赋值给 engine。
​

**注意：**
CHHapticEngine 不是一个单例，你可以应用内创建多个实例，这些实例都是独立运行的，彼此互不影响；
Core Haptics 是线程安全的，这意味着您可以在单独的线程上执行操作。
​

#### 2.2.2 设置 Reset Handler
当 Core Haptics 从错误中恢复时会调用 CHHapticEngine 的 resetHandler，在这里我们应该做以下事情：

1. 重启引擎
1. 重新注册之前注册过的自定义音频资源
1. 重新创建之前创建过的 CHHapticPatternPlayer
```swift
// The reset handler provides an opportunity to restart the engine.
engine.resetHandler = {
    print("Reset Handler： Restarting the engine.")
    do {
        // Try restarting the engine.
        try self.engine.start()
        // Register any custom resources you had registered， using registerAudioResource.
        // Recreate all haptic pattern players you had created， using createPlayer.
    } catch {
        fatalError("Failed to restart the engine： \(error)")
    }
}
```
#### 2.2.3 设置 stoppedHandler
当外部因素导致触觉引擎停止时，例如来电导致的中断或者应用程序退到后台，Core Haptics 会调用 stoppedHandler。
```swift
// The stopped handler alerts engine stoppage.
engine.stoppedHandler = { reason in
    print("Stop Handler： The engine stopped for reason： \(reason.rawValue)")
    switch reason {
    case .audioSessionInterrupt： print("Audio session interrupt")
    case .applicationSuspended： print("Application suspended")
    case .idleTimeout： print("Idle timeout")
    case .systemError： print("System error")
    @unknown default：
        print("Unknown error")
    }
}
```
注意：
stoppedHandler 仅仅当外部原因导致引擎停止才会调用，通过 stop(completionHandler：) 方法手动停止引擎是不会调用的。
​

### 2.3 创建 CHHapticPattern
CHHapticPattern 可以理解为触觉播放的波形，其由多个 CHHapticEvent 以时间维度排列组成，各个 CHHapticEvent 可以在时间上可以重叠，也可以有一段时间完全没有事件，如下图所示：
当 pattern 被播放时，会按时间顺序触发各个事件。
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624815859730-b15bbc17-8cc9-40f6-863e-8b4c8f4eadc6.png#clientId=u6b4cc879-5e39-4&from=paste&height=469&id=uae5e5de8&margin=%5Bobject%20Object%5D&name=image.png&originHeight=938&originWidth=2282&originalType=binary&ratio=2&size=120311&status=done&style=none&taskId=u19716994-9fdc-46a5-bcd8-06b38cbf335&width=1141)
​

如下代码初始化了一个持续振动事件 event1，并且指定其 intensity 为0.5，相对开始时间为 0 (即立即开始)，时长为 1 秒；
初始化了一个瞬时振动事件 event2， 并设置其 intensity 和 sharpness 都为 1，相对开始时间为 1 (即在最末尾开始)
最后根据 event1 和 event2 创建了 pattern。
```swift
let event1 = CHHapticEvent(
            eventType： .hapticContinuous，
            parameters： [CHHapticEventParameter(parameterID： .hapticIntensity， value： 0.5)]，
            relativeTime： 0，
            duration： 1)
        
let event2 = CHHapticEvent(
            eventType： .hapticTransient，
            parameters： [CHHapticEventParameter(parameterID： .hapticIntensity， value： 1)，
                         CHHapticEventParameter(parameterID： .hapticSharpness， value： 1)]，
            relativeTime： 1)

let pattern = try CHHapticPattern(events： [event1， event2]， parameters： [])
```


### 2.4 创建 CHHapticPatternPlayer 并启动播放
通过如下代码，我们通过引擎传入了 pattern 创建了一个 CHHapticPatternPlayer 实例， 然后调用该实例的 start(atTime:) 方法立刻启动了播放。
```swift
try self.engine.start()
let player = try engine.makePlayer(with： pattern)
try player.start(atTime： CHHapticTimeImmediate)
```
### 2.5 通过 AHAP 文件来启动播放
对于简单的振动，我们可以通过如2.3中的代码所示创建 pattern， 但是对于比较复杂的，有很多阶段的振动，这种方式未免太过繁琐。
对此苹果给出的解决方案是使用 AHAP ( Apple Haptic and Audio Pattern )文件， 是类似于 JSON 的文件格式，用于定义触觉模式， 其结构如下图所示：

在 Mac OS 12 上可以图形化预览 AHAP 文件：
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624902150050-133fe102-c0b4-4fa0-9a12-652688483fcd.png#clientId=uc9f5664e-89ed-4&from=paste&height=621&id=u8e266224&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1242&originWidth=2234&originalType=binary&ratio=2&size=580294&status=done&style=none&taskId=u9a0dd8fa-ff72-4534-8641-8a4bd3a6731&width=1117)

## 3. 声音触觉设计原则
现在你应该已经初步掌握了 Core Haptics 的使用了，在将这个技术应用到项目中之前，你需要先了解三个基本设计原则：

1. 因果性( Causality )： 理想情况下，用户应该知道为什么应用会播放触觉模式。如果触觉不能加强因果关系，它可能会令人困惑并且看起来毫无意义。
1. 和谐( Harmony )： 当应用程序的视觉、听觉和触觉反馈协调一致时——就像它们在物理世界中一样——用户体验会更加连贯且看起来更自然。
1. 实用性( Utility )： 只有在可以提供清晰的价值和好处时才添加声音触感反馈。不要仅仅因为你会使用而添加声音触感，这将会使用户觉得突兀和不悦。

## 4.一个遵循设计原则的实例
在 WWDC2021 Practice audio haptic design 上，苹果工程师给出了一个示例程序：[HapticRicochet](https://developer.apple.com/documentation/corehaptics/delivering_rich_app_experiences_with_haptics)
这是一个模拟物理小球滚动的程序，小球撞击边缘、小球在纹理背景上滚动、小球加上护盾时会有声音触觉反馈，效果非常棒，大家可以下载源码编译体验一下。
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624988156854-11b70a95-0c3c-40eb-9080-03508c048a5e.png#clientId=u9c94e9be-391a-4&from=paste&height=437&id=u2d13fe0b&margin=%5Bobject%20Object%5D&name=image.png&originHeight=874&originWidth=1560&originalType=binary&ratio=2&size=1758105&status=done&style=none&taskId=u3db6ca84-86c9-4d74-a0ff-a8dbda0164d&width=780)
在视频中，苹果工程师介绍了 HapticRicochet-Start 工程中需要优化的两个场景：
第一个是小球加护盾这一场景，介绍动画、触觉、声音三个维度事件的处理，并且根据和谐的原则选取了合适的触觉模式和声音资源。
![image.png](https://cdn.nlark.com/yuque/0/2021/png/417753/1624987587535-f2883670-ea55-4e4a-9730-516baf543fc0.png#clientId=u9c94e9be-391a-4&from=paste&height=455&id=u20ab7db4&margin=%5Bobject%20Object%5D&name=image.png&originHeight=910&originWidth=1624&originalType=binary&ratio=2&size=237202&status=done&style=none&taskId=uecacdc19-36ba-482f-8755-15ed5dc3dc0&width=812)
第二个场景是小球在背景纹理上滚动的场景，存在两个问题：

1. 在滚动过程中，触觉反馈只播放了两秒就结束了
1. 滚动过程的触觉反馈和背景的纹理不符合(背景纹理比较粗粒度)

## 总结
整体来说，Core Haptics 的关键概念并不复杂，实现步骤也比较清晰易懂。如果你的 APP 里有合适的场景，不妨现在就可以将 Core Haptics 应用进去提升用户体验。但是一定要记住在实践触觉反馈设计时的三个原则：因果性、和谐、实用性，你可以结合 WWDC2021 Practice audio haptic design 和示例代码了解苹果工程师是如何一步一步优化触觉反馈体验的。
