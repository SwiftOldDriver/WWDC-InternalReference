## HealthKit，ResearchKit 和 CareKit 

在 WWDC 2016 上，Apple 发布了一款名为 HealthKit 的移动应用平台，可以用来收集和分析用户的健康数据。不过，HealthKit 更多面向的是普通用户，收集相对单一的用户数据，医疗价值有限。致力于更高的医疗价值，赋予医学研究者和医生的强大力量，为人们带来医疗领域的洞察和发现。Apple 在 15 年推出了 ResearchKit 让医学研究人员能够了解疾病，随着 ResearchKit 的成功，Apple 将目光从医学研究领域拓展至个人护理领域，进而在 16 年推出了 CareKit ，至此你将不再仅仅依靠医生的诊治，而是可以定期跟踪记录自己的症状和用药情况，还能与护理团队共享这些信息，让他们更全面、更深入地了解你的健康状况。

###  ResearchKit 和 CareKit 

Apple 已将 ResearchKit 和 CareKit 打造成开源架构，他们希望每个人都能为医学研究贡献一己之力，并能从更周到的个人护理中获益良多。开源架构是一个非常好的途径，能够鼓励医学界与开发者进行协作并分享各自的 app 和方法。

为方便大家理解，我先对 ResearchKit 和 CareKit 做下简单的概述，并大致介绍它们的功能和使用场景。

<img src="https://raw.githubusercontent.com/LoneyIsError/blog_images/main/researchKit.png" style="zoom: 50%;" />

ResearchKit 是 Apple 与 GitHub 社区合作维护的开源 iOS 框架。它提供了 UI 组件，用于在参与者执行活动（例如身体或认知练习）时收集同意、管理调查和记录传感器数据。

<img src="https://raw.githubusercontent.com/LoneyIsError/blog_images/main/careKit.png" style="zoom:50%;" />

同样，CareKit 也是一个完全开源的框架，旨在帮助构建面向患者的护理应用程序。你可以安排需要患者执行的任务，比如服药或者打电话给他们的医生。它还存储那些健康相关的数据，你可以基于这些数据绘制图表。

在过往的 WWDC Session 中，对 ResearchKit 和 CareKit 都是单独的介绍，但是，它们可以相辅相成。

你可以通过访问 [ResearchKit 和 CareKit 官网](https://www.researchandcare.org/)来获取更多的信息。

## 无缝衔接的 ResearchKit 和 CareKit 应用程序

我们所创建的是一个用于帮助膝关节手术者的术后物理治疗的应用程序 —— RecoverApp，通过完成研究者制定的动作、问卷调查和患者的任务清单，来实现医疗研究和术后护理，那么，我们开始吧！

对我们创建的 Recover 应用程序来说，有两个 `tab` ，Care Feed（日常护理）和 Insights（分析）。

初始设置阶段我们需要基于 CareKit 的应用程序并完成上述告知和授权操作，我们可以从 [WWDC21-RecoverApp](https://github.com/carekit-apple/WWDC21-RecoverApp) 获取项目的初始代码。

#### 建立标准的 CareKit 应用程序

首先我们需要在 `AppDelegate.swift` 文件中创建一个持久化任务 —— 注册参与者。

每当我们创建任务时，我们都需要定义一个计划，指定任务出现的频率。

```swift
private func seedTasks() {
    let onboardSchedule = OCKSchedule.dailyAtTime(
        hour: 0, minutes: 0,
        start: Date(), end: nil,
        text: "Task Due!",
        duration: .allDay
    )

    var onboardTask = OCKTask(
        id: TaskIDs.onboarding,
        title: "Onboard",
        carePlanUUID: nil,
        schedule: onboardSchedule
    )
    onboardTask.instructions = "You'll need to agree to some terms and conditions before we get started!"
    onboardTask.impactsAdherence = false //该任务并不会影响任务的完成度，就像是一个环而已

    storeManager.store.addAnyTasks(
        [onboardTask],
        callbackQueue: .main) { result in

        switch result {

        case let .success(tasks):
            Logger.store.info("Seeded \(tasks.count) tasks")

        case let .failure(error):
            Logger.store.warning("Failed to seed tasks: \(error as NSError)")
        }
    }
}
```

当我们键入上述代码时，在我们的每日清单中都会出现这个任务，直至该任务被完成。

我们创建一个方法 `checkIfOnboardingIsComplete(...)` 用来检测注册参与者任务是否完成

```swift
private func checkIfOnboardingIsComplete(_ completion: @escaping (Bool) -> Void) {
   // 根据任务 ID 获取 ID 为 onboarding 的任务
    var query = OCKOutcomeQuery()
    query.taskIDs = [TaskIDs.onboarding]
    // 查询对应任务的结果
    storeManager.store.fetchAnyOutcomes(
        query: query,
        callbackQueue: .main) { result in

        switch result {

        case .failure:
            Logger.feed.error("Failed to fetch onboarding outcomes!")
            completion(false)

        case let .success(outcomes):
            completion(!outcomes.isEmpty)
        }
    }
}
```

当我们的查询返回时，我们将检查是否找到任何结果。 如果没有，则意味着尚未完成。 如果有，则意味着注册任务已经完成。

对用户来说，只有注册成为参与者参与研究，否则无法使用 APP 的功能。


#### 获取授权

除了在 WWDC 2020 上的引入的 `HealthKitPermissionType` 权限以外，今年又引入了两个新的收集权限的类型 —— `ORKNotificationPermissionType` 和 `ORKMotionActivityPermissionType`。

我们希望获得  HealthKit 读和写入两种权限，所以我们这么操作：

```swift

    HKObjectType.quantityType(forIdentifier: .activeEnergyBurned)!,
    HKObjectType.workoutType()
]

let healthKitTypesToRead: Set<HKObjectType> = [
    HKObjectType.characteristicType(forIdentifier: .dateOfBirth)!,

    HKObjectType.quantityType(forIdentifier: .appleStandTime)!,
    HKObjectType.quantityType(forIdentifier: .appleExerciseTime)!

    sampleTypesToWrite: healthKitTypesToWrite,
```

然后我们切换到 `CareFeedViewController.swift` 文件中，我们要告诉 `CareKit` 如何展示这个新任务。为了使解决方案更通用一些，我们将获取当前日期的所有任务，然后对于每个任务，我们将创建对应的视图控制器，并将该视图控制器添加到我们的列表中。这样，如果我们需要添加更多任务，代码会更容易拓展。

我们创建一个 `fetchTasks` 方法，其参数为 `date`，我们在方法中根据传入的 `date` 进行任务查询并返回符合条件的任务。

```swift
private func fetchTasks(
    on date: Date,
    completion: @escaping([OCKAnyTask]) -> Void) {
    //根据日期查询任务
    var query = OCKTaskQuery(for: date)
    //如果任务只是空任务，那么必要查询出来
    query.excludesTasksWithNoEvents = true

    storeManager.store.fetchAnyTasks(
        query: query,
        callbackQueue: .main) { result in

        switch result {

        case .failure:
            Logger.feed.error("Failed to fetch tasks for date \(date)")
            completion([])

        case let .success(tasks):
            completion(tasks)
        }
    }
}
```

然后如同添加注册任务一样，我们需要根据处理获取的任务，并且返回一个视图控制器加到 `listViewController` 中，这里我们创建一个新的 `OCKSurveyTaskViewController` 来处理我们的每日任务，当然在此之前，我们先定义每日任务需要完成的问题项， 在 `Surveys.swift` 中定义每日任务的两个问题——疼痛等级和睡眠时间。我们使用 CareKit 所提供的新的答案格式 `scale` ，就像 `slider` 一样通过滑动选择答案，这样能提供更好的用户体验。

你可以参照 [Getting Started with Apple’s ResearchKit](https://www.appcoda.com/researchkit/) 来学习如何创建其他问题的答案格式。

我们创建问题，最终生成一个步骤顺序一致的任务 —— `ORKOrderedTask`，并将结果返回。

```swift
static let checkInIdentifier = "checkin"
static let checkInFormIdentifier = "checkin.form"
static let checkInPainItemIdentifier = "checkin.form.pain"
static let checkInSleepItemIdentifier = "checkin.form.sleep"

static func checkInSurvey() -> ORKTask {

    let painAnswerFormat = ORKAnswerFormat.scale(
        withMaximumValue: 10,
        minimumValue: 1,
        defaultValue: 0,
        step: 1,
        vertical: false,
        maximumValueDescription: "Very painful",
        minimumValueDescription: "No pain"
    )

    let painItem = ORKFormItem(
        identifier: checkInPainItemIdentifier,
        text: "How would you rate your pain?",
        answerFormat: painAnswerFormat
    )
    painItem.isOptional = false

    let sleepAnswerFormat = ORKAnswerFormat.scale(
        withMaximumValue: 12,
        minimumValue: 0,
        defaultValue: 0,
        step: 1,
        vertical: false,
        maximumValueDescription: nil,
        minimumValueDescription: nil
    )

    let sleepItem = ORKFormItem(
        identifier: checkInSleepItemIdentifier,
        text: "How many hours of sleep did you get last night?",
        answerFormat: sleepAnswerFormat
    )
    sleepItem.isOptional = false

    let formStep = ORKFormStep(
        identifier: checkInFormIdentifier,
        title: "Check In",
        text: "Please answer the following questions."
    )
    formStep.formItems = [painItem, sleepItem]
    formStep.isOptional = false

    let surveyTask = ORKOrderedTask(
        identifier: checkInIdentifier,
        steps: [formStep]
    )

    return surveyTask
}
```

我们还需要一个将 ResearchKit 的问卷结果转换为 CareKit 结果值数组的函数。这样当我们完成我们的每日任务时，日期的圆环将被填充。一个很重要的点是 `ORKTaskResults` 是一个嵌套类型，其结果由两个问卷问题——疼痛和睡眠，我们需要根据任务 `identifier`  获取对应任务的响应，然后获取问卷结果，然后根据问题的 `identifier` 获取对应问题的解答，并且将解答填充到 CareKit 结果值数组。这个例子中，问题的答案格式是  `scaleAnswer`。

这里的 `kind` 属性是可选的，可以不设置它，但如果稍后想查找某个值，它会很有帮助。

```swift
static func extractAnswersFromCheckInSurvey(
    _ result: ORKTaskResult) -> [OCKOutcomeValue]? {

    guard
        let response = result.results?
            .compactMap({ $0 as? ORKStepResult })
            .first(where: { $0.identifier == checkInFormIdentifier }),

        let scaleResults = response
            .results?.compactMap({ $0 as? ORKScaleQuestionResult }),

        let painAnswer = scaleResults
            .first(where: { $0.identifier == checkInPainItemIdentifier })?
            .scaleAnswer,

        let sleepAnswer = scaleResults
            .first(where: { $0.identifier == checkInSleepItemIdentifier })?
            .scaleAnswer
    else {
        assertionFailure("Failed to extract answers from check in survey!")
        return nil
    }

    var painValue = OCKOutcomeValue(Double(truncating: painAnswer))
    painValue.kind = checkInPainItemIdentifier

    var sleepValue = OCKOutcomeValue(Double(truncating: sleepAnswer))
    sleepValue.kind = checkInSleepItemIdentifier

    return [painValue, sleepValue]
}
```

```swift
            extractOutcome: Surveys.extractAnswersFromCheckInSurvey
        )

        return survey
    default:
        return nil
    }
}
```

## 制定动态任务

具体来说，我们想设置一个时间表，让参与者在第一周每天测量他们的膝关节运动范围，但之后每周只测量一次，直到月底，之后再也不测量了。我们需要利用 Carekit 的 `OCKScheduleElement` 来帮助我们完成这种细微的计划。

首先我们定义几个关键日期：`thisMorning`、`nextWeek` 和 `nextMonth`。

```swift
let thisMorning = Calendar.current.startOfDay(for: Date())

let nextWeek = Calendar.current.date(
    byAdding: .weekOfYear,
    value: 1,
    to: Date()
)!

let nextMonth = Calendar.current.date(
    byAdding: .month,
    value: 1,
    to: thisMorning
)
```

`OCKScheduleElement` 有开始日期、结束日期，并在此期间以某个时间间隔重复。我们创建两个新的计划元素 —— `dailyElement` 和 `weeklyElement`。

`dailyElement` 是今天上午开始到下周结束，期间每天重复，而 `weeklyElement` 是下周开始，下个月结束，期间每周重复。

我们可以将它们组合在一起以创建一个复合时间表，然后我们可以使用该时间表创建一个新的测量膝关节运动范围任务。

```swift
let dailyElement = OCKScheduleElement(
    start: thisMorning,
    end: nextWeek,
    interval: DateComponents(day: 1),
    text: nil,
    targetValues: [],
    duration: .allDay
)

let weeklyElement = OCKScheduleElement(
    start: nextWeek,
    end: nextMonth,
    interval: DateComponents(weekOfYear: 1),
    text: nil,
    targetValues: [],
    duration: .allDay
)

let rangeOfMotionCheckSchedule = OCKSchedule(
    composing: [dailyElement, weeklyElement]
)

let rangeOfMotionCheckTask = OCKTask(
    id: TaskIDs.rangeOfMotionCheck,
    title: "Range Of Motion",
    carePlanUUID: nil,
    schedule: rangeOfMotionCheckSchedule
)
```

当然也不要忘记将其加入到我们 `storeManager` 的任务清单中

当然，我们还需要一个将 ResearchKit 结果转换为 CareKit 结果值的方法，我们有很多方法可以实现这个功能，但今天我们将进行递归查询，直到获得第一个 `ORKRangeOfMotionResult` 的结果，因为我们知道只会有一个。

`ORKRangeOfMotionResult` 有很多有用的属性，但是这里我们关注的是范围，它是测量参与者能够弯曲膝盖的程度的结果值，也就是角度。

我们将 `kind` 字段与 `keyPath` 一起使用，以便我们能够很快的访问到它。

```swift
static func extractRangeOfMotionOutcome(
    _ result: ORKTaskResult) -> [OCKOutcomeValue]? {

    guard let motionResult = result.results?
        .compactMap({ $0 as? ORKStepResult })
        .compactMap({ $0.results })
        .flatMap({ $0 })
        .compactMap({ $0 as? ORKRangeOfMotionResult })
        .first else {

        assertionFailure("Failed to parse range of motion result")
        return nil
    }

    var range = OCKOutcomeValue(motionResult.range)
    range.kind = #keyPath(ORKRangeOfMotionResult.range)

    return [range]
}
```

#### 完善调查

##### 显示答案

让我们先处理参与者给出的答案，然后将给出的答案显示在卡片上。

现在，在 CareKit 中，调整任务卡的外观是通过将 `viewSynchronizer` 传递到其初始值设定项来完成的。

我们来看看如何进行自定义。我们将继承现有的 `OCKSurveyTaskViewSynchronizer`，在这个类中我们可以重写这两个方法：`makeView` 和 `updateView`。

每次 `store` 中的数据发生更改时都会调用 `updateView`，我们可以在此处进行自定义视图。

当我们重写父类的方法时，我们总是从调用父类的实现开始，并在此基础上添加我们的代码。我们通过之前设置的 `kind` 属性来帮助我们获取对应的答案，然后使用字符串将答案插入到任务卡的说明标签中。

```swift
final class SurveyViewSynchronizer: OCKSurveyTaskViewSynchronizer {

    override func updateView(
        _ view: OCKInstructionsTaskView,
        context: OCKSynchronizationContext<OCKTaskEvents>) {

        super.updateView(view, context: context)

        if let event = context.viewModel.first?.first, event.outcome != nil {
            view.instructionsLabel.isHidden = false
            
            let pain = event.answer(kind: Surveys.checkInPainItemIdentifier)
            let sleep = event.answer(kind: Surveys.checkInSleepItemIdentifier)

            view.instructionsLabel.text = """
                Pain: \(Int(pain))
                Sleep: \(Int(sleep)) hours
                """
        } else {
            view.instructionsLabel.isHidden = true
        }
    }
}
```

##### 编辑任务

对于我们今天的调查，哪怕参与者已经回答过，我们让参与者可以重新测量或删除这个调查，然后对于今天以前的任务我们只可以查看内容进制编辑而对于今天以后的调查任务，我们不能对其进行操作。

通过实现 `SurveyTaskViewControllerDelegate` 的 `shouldAllowDeletingOutcomeForEvent` 方法，通过返回一个布尔值，告诉 CareKit 是否应该允许删除特定事件的结果。

```swift
func surveyTask(
    viewController: OCKSurveyTaskViewController,
    shouldAllowDeletingOutcomeForEvent event: OCKAnyEvent) -> Bool {
  //事件发生的日期与今天进行比较
    event.scheduleEvent.start >= Calendar.current.startOfDay(for: Date())
}
```

这样，我们已经阻止删除过去的数据。我们还应该阻止参与者在未来的日期去完成调查。为此，我们重新处理 `dailyPageViewController(...)` 函数，当我们完成参与者注册之后，我们利用 `Calendar` 类来判断所填充的日期是否在将来。
如果是将来的日期，我们将禁用与任务卡的交互，并且我们用文字半透明以直观地指示它已被禁用。

```swift
let isFuture = Calendar.current.compare(
    date,
    to: Date(),
    toGranularity: .day) == .orderedDescending

self.fetchTasks(on: date) { tasks in
    tasks.compactMap {

        let card = self.taskViewController(for: $0, on: date)
        card?.view.isUserInteractionEnabled = !isFuture
        card?.view.alpha = isFuture ? 0.4 : 1.0

        return card

    }.forEach {
        listViewController.appendViewController($0, animated: false)
    }
}
```
