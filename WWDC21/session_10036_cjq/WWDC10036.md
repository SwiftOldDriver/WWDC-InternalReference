## 前言

2019年苹果推出[CreateML](https://developer.apple.com/documentation/createml)，支持在 Mac 上创建和训练自定的机器学习模型。

![](https://images.xiaozhuanlan.com/photo/2021/c00fe303ba7b8e88a6e502720c4d35fc.png)

同时还发布了 [SoundAnalysis](https://developer.apple.com/documentation/soundanalysis) 声音分析框架，通过分析音频文件或流对各种声音进行分类。可以很容易的将其部署到 Apple 设备中，当使用此框架时，所有计算都针对硬件加速进行了优化，并在本地设备上完成。这样音频永远不会发送到云端，进而有助于保护用户的隐私。还利用声音分析框架引入了称为声音识别的辅助功能，比如当在环境中听到某些声音（例如警报、宠物和其他家庭声音）时，此功能可以向用户提供通知。

![](https://images.xiaozhuanlan.com/photo/2021/cd42b23443f7bd49c8bfa1c6678038c2.png)

## 内置分类器

今年，苹果在声音分析框架 SoundAnalysis 中内置了一个声音分类器，只需几行代码即可启用声音分类，并且支持所有平台。

内置分类器旨在简化开发人员体验，它消除了收集和标记大量数据、专门的机器学习和音频专业知识以及大量计算能力以开发高精度模型的需要。这样开发人员就不用在怎么考虑这些细节，就可以有更多的时间来丰富您的应用程序的用户体验。

目前已有超过 300 个即用型类别提供使用，比如：

- 对家畜、家畜和野生动物的声音进行分类。
- 键盘乐器、打击乐器、弦乐器、管乐器等乐器识别。
- 集体活动、呼吸音、发声等人类声音检测。
- 车辆、警报、工具、液体等等之类的声音。

![](https://images.xiaozhuanlan.com/photo/2021/f6b2729f958597adeb0039bf67ef9792.png)

## 快速接入

文章接下来我们来快速实现一个声音分类功能，完整流程建议可以下载[例子工程](https://developer.apple.com/documentation/soundanalysis/classifying_live_audio_input_with_a_built-in_sound_classifier)跟着文章一起实现。

为了执行声音分类，需要准备三个对象：

首先，需要一个分类请求 `SNClassifySoundRequest`，可以用它来配置声音分类参数。
其次，需要一个分析器 `SNAudioAnalyzer`，它可以让我们针对特定数据进行分析分类。目前主要分为两类：

- `SNAudioStreamAnalyzer` 音频流分析器
- `SNAudioFileAnalyzer` 音频文件分析器

最后，还需要一个继承自 NSObject并符合 `SNResultsObserving` 协议裸观察者来获取检测结果。

```swift
// Create a Observer
let observer = ResultsObserving()

// Create a file analyzer
let fileAnalyzer = try SNAudioFileAnalyzer(url: URL(fileURLWithPath: "sound.caf"))

// Create the request with a known classifier.
let request = try SNClassifySoundRequest(classifierIdentifier: .version1)
request.windowDuration = CMTimeMakeWithSeconds(inferenceWindowSize, preferredTimescale: 1000)
request.overlapFactor = overlapFactor

// Add the request to the analyzer
try fileAnalyzer.add(request, withObserver: observer)

// Analyzer the file
fileAnalyzer.analyze()
```

上述中使用想要搜索的声音标签初始化实例，并且添加一个 CMTime 成员变量来存储我检测到声音的时间。然后只需要在观察者中实现 `didProduce result`方法。当声音分类产生结果时将调用此方法，并返回 [SNClassificationResult](SNClassificationResult) 的实例。

```swift
class ResultsObserving: NSObject, SNResultsObserving {
    func request(_ request: SNRequest, didProduce result: SNResult) {
        guard let result = result as? SNClassificationResult else { return }
        
        // Get the prediction that you pay attention to.
        guard let classification = result.classification(forIdentifier: "wind") else { return }

        // Convert the confidence to a percentage string.
        let percent = classification.confidence * 100.0
    }

    func request(_ request: SNRequest, didFailWithError error: Error) {
        // Handle error
    }

    func requestDidComplete(_ request: SNRequest) {
        // Handle successful end of analysis
    }
}
```

现在我们可以使用实例的 `classificationForIdentifier` 方法来提取有关我们需要的标签的信息对象 [SNClassification](https://developer.apple.com/documentation/soundanalysis/snclassification)。其中饱和了对于标签的关联的置信度分数，如果该分数超过某个阈值，就可认为检测到对于声音。

这个例子中涉及两个重要属性，`windowDuration` 检测时间和 `overlapFactor` 检测阈值。

### 检测时间

当对音频进行分类时，信号会被分解为重叠的窗口。 对于这些窗口中的每一个，都会得到一个结果，表示检测到了哪些声音以及有多大把握。其中还有一个时间范围，它表示音频的哪一部分被分类。当检测到声音时，将使用结果的时间范围来确定声音发生的时间。但是，当窗口的持续时间发生变化时，时间范围可能会受到影响。
我们可以根据具体用例自定义窗口持续时间以使其变大或变小。

![](https://images.xiaozhuanlan.com/photo/2021/1939a47992d044add5c4bebbe7823c85.png)

小窗口持续时间可以在很短的时间内捕捉到该声音的所有重要特征，且不会删减任何重要信息，支持更精确地确定声音发生的时刻。例如，敲击鼓声。
大窗口持续时间可以在单个窗口捕获所有声息信息，更利于处理较长的声音。一般来说，最好使用足够长的窗口持续时间来捕获您感兴趣的声音的所有重要部分。例如，警笛可能在较长时间内同时包含上升和下降音调。

> 但请注意，并非所有窗口持续时间都受支持。 不同的分类器可能支持不同的窗口持续时间。您可以通过阅读 `SNClassifySoundRequest` 的 `windowDurationConstraint`属性来检查支持哪些窗口持续时间。内置分类器支持 0.5 秒到 15 秒之间的窗口持续时间。在我们的应用程序中采用分类器时，一秒或更长时间的持续时间是一个很好的起点。

### 置信阈值

在我们的应用程序中，只要对该声音的置信度上升到固定阈值以上，就可以认为检测到该声音。

但在我们应用程序中选择阈值时需要考虑一些事项：

- 置信度独立，不应相互比较。分类器可以同时检测多个声音，发生这种情况时，可能会注意到有几个标签具有高置信度。 与使用 CreateML 训练的自定义模型不同，标签分数相加不等于1，所以我们需要为不同的声音选择不同的置信度阈值

![](https://images.xiaozhuanlan.com/photo/2021/84068f1f7e43b6a1491b29ea0ebfff1e.png)

- 阈值权衡。较高的置信度阈值会降低声音被错误检测到的可能性，但也会增加由于声音不够强而错过真实检测的可能性。当为应用程序选择一个阈值时，需要为用例找到一个能够实现这些因素的正确平衡的值。

![](https://images.xiaozhuanlan.com/photo/2021/4a3fe5d62c0013e15d80ff6e59244a1f.png)

- 关注预期的声音。在分类器可以识别的大量声音中，有几组声音可能很难单独使用音频来区分，即使对于人类也是如此。所以尽量只注意在应用程序将被使用时上下文中可能出现的声音。

![](https://images.xiaozhuanlan.com/photo/2021/98270eaffb741f89c26f721b73cf828a.png)

## 自定义模型

内置分类器是经过大量的数据训练的产物，包含了很多关于声音分类的知识。所有这些知识都可以用于使用 CreateML 训练自定义模型。

声音分类器可以分成两个不同的网络：特征提取器与分类器模型。

![](https://images.xiaozhuanlan.com/photo/2021/0d284af5cf4405b4ada377bab84c04bf.png)

### 特征提取器

特征提取器，有时被称为嵌入模型，是网络的骨干。它采用音频波形并将其转换为低维空间，训练有素的特征提取器会将声学相似的声音组织到空间中的附近位置。 例如，吉他的声音会聚集在一起，但远离鼓声和汽车的声音。

### 分类器模型

分类器模型获取特征提取器的输出并计算类概率，最终结果与之配对的特征提取器的好坏影响较大。

### 优化

目前苹果提供了内置分类器的特征提取器，它被称为 Audio Feature Print。

![](https://images.xiaozhuanlan.com/photo/2021/23e4edd555cba37afae2befade6b486c.png)

当在 CreateML 中训练自己的自定义模型时，自定义的模型将与 Audio Feature Print 配对。这样，自定义的模型就可以从内置分类器中包含的所有知识中受益。与上一代特征提取器相比，Audio Feature Print 有全面的改进。尽管这个网络更小更快，但它在比较的所有基准数据集上都实现了更高的准确性。与内置分类器一样，使用 Audio Feature Print 的模型支持灵活的窗口持续时间。可以选择较长的窗口持续时间来优化警笛之类的声音，或者选择较短的窗口持续时间来优化敲击声等声音。

当使用 CreateML 训练自定义模型时，音频特征打印是新的默认特征提取器。 窗口持续时间是训练时用于生成单个特征的音频长度。默认为三秒，但可以根据需要进行调整。目前 CreateML 支持选择 0.5 秒到 15 秒之间的窗口持续时间。

有关训练自定义模型的更详细示例，可以查看 2019 年关于“[Training Sound Classification Models in Create ML](https://developer.apple.com/videos/play/wwdc2019/425)”的课程。它还向您展示了如何使用声音分析框架来运行自定义模型。
