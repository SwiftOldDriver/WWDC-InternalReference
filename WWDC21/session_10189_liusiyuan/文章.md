# WWDC21 10189 - 使用Group Activity在Safari中共享媒体
[![RV23AU.jpg](https://z3.ax1x.com/2021/06/22/RV23AU.jpg)](https://imgtu.com/i/RV23AU)

GroupActivity提供了在FaceTime Call中共享媒体和播放状态的能力，这个能力不仅仅只适用于iOS上的App，本文将介绍如何使用GroupActivity在Safari中体验由iOS端发起的共享媒体。

## 一个简单的回顾
在[WWDC21 10225 - 使用Group Activity共享媒体]中我们详细地介绍了如何使用GroupActivity来让iOS上的App具有互相共享媒体的能力。首先需要定义的就是一个`GroupActivity`的结构体，其中包含了需要分享的活动内容。其中，metadata的fallbackURL包含了所分享资源的URL。同样，这个`GroupActivity`也可以被分享给Mac端的用户。
```swift
// A group activity to watch a movie together.
struct MovieWatchingActivity: GroupActivity {
    
    // The movie to watch.
    let movie: Movie
   
    // Metadata that the system displays to participants.
    var metadata: GroupActivityMetadata {
        var metadata = GroupActivityMetadata()
        metadata.type = .watchTogether
        metadata.fallbackURL = movie.url
        metadata.title = movie.title
        return metadata
    }
}
```

## 在Safari上观看共享视频
在Big Sur之后，苹果的不同设备之间的联系越来越紧密。GroupActivity可以使得用户在mac上观看其他用户从iOS App或者iPad App上共享的媒体，并且不需要安装对应的应用。还记得前面所定义的分享活动吗，Safari可以读取metadata内部的fallbackURL，并直接在浏览器中播放。需要注意的是，这个fallbackURL内部包含的不仅是资源地址，还需要包含这个资源的identifier。

Mac端播放视频的一个优势也在于，可以方便的使用导航栏的NowPlaying View进行播放体验的控制。同时在实现时需要额外考虑，将NowPlaying View的状态和媒体实际的状态进行同步。

支持环境需要MacOS Monterey 以及Safari 15。

### 共享媒体状态
在Safari上共享媒体的体验，和在iOS端上并不差别。无论在哪一平台的用户，都对媒体有着完全的控制权。也就是说，无论两个用户都在iOS设备，还是其中一个在Mac端，都可以控制视频的进度，进行播放、暂停、seek甚至是终止（包含对自己终止和对所有人终止）。

关于MacOS上共享媒体的API，需要注意以下几点：
- API目前还在实验阶段，并没有正式发布
- 目前在Mac上共享媒体的功能也仅仅能用在Safari浏览器上，不过苹果也希望未来可以让更多的浏览器获得这个能力
- 浏览器只能被动的接受分享的视频，想要发起视频必须通过iOS、iPadOS或者MacOS上的应用程序

## 总结
Neat！这篇session的主要内容就结束了。我们总结一下内容：
- SharePlay允许Mac用户在不下载程序的前提下，直接使用Safari参与共享媒体
- Safari是如何实现媒体共享、导航栏控制和共享状态同步这些功能的

这是一篇短小精悍的session，技术层面的介绍实际上比较简单。更多的是介绍了一个SharePlay的特性，即**允许Mac用户在不下载程序的前提下，直接使用浏览器参与共享媒体**，并且使用浏览器的用户对媒体也有着和发起侧同样的媒体的控制权利。因为目前浏览器侧的API还没有开放，这个功能只能在Safari中先尝尝鲜，开发者能做的还很少。不过这些跨平台特性所带来的体验往往也令人感到舒适，期待后面苹果还会有什么令人激动的操作。
