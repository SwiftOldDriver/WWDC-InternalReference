# WWDC21 10189 - 使用Group Activity在Safari中共享媒体
[![RV23AU.jpg](https://z3.ax1x.com/2021/06/22/RV23AU.jpg)](https://imgtu.com/i/RV23AU)

GroupActivity 提供了在 FaceTime Call 中共享媒体和播放状态的能力，这个能力不仅仅只适用于 iOS 上的 App ，本文将介绍如何使用 GroupActivity 在 Safari 中体验由 iOS 端发起的共享媒体。
在阅读本文之前，强烈建议先阅读[WWDC21 10225占位符/文章发出后补充链接]这篇文章。这篇 session 非常短小精悍，文中所提到的所有技术细节均在上文中做了详细介绍。本 session 更像是 session 10225 的一个 extension.

## 一个简单的回顾
在[WWDC21 10225 - 使用Group Activity共享媒体]中我们详细地介绍了如何使用 GroupActivity 来让 iOS 上的 App 具有互相共享媒体的能力。首先需要定义的就是一个 `GroupActivity` 的结构体，其中包含了需要分享的活动内容。其中，metadata 的 fallbackURL 包含了所分享资源的 URL. 同样，这个 `GroupActivity` 也可以被分享给 Mac 端的用户。
```swift
// A group activity to watch a movie together.
struct MovieWatchingActivity: GroupActivity {
    
    // The movie to watch.
    let movie: Movie
   
    // Metadata that the system displays to participants.
    var metadata: GroupActivityMetadata {
        var metadata = GroupActivityMetadata()
        metadata.type = .watchTogether
        metadata.fallbackURL = movie.url
        metadata.title = movie.title
        return metadata
    }
}
```

## 在Safari上观看共享视频
在 Big Sur 更新之后，苹果的不同设备之间的联系越来越紧密。GroupActivity 使得用户可以在 Mac 上观看其他用户从 iOS App 或者 iPad App 上共享的媒体，并且不需要安装对应的应用。还记得前面所定义的分享活动吗，Safari 可以读取 metadata 内部的 fallbackURL, 并直接在浏览器中播放。需要注意的是，这个 fallbackURL 内部包含的不仅是资源地址，还需要包含这个资源的 identifier。

在Mac端播放视频，用户也可以使用导航栏的 NowPlaying View 进行播放体验的控制。因此开发者需要额外考虑，在媒体播放时将导航栏中 NowPlaying View 的状态和媒体实际的状态进行同步。

> 在Safari上观看共享视频，需要环境 MacOS Monterey 以及 Safari 15。

### 共享媒体状态
在 Safari 上共享媒体的体验，和在 iOS 端上并无差别。无论在哪一平台的用户，都对媒体有着完全的控制权。也就是说，无论两个用户都在 iOS 设备，还是其中一个在 Mac 端，都可以控制视频的进度，进行播放、暂停、拖动进度甚至是终止（包含对自己终止和对所有人终止）。

关于 MacOS 上共享媒体的 API, 需要注意以下几点：
- API 目前还在实验阶段，并没有正式发布
- 目前在 Mac 上共享媒体的功能也仅仅能用在 Safari 浏览器上，不过苹果也希望未来可以让更多的浏览器获得这个能力
- 浏览器只能被动的接受分享的视频，想要发起视频必须通过 iOS、iPadOS 或者 MacOS 上的应用程序

## 总结
Neat！这篇 session 的主要内容就结束了。我们总结一下内容：
- SharePlay 允许 Mac 用户在不下载程序的前提下，直接使用 Safari 参与共享媒体
- Safari 是如何实现媒体共享、导航栏控制和共享状态同步这些功能的

这是一篇短小精悍的 session, 技术层面的介绍实际上比较简单。更多的是介绍了一个 SharePlay 的特性，即**允许Mac用户在不下载程序的前提下，直接使用浏览器参与共享媒体**，并且使用浏览器的用户对媒体也有着和发起侧同样的媒体的控制权利。因为目前浏览器侧的 API 还没有开放，这个功能只能在 Safari 中先尝尝鲜，开发者能做的还很少。不过这些跨平台特性所带来的体验往往也令人感到舒适，期待后面苹果还会有什么令人激动的操作。
