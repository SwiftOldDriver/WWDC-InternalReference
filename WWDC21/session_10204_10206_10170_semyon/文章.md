## 引言

iOS 应用程序签名打包上传商店一直比较复杂，对初学者来说不是很友好。今年苹果推出了 Cloud signing 功能，让签名更加简单快捷，无需理解各种证书、描述文件就可以上手。如果你是初学者，是不是很心动？此外 Mac 端的 App 公证流程也做了优化，让公证流程更加简单，这里会介绍新工具 notarytool 的使用方法。最后介绍即将推出的 Mac 版 TestFlight ，用法跟 iOS 和 tvOS 版本的类似，而且是可以搭配在 Xcode Cloud 中，让我们的构建流程更加顺畅。

> 本文基于 WWDC21 [Session 10204](https://developer.apple.com/videos/play/wwdc2021-10204) 、[Session 10261](https://developer.apple.com/videos/play/wwdc2021-10261)、[Session 10170](https://developer.apple.com/videos/play/wwdc2021-10170) 梳理。  

## 使用云签名在 Xcode 13 中分发应用程序

此次 Xcode 13 为分发应用程序带来了 3 个主要新功能，内容如下：

- 云签名
- 应用程序记录创建
- 内部版本号管理

通常我们开发完 App 之后，需要上传 TestFlight 进行内测，内测通过后上传 App Store 以供下载。中间管理分发需要使用 App Store Connect。

关系如图所示：

![avatar](https://images.xiaozhuanlan.com/photo/2021/e3d69816f5bdd9771f297f97fc70c7f8.png)

Xcode 13 在发行版中有一些新特性比较实用。在上传应用的时候，不需要在 App Store Connect  或本地 Mac 上执行任何操作就可以直接上传 App。

此外我们可以在上传之前直接在 Xcode 中创建或修改应用记录信息。

不同于之前手动修改 build version，Xcode 13 可以帮助我们自动管理 build version。

通过云签名，证书和私钥被存储在云中，这意味着在使用自动签名时，不再需要在本地手动设置证书和私钥。

![](https://images.xiaozhuanlan.com/photo/2021/b833247ddecf84c4b2239b8cfb26b06a.png)

这里可以选择签名方式：自动签名，手动签名。这个之前只在 Xcode 的设置页可以选择，现在这里也可以选择了。选择自动签名选项，可以让 Xcode 管理我们的签名，手动签名选项可以让我们操控整个签名的流程。

这里介绍下自动签名流程。我们之前本地签名的时候都是需要确保本机有证书、私钥等信息。在 Xcode 13 中，如果选择自动签名，我们不需要设置任何证书。证书和私钥都安全的存在云端，可以自动在云端完成签名。这种新的签名方式是支持 Xcode 所有的签名方式， 例如：App Store uploads, Developer ID, Enterprise。

这里我们有点疑问，这怎么保证安全的呢？接下来介绍下签名流程。

### 云签名流程


![](https://images.xiaozhuanlan.com/photo/2021/a941ed6b64be4069aa11bcae9a70092e.png)

整个过程如下：

1. Xcode 把开发者签名的 App 应用程序放在你的 Archive 中，并为它生成一个部分签名。此部分签名包含应用程序内容的哈希值。
2. 然后会 copy 一份哈希值副本上传到云端服务器。
3. 服务器使用私钥和证书在云中为你的应用程序签名。然后返回签名的剩余部分，并将其插入上面  Archive 生成的部分签名以完成整个签名过程。

如果想让具有开发人员角色的人使用云签名上传 App，我们可以通过选中 “Access to Cloud Managed Distribution Certificate” 复选框来授予他们在 App Store Connect 上的权限。如下图：

![](https://images.xiaozhuanlan.com/photo/2021/0a1f9c9f8aecfb2a0b0e98868cc0351a.png)

上传 App Store Connect 后，可以使用 TestFlight 进行测试，收集问题反馈，然后修复完后再继续打包上传。如此形成闭环流程如下图。

![](https://images.xiaozhuanlan.com/photo/2021/f5818e089d76a68c03ca2f6541451b64.png)



### Mac 应用程序分发流程

Developer ID 和 Mac App Store 是 macOS App 的两种主要发行方式，它们都是让你的应用走向世界的绝佳选择。

#### Mac App Store 方式

默认情况下，大多数 iOS 应用程序都可以在 Apple Silicon Mac 上直接使用。

如果想要在 Intel Macs 上分发，需要在新建一个程序。

我们同样可以使用云签名的分发方式进行上传，跟上面 iOS 的流程差不多。

我们可以使用跟 iOS 应用程序相同的 Bundle ID，我的 Mac 应用程序将使用与 iOS 版本相同的应用程序记录。这也意味着某些功能，将在两个平台上共享，如应用内购买。

Mac 应用程序也可以在 TestFlight 中使用了，这是 MacOS Monterey 中新推出的功能。文章下面会继续介绍 Mac 上的 TestFlight，见「Mac 上的 TestFlight 介绍」。

#### Developer ID 方式

开发者 ID 分发云（Developer ID distribution cloud）用开发者 ID 证书与 App 签约，并上传应用程序进行恶意软件检查，过程称为公证。一旦 App 签署并公证，macOS 将信任该应用程序在任何 Mac 上运行。

更多 Developer ID 公证的介绍，可以翻阅 2019 WWDC session：[All about notarization](https://developer.apple.com/videos/play/wwdc2019/703)。

### iOS App 内测的三种方式

#### Ad Hoc 证书分发

这种类型比较方便，使用个人开发者账号、公司开发者账号都可以，但是需要受限 100 个注册到我们团队的设备才能安装。如果有多个开发者账号设备 ID 的注册、管理也是一个比较头疼的事情。

#### 自定义应用程序

我们需要在 App Store Connect 设置自定义应用程序。这种方式可以定制应用自定义分发给某些企业、供应商或用户。企业可使用管理工具来管理哪些人能在应用市场下载该应用，可以选择向授权用户提供兑换代码，以便在 App Store 上下载该应用。

更多介绍，可以翻阅 2020 WWDC session: [Custom app distribution with Apple Business Manager](https://developer.apple.com/videos/play/wwdc2020/10667/)([《WWDC20 10667 利用 Apple Business Manager 定向分发 Custom App》](https://xiaozhuanlan.com/topic/1428697305))。

#### 企业证书分发

企业分发需要使用企业开发者账号，也就是 $299 的那个。这种方式不受限与设备个数，可以任何可用系统的设备安装。但是有过期时间，需要一年一换。

## Mac app 公证新工具 notarytool

自 macos10.14 以来，公证一直在帮助保护客户的应用程序，有关公证的更多介绍，可以翻阅 WWDC 2019 [All about Notarization](https://developer.apple.com/videos/play/wwdc2019/703/)。

### 公证的工作流程

![](https://images.xiaozhuanlan.com/photo/2021/968f12b2ce42118aa15d3ce62e66cd18.png)

步骤如下：

1. 开发人员构建他们的软件并将其发送给苹果服务后端进行公证。

2. 苹果公证服务将会运行自动分析工具，扫描软件中的恶意内容并检查代码签名问题。

3. 如果没有任何问题，公证服务将发布一张票据，当用户启动该软件时，该票据将被用做用户的 Mac 检索。

4. 验证完成后，苹果将验证结果返回给我们的开发人员。
5. 我们这时可以提供应用程序的网站供用户下载，用户下载软件后，macOS 会在软件运行之前使用公证票据对其进行检查是否合法。

苹果说在 15 分钟内完成 98% 的公证提交，5 分钟内可以完成大部分公证提交。

下面介绍下新推出的公证工具 notarytool。

### notarytool 介绍

notarytool 是公证的新命令行工具，提供了一个用于提交软件并等待结果的单行命令，更加方便简单。除了对客户端软件的更改之外，苹果还简化了服务。现在苹果已经建立了一个新的专门的服务后端，专注于可靠性只是公证。之前的 altool 方式已经废弃，我们需要尽快迁移到 notarytool 。

苹果号称 notarytool 方式比以前速度快 4 倍。反正苹果吹的是又好用又安全还快。

altool 跟 notarytool 使用对比：

![](https://images.xiaozhuanlan.com/photo/2021/43b691c8e8997b44f80e03affabf1117.png)

上图看到 altool 方式我们需要些很多代码，而 notarytool 仅需要一行即可。

使用 xcrun 调用 notarytool 示例：

```bash
% xcrun notarytool submit OvernightTextEditor_11.6.8.zip
                   --keychain-profile "AC_PASSWORD"
                   --wait
                   --webhook "https://example.com/notarization"
```

参数说明：

- submit：后面添加压缩资源包

- --keychain-profile：应用专用密码。（登录 [appleid.apple.com](http://appleid.apple.com/) ，点击 Generate Password，创建一个应用专用密码，这样可以避免暴露真实的密码。

- --wait：标志，允许您在一个命令中提交并等待公证结果。

- --webhook：运行公证完成后的通知链接。

如果公证成功，`notarytool`打印提交标识符：

```bash
Successfully uploaded file. 
 id: 2efe2717-52ef-43a5-96dc-0797e4ca1041
 path: /Users/janea/Desktop/OvernightTextEditor_11.6.8.zip
```

保存该`id`值以供稍后检查请求状态时使用。

上传您的应用程序后，公证过程通常需要不到一个小时。您可以使用`notarytool`该`history`命令来检查所有公证请求的状态：

```bash
% xcrun notarytool history --keychain-profile "AC_PASSWORD"
```

有关`notarytool`及其选项的更多信息，请参阅其手册页，或使用该`help`选项：

```bash
% xcrun notarytool --help
```

## Mac 上的 TestFlight 介绍

我们已经在 iOS 和 tvOS 上使用 TestFlight 很长时间了。现在苹果将在今年（2021年）秋天推出 Mac 版本的 TestFlight。

Mac 版本 TestFlight 功能：

- 安装测试版应用程序
- 自动更新应用
- 共享反馈信息
- 原生 Mac 应用程序
- M1 芯片 Mac 上的 iOS 应用程序

Mac 上的 TestFlight 如下图所示：

![](https://images.xiaozhuanlan.com/photo/2021/866018ff9b9bc3f1c5faca48ab7686b8.png)

#### 如何安装应用程序

我们可以通过发送电子邮件邀请或共享公共链接来招募测试人员。

他们可以浏览按版本分组的不同版本，选择要测试的特定版本，然后单击“安装”按钮下载并在 Mac 上开始测试。

![](https://images.xiaozhuanlan.com/photo/2021/fe5af40834df218c2842d7052cd6b56a.png)

为了跟其他应用程序区分，测试版本应用程序在 Dock、Launchpad 、Finder 中展示的时候旁边会有一个黄点。

![](https://images.xiaozhuanlan.com/photo/2021/17669629572118ec618d7a24efe7bfe2.png)

测试人员可以将自动更新配置为自动安装最新的可用版本。

#### 问题反馈

当我们遇到问题需要问题反馈的时候，找到我们的 App，点击 Send FeedBack 按钮，然后可以将截图拖进输入框，并写上文字描述即可。

![](https://images.xiaozhuanlan.com/photo/2021/28259401f78f2b259d79969ccf88912f.png)

随后，开发人员可以在 App Store Connect 的“截图反馈”部分下查看此反馈。

如果 beta 应用程序崩溃，那么 TestFlight 会自动捕获崩溃日志，并提供一个对话框供测试人员输入其他注释。

![](https://images.xiaozhuanlan.com/photo/2021/0a7e090ffb497cfbcb54f2d365e748cc.png)

我们可以在 App Store Connect crashes feedback 部分和 Xcode Organizer 中查看并下载反馈。

Xcode Cloud 详细介绍，可以翻阅 [窥探 Xcode Cloud](https://xiaozhuanlan.com/topic/7496513820) 和 [Meet Xcode Cloud](https://developer.apple.com/videos/play/wwdc2021/10267)([《【WWDC21 10267/10268/10269】窥探 Xcode Cloud》](https://xiaozhuanlan.com/topic/7496513820))。

## 总结

通过本文介绍，可以看出苹果对流程简化的大力投入，以及为了让我们的开发流程变得更加简单快捷在做不断改进优化：

1. iOS 签名技术的改进，构建过程中云签名的引入
2. Mac app 分发流程的优化
3. iPhone 、iPad App 可以直接运行在 Apple Silicon Mac 
4. Mac 版 TestFlight 的引入

同时也看出苹果对于 Mac 应用程序生态的布局与发力， 至于后面的发展是否能跟 iOS App 一样繁荣，我们可以拭目以待。
