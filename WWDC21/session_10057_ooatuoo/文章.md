# WWDC21 10057 - 让你的 iPad 应用更进一步

> 作者：atuo, iOS 开发者
>
> 审核：邦Ben，老司机技术周报编辑，就职于 BIGO 计算平台和商业化技术部，目前侧重于 Flutter 跨端开发。

> 本文基于 [WWDC21 10057 - Take your iPad apps to the next level](https://developer.apple.com/videos/play/wwdc2021/10057/) 梳理。

iPad 因为它强大的性能和可用性，正成为越来越多人的主力工具，iPadOS 15 带来了一些能让使用效率更进一步的新功能，让我们来一探究竟。

## 多任务
iPadOS 13 引入了多任务的能力，iPadOS 15 提供了新的 API 来提高这方面的体验。关于多窗口相关的基础概念在这里就不过多介绍了，想了解的可以观看 [WWDC 2019 - Introducing Multiple Windows on iPad](https://developer.apple.com/videos/play/wwdc2019/212/)。

### 窗口的展现方式
当向系统请求打开新的 scene 的时候，可以通过 `UIScene.ActivationRequestOptions` 来自定义一些效果，iPadOS 15 新加了一个其子类 `UIWindowScene.ActivationRequestOptions`，可以用来自定义 scene 的展现方式，有三个选项：`prominent`，`standard`，`automatic`。

`prominent` 是指在当前工作区中以模态呈现，其背后的场景变暗，突出新场景：
![](https://images.xiaozhuanlan.com/photo/2021/6ec3c569c689945249e2a2ef68805e37.png)

它像其他多任务的场景一样，可以被拖动调整，也可以移到应用程序架中以备后用。在考虑这种风格是否适合时，需要牢记一些准则：
1. prominent 出来的场景应当只为它自己使用，而不是用来为别的场景提供选项或者工具；
2. 应该提供“完成”或“关闭”按钮，并且应该用于显示应用中特定的内容，例如文档或文件；
3. 这个特定的内容范围应该在场景的激活条件 `UISceneActivationConditions` 中定义好。想要了解更多关于激活条件的内容，可以观看 [WWDC 2019 - Targeting Content with Multiple Windows](https://developer.apple.com/videos/play/wwdc2019/259/)。

`standard` 和原先的一样，即两个窗口并列的效果，允许交互，并且两边都提供主场景的功能。
`automatic` 即让系统去选择最适合的效果。

### Activation Action
除了支持自定义展现的方式，iPadOS 15 还新加一个自带的从新窗口打开的菜单按钮，你可以通 `UIWindowScene.ActivationAction` 来实现，用法如下：

```swift
// 创建一个 "Open In New Window" 的 action
let newSceneAction = UIWindowScene.ActivationAction({ _ in
    // 用来表示新 scene 中的内容
    let userActivity = NSUserActivity(activityType: "com.myapp.detailscene")
    // 返回激活的配置
    return UIWindowScene.ActivationConfiguration(userActivity: userActivity)
})

// 添加到菜单
let menu = UIMenu(children: [
    newSceneAction,
	  flagAction,
    shareAction
])
```

上面的代码，在 iPad 和 Mac Catalyst 下，菜单里会展示 “Open In New Window” 按钮，在 iPhone 上，因为不支持多场景，则不会显示这个按钮：
![](https://images.xiaozhuanlan.com/photo/2021/947f0eff1e065aa419b43ffb57e6a359.png)

如果你想在不支持多场景的时候展示别的按钮的话，可以提供一个可选 action，这样不用任何条件判断就能完成各种系统的适配：

``` swift
let showDetailSheetAction = UIAction(title: "Show Detail",
                                     image: UIImage(systemName: "ellipsis"),
                                     handler: { [weak self] _ in
    self?.present(DetailViewController(), animated: true, completion: nil)
})

let newSceneAction = UIWindowScene.ActivationAction(alternate: showDetailSheetAction) { _ in
	 // ...
})
```
![](https://images.xiaozhuanlan.com/photo/2021/6230ab807cf4c308177d24ab0be8d44e.png)

当然，打开新窗口的交互不止这一种，iPad 是一个触摸设备，可以很容易通过一个手势来打开新窗口，在记事本 app 中，你可以用双指捏合一个 cell 在新窗口打开笔记，UICollectionView 就内嵌了这一交互：

```swift
func collectionView(_ collectionView: UICollectionView, sceneActivationConfigurationForItemAt indexPath: IndexPath, point: CGPoint) -> UIWindowScene.ActivationConfiguration? {

    guard let activity = itemStore.sceneActivity(for: indexPath) else {
        // 如果不需要打开新的窗口，返回 nil
        return nil
    }

    return UIWindowScene.ActivationConfiguration(userActivity: activity)
}
```
如果想要支持别的视图，可以为其添加  `UIWindowScene.ActivationInteraction`：
``` swift
let interaction = UIWindowScene.ActivationInteraction { interaction, point in
    let activity = NSUserActivity(activityType: "customActivity")
    return UIWindowScene.ActivationConfiguration(userActivity: activity)
} errorHandler: { error in
    // 如果不支持此交互，在这里处理错误，可能是因为不支持多窗口，或是 scene 配置问题，以及系统资源却放
}

someView.addInteraction(interaction)
```
你可能注意到上面这些代码在创建 `UIWindowScene.ActivationConfiguration` 都只提供了 activity，其实它还可以自定义 `options` 和 `preview`，这两参数是可选的，如果没有提供，系统会尽可能地创建它：

``` swift
convenience init(userActivity: NSUserActivity, options: UIWindowScene.ActivationRequestOptions? = nil, preview: UITargetedPreview? = nil)
```

`options` 可以用来设置上文提到的窗口展现方式，`preview` 可以设置动画过程使用的视图，默认情况下，是把整个 cell 作为 target view：

![](https://images.xiaozhuanlan.com/photo/2021/09f8d79489bbda6939614321037ba58c.png)

但从整个 cell 展开成下面的新窗口的话，会不太自然， 这时候我们可以把 cell 中的 imageView 设为 target view，过渡的过程就会顺畅不少。

![](https://images.xiaozhuanlan.com/photo/2021/869edaba18cb9abf7b0c892f5d1537f2.png)

另外，有些时候你可能只是想简单地预览某个文件，iPadOS 15 也贴心地增加了 `QLPreviewSceneActivationConfiguration`，用来提供系统的的文件预览组件，不用自己再去写相关的 scene 了。

### 状态恢复
为了提供更好的体验，保存状态的时候，除了本身的内容之后，还应该保存视觉和交互的状态，如滚动位置、光标位置和第一响应者的状态。为此，UITextField 和 UITextView 都新增了  `interactionState` 属性，来表示当前交互的状态。

在之前，如果在 scene 连接时尝试恢复状态，这时 storyboard 和视图可能还未完全加载好，如果稍后恢复，当场景过渡到前台时，你又需要判断它是否是第一次展示。所以 iPadOS 15 为 UISceneDelegate 新加了一个方法  `scene(_:restoreInteractionStateWith:)` ，在 scene 被连接，并且 storyboard 加载完后和第一次过渡到前台前被调用。

恢复状态时你可能需要访问数据库或加载文件，并且在此期间不希望 UI 为空，考虑到这一点，iPadOS 15 允许请求短期延期。在此期间，启动图像将保持可见，同时仍允许主 RunLoop 执行。当 app 的内容加载完，需要发出信号来完成恢复。这个时间很短，所以不建议用来做类似网络请求这种耗时的操作。如果没有发出完成信号，或者耗时过久，系统将取消恢复：
```swift
func scene(_ scene: UIScene, restoreInteractionStateWith stateRestorationActivity: NSUserActivity) {
    guard let viewController = window?.rootViewController as? ViewController else { return }
    // 请求延时
    scene.extendStateRestoration()
    // 异步获取内容
    asyncDataStore.lookupString(for: "messageOfTheDay", thenCall: { message in
        // 恢复内容
        viewController.messageOfTheDayLabel.text = message
        // 发出恢复完成的信号
        scene.completeStateRestoration()
    })
}
```

## 键盘快捷键

iPadOS 15 带来了一个全新的展示键盘快捷键的页面，像 macOS 一样，把快捷命令分门别类，并且在底部增加了快捷键的搜索按钮，你可以长按 ⌘ 键来呼出。不过，样式上和 macOS 还是会有些不一样，iPadOS 会把子菜单拍平显示，并且不会有置灰的状态，而是直接隐藏。另外还新支持了“打印”的快捷键，只需在 info.plist 里添加 `UIApplicationSupportsPrintCommand`。
![](https://images.xiaozhuanlan.com/photo/2021/38ac2b8cee602f2441e9cbab0d5be175.png)

官方建议统所有快捷键都统一在 AppDelegate 的 `buildMenu(withbuilder:)` 方法中添加，而不是像之前那样分散在各个响应的地方，之前在别处创建的还能工作，只不过会被放到一个单独的未归类的栏目里：

``` swift
override func buildMenu(with builder: UIMenuBuilder) {
    super.buildMenu(with: builder)

    // 确保是在修改主系统菜单
    guard builder.system == .main else { return }

    // 创建指令
    let tabMenu = UIMenu(options: .displayInline, children: [
        UIKeyCommand(title: NSLocalizedString("New Tab", comment: ""),
                     action: #selector(ViewController.newTab(_:)),
                     input: "t",
                     modifierFlags: .command)
    ])

    // 插入到“文件”栏目下
    builder.insertChild(tabMenu, atStartOfMenu: .file)
}
```
系统提供了一些默认的菜单类别，像“文件”，“视图”等，当然你也可以自定义类别。UIMenuBuilder 要求每个类别必须有唯一的标识，包括每个指令以及其快捷键。如果重复添加，会不显示，并且输出相应的错误日志。比如，有两个快捷键分别用来切换列表或网格显示，通常我们会共用一个 action：`changeViewMode(_:)`，但这会导致添加失败，因为它们的标识相同，你可以通过设置 `propertyList` 来区分，或者使用不同的 action：

![](https://images.xiaozhuanlan.com/photo/2021/6cb06f08240a57c4382740564adad63b.png)

当快捷键被触发的时候，UIKit 会遍历响应链，找到响应者，调用 action 方法。关于响应链相关的内容，可以观看 [WWDC 2020 - Support hardware keyboards in your app](https://developer.apple.com/videos/play/wwdc2020/10109/) 和 [WWDC 21 - Qualities of a great Mac Catalyst app ](https://developer.apple.com/videos/play/wwdc2021/10053)。

在遍历响应链时，会调用两个 UIResponder 的方法：`canPerformAction (_:withSender:)`和`validate(_ command:)`，这让响应者有机会决定是否响应快捷键以及调整快捷键的样式。

在 iPadOS 15 中，UIKit 对响应链进行了重大调整，当应用通过焦点系统采用键盘导航时，响应链的遍历将从焦点开始，而不是第一响应者，这种变化能让快捷键更好地表现。举个例子，在相册列表中，你可以聚焦到一个 cell，按空格键来浏览图片，然后按 `⌘C` 来复制到别的 app ，你完全可以只用键盘来使用相册。想了解更多关于焦点系统的内容，可以观看 [WWDC 21 - Focus on iPad keyboard navigation](https://developer.apple.com/videos/play/wwdc2021/10260)。

除此之外，iPadOS 15 和 macOS 12 还加入了键盘快捷键的本地化，系统会自动根据键盘的类型来本地化快捷键。比如，美式键盘下的快捷键 `⌘ \`，在日式键盘中会改成 `⌘ ¥`，因为日式键盘没有 `\`。另外系统还会为 RTL 镜像快捷键，比如 `⌘[` -> `⌘ ]`。当然，你也可以禁用掉这些功能。

## 指针
iPadOS 15 带来了一些和 macOS 上相似但是使用 iPad 设计语言设计的指针交互。对于 iPad 的鼠标指针还不太熟悉的，可以查看 [WWDC 2020 - Build for the iPadOS pointer](https://developer.apple.com/videos/play/wwdc2020/10093/) 和 [Design for the iPadOS pointer](https://developer.apple.com/videos/play/wwdc2020/10640/)。

### 拖动框选
这是一种新的特定于指针的多选交互，使用过 Mac 的人都应该很熟悉。当您在集合视图中单击并拖动时，指针会拉伸成一个矩形，集合视图会选中矩形中包含的内容。

![](https://images.xiaozhuanlan.com/photo/2021/8cc2d1f890735783eb49712eeadc436a.png)这个交互内置在 UICollectionView 中，之前支持过双指多选手势的，将自动支持这一交互。

除了 UICollectionView，还新增了 `UIBandSelectionInteraction`，你可以在任意视图上支持这一交互：

```swift
let selectionInteraction = UIBandSelectionInteraction { interaction in
    if interaction.state == .selecting {
        // 根据 selectionRect 添加相应的内容
        self.selectItemsInRect(interaction.selectionRect)
    } else if interaction.state == .ended {
        self.finalizeSelection()
    }
}

view.addInteraction(selectionInteraction)
```

UICollectionView 的内置框选还支持一些常用的键盘快捷键，例如，在拖动开始时按住 Shift 会将选中内容添加到当前已选中去，而不是替换当前选中的内容；按住 ⌘ 可以切换选中状态。其他视图也可以通过 `UIBandSelectionInteraction.initialModifierFlags` 来实现，它会传入开始拖动时按住的键位。

### 辅助元素
另一个改进是可以给指针添加辅助元素，用以传达附加信息和提供上下文的提示。它们是独立的，围绕在指针周围以达到不同的效果，可以与任意指针样式组合：
![](https://images.xiaozhuanlan.com/photo/2021/5fce4e2773b65e5737c8acb7f88839cc.png)

通过 `UIPointerAccessory` 来创建，可以设置 `shape` 和 `position`，并且系统都提供一些默认的实现：

```swift
func pointerInteraction(_ interaction: UIPointerInteraction, styleFor region: UIPointerRegion) -> UIPointerStyle? {
    let preview = UITargetedPreview(view: view)
    let style = UIPointerStyle(effect: .lift(preview))
    style.accessories = [
        .arrow(.left),
        .arrow(UIPointerAccessory.Position(offset: 23, angle: .pi))
    ]
    return style
}
```

另外，`UIPointerRegion` 还引入了 `latchingAxes`，用来设置锁定轴，让你可以更方便地自由拖动，而不用担心指针脱离目标物：

![](https://images.xiaozhuanlan.com/photo/2021/5a4a626e13e72da9c8555ea204abef1b.png)

比如，下面这种需要拖动上下展开的交互，就可以将 `latchingAxes` 设置成 `vertical`，让指针更好地吸附在指示符上：

![](https://images.xiaozhuanlan.com/photo/2021/af3def5504c70d81e112e3692eeab1f4.png)

## 总结

iPadOS 15 带来了更轻松简便的多任务处理，全新的键盘快捷键和重新设计的菜单栏，以及鼠标指针的增强优化，能让用户更高效地完成任务，

从多窗口、键盘快捷键、指针这些优化和新特性来看，你会感觉 iPadOS 更像 macOS 了，好好利用这些特性，无疑能让你 iPad 应用更进一步。

Apple 今日预览了 iPadOS 15，包含多项充分利用了 iPad 独特性能的全新功能，以帮助用户提高效率，进一步提升 iPad 的多用性

以上就是 iPad OS 15 带来的主要更新，相信你能感受到这些变化带来的效率的提升。
