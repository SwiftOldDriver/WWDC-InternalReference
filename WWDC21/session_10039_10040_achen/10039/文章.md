# WWDC21 - Create ML 新妙用：手势分类和手部动作分类

本文基于 [Session 10039](https://developer.apple.com/videos/play/wwdc2021/10039) 梳理，如需了解更多，可以浏览文末的参考来源。

## 前言

WWDC18 苹果推出了 Create ML 框架,它极大的降低了机器学习的入门门槛。通过该框架，一个没有机器学习基础的苹果开发者也可以快速训练出自己的机器学习模型并将其应用到 iOS/Mac 应用中，且其模型在大小和运行速度方面都具备一定的优势，因此在发布当日即引起了开发者的追捧。过去几年，Create ML 及其他相关技术框架（例如 Vision）也得到了长足的发展，能力逐步细化到了各个实际应用场景中。今年，苹果给我们带来了新的应用场景：区分不同的手势和手部动作，同时还支持左右手的识别。在这篇文章中，我们一起来看看今年苹果带给我们的惊喜吧。

## 手势分类

讲完背景我们就开始实操；相信大家早已迫不及待想跟老司机一起上车了吧。手势分类作为一个典型的机器学习分类任务，其训练用的数据是静态图片。完整的工作流包含三大步骤：

- 收集必要的数据并分类和处理；
- 使用 Create ML 训练出分类模型；
-  集成模型到项目中使用。

![](https://cdn.nlark.com/yuque/0/2021/png/21877648/1624072231560-a074da54-2084-40ec-972c-a003e7a99ded.png)

### 步骤一：数据收集

首先你需要设计你需要识别的手势分类，同时引入一个 background 类别，用来降低误判。在 demo 演示中，苹果工程师设计了三种手势：单指、双指、五指张开，分别代表了不同的含义。分别需要为三种手势准备数十至数百张图片，作为训练的素材。图片的张数和质量很大程度上决定了训练得到的模型质量，因此，我们需要对其进行严格把关。多种手势的照片数量建议保持相对均匀（演示中每一种手势大概 500 张图），background 类别可以适当增加数量。background 类别中需要放置一些无手势或者容易混淆的过渡手势、不用识别的其他手势等等，这样就可以把它们和你关心的手势区分开。准备好图片后把它们分别放到类别命名的文件夹中：
![](https://cdn.nlark.com/yuque/0/2021/png/21877648/1624098660631-f5e01cc9-e055-4412-9bd4-03ac255227d4.png)

### 步骤二：模型训练

Create ML 新增了一个手势识别分类的模版，使用起来非常容易。我们使用模版导入图片文件夹（在 Model Source 的标签下新建并导入）后就可以开始模型的训练：

![](https://cdn.nlark.com/yuque/0/2021/png/21877648/1624072197024-b9a7c61c-60c4-4231-90e8-2ac335611879.png)

苹果的图片任务训练都分为两步骤，其中第一步骤是特征提取，此步骤是比较耗时的。选取合适的迭代次数和其他的一些训练设置后，我们点击训练，经过一段时间的等待，不出意外我们就可以得到一个训练好的模型了。

![](https://cdn.nlark.com/yuque/0/2021/png/21877648/1624072237873-be324b88-7090-4c27-8a0c-6f9e406ad0c2.png)

在 Create ML 中，预览是一个非常实用的功能；通过它我们可以轻而易举的验证我们的成果。在示例中，工程师通过摄像头演示了几个手势，可以看到屏幕中实时的显示出检测到的手势并展示了分类的置信度；这看起来相当酷，也非常实用。

![](https://cdn.nlark.com/yuque/0/2021/png/21877648/1624072192126-2449f4c7-18c8-4bcf-a6b5-dcec29c91793.png)

### 步骤三：应用集成

一位来自 Vision 框架团队的苹果工程师通过一个 ARKit 相关的 demo 来演示该模型的能力。具体而言，她通过摄像头对手势的实时监测，在手指上加特效（类似魔法光束，看起来很酷炫）；一根手指、两根手指、整个张开的手掌这三个手势分别是不同的特效，并且特效还可以跟随手指的位置移动。这个 demo 很好的利用了以下的技术能力：

- 利用ARKit来获取摄像头的实时图像帧（也可以使用其他方法，比如AVCaptureOutput来实现类似的功能）；
- 利用Vision框架来实时的识别和跟踪手掌；
- 利用CreateML训练得到的模型来识别手势。

除此之外，Vision 框架中新增了一个枚举属性可以用来区分左右手。这个能力的检测是每个手独立的（左手不依赖右手，所以如果有误判的话可能两只都是左手或者右手，哈哈）。通过这个 api，在应用中我们可以实现两只手的分别控制，使用起来很简单，却很使用。

## 手部动作的分类

手势分类应用讲完了我们下面再聊聊手部动作的分类。 WWDC20 有一个 session 给大家介绍了如何通过 Create ML 训练一个动作分类器，手部动作的分类只是训练所使用的素材不同，其他的过程很相似，因此建议大家如果要自己上手训练手部动作分类模型的话，也可以先找到那个 session 来回顾下（参看文末的参考来源连接）。

动作和手势的不同点在于一个是手势的序列，一个是单个手势；对应来说就是视频和单帧图像的区别。对于视频的素材收集相对来说更难，数据量也更大。
素材的使用有两种方式，一种是和之前的手势识别类似，只是素材改为视频：

下面我们介绍下如何使用训练得到的模型来在 App 中实现动作检测。

使用FIFO(先入先出)的队列来累计存放手势，从而实现特定手部动作的识别：

	var queue = [MLMultiArray]()
	// ...
	//控制数据填充的速度
	frameCounter += 1
	if frameCounter % 2 == 0 {
	    let hands: [(MLMultiArray, VNHumanHandPoseObservation.Chirality)] = getHands()
		//判断左右手，仅右手才进行相关操作
	    for (pose, chirality) in hands where chirality == .right {
	        queue.append(pose)
	        queue = Array(queue.suffix(queueSize))
	        queueSamplingCounter += 1
	        if queue.count == queueSize && queueSamplingCounter % queueSamplingCount == 0 {
				// 组装输入
	            let poses = MLMultiArray(concatenating: queue, axis: 0, dataType: .float32)
				// 执行分类操作
	            let prediction = try? handActionModel?.prediction(poses: poses)
	            guard let label = prediction?.label, 
	              let confidence = prediction?.labelProbabilities[label] else { continue }
	            if confidence > handActionConfidenceThreshold {
	                DispatchQueue.main.async {
	                    self.renderer?.renderHandActionEffect(name: label)
	                }
	            }
	        }
	    }
	}
	

输入的维度多了一个 45，这个 45 是怎么来的呢？它实际和我们选取的动作长度有关： 45 等于 1.5 乘以 30 ， 1.5 是动作的时长，30 是帧速率。可以看到上面的代码里每 2 帧采样 1 次，是因为 demo 中的代码 ARSession 刷新频率是每秒 60 次，通过这种采样方式才可以和我们的模型对应上。
![](https://cdn.nlark.com/yuque/0/2021/png/21877648/1624072233948-98874754-a248-4e0a-b05f-cbd9db92308e.png)

另外一个值得注意的是性能问题：可以想象到，这种基于动作序列的预测性能上的代价应该远大于单个手势识别。因此，多久执行一次动作序列的预测成为了一个关键决策。你需要在应用流畅性和检测的实时性上进行一个平衡，选取合适的检测频率。

## Tips

基于实际项目经验，我们再送给大家几个小Tip：

- 手到镜头的距离，尽量不要超过3.5米；
- 光线不要太强也不要太暗；
- 最好不要戴手套，这会让识别变得困难；
- 保证充足、高质量的训练数据，包括背景分类的数据。
