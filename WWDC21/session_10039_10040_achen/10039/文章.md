# WWDC21 - Create ML新妙用：手势分类和手部动作分类

## 前言
2018年WWDC苹果推出了Create ML框架,它极大的降低了机器学习的入门门槛。通过该框架，一个没有机器学习基础的苹果开发者也可以快速训练出自己的机器学习模型并将其应用到iOS/Mac应用中，其模型在大小和运行速度方面都具备一定的优势，在发布当日即引起了开发者的追捧，笔者有幸得以在现场见证了这个激动人心的时刻。过去几年，Create ML和相关框架也得到了长足的发展，其能力逐步细化到了各个实际应用场景中。今年，苹果给我们带来了新的应用场景：区分不同的手势和手部动作，同时还支持左右手的识别。在这篇文章中，我们一起来看看今年苹果带给我们的惊喜吧。
## 关于手，手势，和手部动作
![](https://cdn.nlark.com/yuque/0/2021/png/21877648/1624072234951-56a86777-acb7-45bf-aa08-72a87d7464d0.png)
在我们深入新技术之前，我们先来聊一聊手:手是人最重要的器官之一，它由很多的骨头、关节和肌肉组成，可谓是一个工程学上的奇迹。抛开它的复杂性不提，手是新生儿与世界互动的第一个工具。在婴儿会说话之前他们就开始尝试使用简单的手部动作来表达自己的意图。在我们学会说话以后，手继续在表达信息方面扮演着重要角色。在新冠大流行的一年里，人们通过手势来拉近彼此之间的距离。

2020年，vision 框架引入了手势识别，它可以帮助开发者识别出画面中的手，以及每一只手所包含的21个关节点。如果你想识别出屏幕里是否有一只手，那么它是一个很棒的工具；但如果你想知道屏幕里的手具体在干什么，仅仅使用手势识别功能目前还无法做到。而这就是我们今天要给大家介绍的：使用Create ML来对手势和手部动作分类，从而获知屏幕中的手势/动作所表达的含义。举个例子，我们可以通过手势来表达停止、安静、和平；通过手部动作表达后退、走开、过来等等，大家可以尽情想象它们可以为我们的应用带来什么变化！

手势分类（静态图像）：
众所周知，把大象放进冰箱只需要三步；而手势分类作为一个典型的机器学习分类任务，工作流包含3个大的步骤：
1. 收集必要的数据并分类和处理；
2. 使用Create ML训练出分类模型；
3. 集成模型到项目中使用。

step1（数据收集）:
设计你需要识别的手势分类，同时引入一个background类别，用来降低误判。在demo演示中，苹果工程师设计了3种手势：单指、双指、五指张开，分别代表了不同的含义。分别需要为3种手势准备数十至数百张图片，作为训练的素材。对于一个机器学习任务来说，图片的张数和质量很大程度上决定了训练得到的模型质量；因此，这一步骤的工作虽然枯燥，但却非常重要。多种手势的照片数量建议保持相对均匀（演示中每一种手势大概500张图），background类别也可以适当增加数量。background类别中需要放置一些无手势或者容易混淆的过渡手势、不用识别的其他手势等等，这样就可以把它们和你关心的手势区分开。准备好图片后把它们分别放到类别命名的文件夹中。

step3（应用集成）:
一位来自Vision框架团队的苹果工程师通过一个arkit相关的demo来演示该模型的能力。具体而言，她通过摄像头对手势的实时监测，在手指上加特效（类似魔法光束，看起来很酷炫），1根手指，2根手指，整个手掌分别是不同的特效，并且魔法光束还可以跟随手指的位置移动。这个demo很好的利用了以下的技术能力：
1.利用ARKit来获取摄像头的实时图像帧（也可以使用其他方法，比如AVCaptureOutput来实现类似的功能）；
2.利用Vision框架来实时的识别和跟踪手掌；
3.利用CreateML训练得到的模型来识别手势。

Vision框架中新增了一个枚举属性可以用来区分左右手。这个能力的检测是每个手独立的（左手不依赖右手，所以如果有误判的话可能两只都是左手或者右手，哈哈）。通过这个api，在应用中我们可以实现两只手的分别控制，使用起来很简单，却很使用。

手势的识别讲完了我们下面再聊聊对于手部动作的识别。 WWDC20有一个session给大家介绍了如何通过CreateML训练一个动作分类器，我们这个手部动作的识别只是训练所使用的素材不同，其他的过程很相似，建议大家如果要自己上手训练手部动作分类模型的话，也可以先找到那个session来回顾下。

动作和手势的不同点在于一个是手势的序列，一个是单个手势；对应来说就是视频和单帧图像的区别。对于视频的素材收集相对来说更难，数据量也更大。在文件夹中需要用到.csv或者.json来存放一些视频相关的辅助信息（WWDC20有介绍，这里就不展开说了）。

在展示的demo中，苹果演示了通过动作触发的新魔法，效果看起来也相当酷。下面我们介绍下如何使用训练得到的模型来在App中实现动作检测。

使用FIFO(先入先出)的队列来累计存放手势，从而实现特定手部动作的识别：
	var queue = [MLMultiArray]()
	// . . .
	//控制数据填充的速度
	frameCounter += 1
	if frameCounter % 2 == 0 {
	    let hands: [(MLMultiArray, VNHumanHandPoseObservation.Chirality)] = getHands()
	//判断左右手，仅右手才进行相关操作
	    for (pose, chirality) in hands where chirality == .right {
	        queue.append(pose)
	        queue = Array(queue.suffix(queueSize))
	        queueSamplingCounter += 1
	        if queue.count == queueSize && queueSamplingCounter % queueSamplingCount == 0 {
	// 组装输入
	            let poses = MLMultiArray(concatenating: queue, axis: 0, dataType: .float32)
	// 执行分类操作
	            let prediction = try? handActionModel?.prediction(poses: poses)
	            guard let label = prediction?.label, 
	              let confidence = prediction?.labelProbabilities[label] else { continue }
	            if confidence > handActionConfidenceThreshold {
	                DispatchQueue.main.async {
	                    self.renderer?.renderHandActionEffect(name: label)
	                }
	            }
	        }
	    }
	}

输入的维度多了一个45，这个45是怎么来的呢？它实际和我们选取的动作长度有关：45等于1.5乘以30， 1.5是动作的时长，30是帧速率。可以看到上面的代码里每2帧采样1次，是因为demo中的代码ARSession刷新频率是1秒60次，通过这种采样方式才可以和我们的模型对应上。另外一个值得注意的是性能问题：可以想象到，这种基于动作序列的预测性能上的代价应该远大于单个手势识别。因此，多久执行一次动作序列的预测成为了一个关键决策。你需要在应用流畅性和检测的实时性上进行一个平衡，选取合适的检测频率。

基于实际项目经验，我们再送给大家几个小Tip：
1.手到镜头的距离，尽量不要超过3.5米；
2.光线不要太强也不要太暗；
3.最好不要戴手套，这会让识别变得困难；
4.保证充足、高质量的训练数据，包括背景分类的数据。
