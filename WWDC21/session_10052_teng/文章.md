## Mac Catalyst 是什么

Apple 在 WWDC19 上宣布了 [Mac Catalyst](https://developer.apple.com/documentation/uikit/mac_catalyst?language=objc) 技术，它可以让我们为现有的 iPad App 创建 Mac 版本，并且可以充分利用 Mac 的大显示屏、键盘、鼠标或触控板，同时我们只需要维护一份工程代码。在 Xcode 11 及更高版本中，我们只需要勾选一个复选框即可 [开启 Mac Catalyst 技术](https://developer.apple.com/cn/documentation/xcode/creating_a_mac_version_of_your_ipad_app/)。如果你的 App 只有 iPad 版本而没有 Mac 版本，而且主要功能不是 macOS 不支持的（比如 ARKit），或者你不想继续维护你的 Mac App，仅想通过一份工程代码构建 iPad 和 Mac App，那么 Mac Catalyst 非常适合你，你可以用极少的成本来获得一个 Mac App。

Mac Catalyst 是将 iPad App 带到 Mac 上的一种方式，它既支持 Intel Mac 也支持 ARM Mac（M1 Mac）。还有一种方式是 [iPhone and iPad Apps on Apple Silicon Macs](https://developer.apple.com/macos/iphone-and-ipad-apps/)，它只支持 ARM Mac，你可以在[《WWDC20 内参》- WWDC20 10114 - Apple Silicon Mac 上的 iPad 与 iPhone App 运行](https://xiaozhuanlan.com/topic/1845067932) 中了解更多。

下面我将开始讲解今年 WWDC 中 Mac Catalyst 的更新。如果你还没有接触过 Mac Catalyst 但对此感兴趣的话，可以在文末获取相关学习资料。

## Mac Catalyst 有哪些更新

- 更多的按钮样式
- ToolTips

- 支持为窗口添加副标题
- 支持快捷指令
- 支持在 Mac Idiom 下保持自定义的 button 和 slider 样式
- 支持禁用自由组合窗口功能
- 光标的更新

### 更多的按钮样式

我们先来看一个 Mac Catalyst 的配置选项。当你勾选 Mac Catalyst 复选框时，你可以 [选择以 iPad 或是以 Mac 风格的用户界面来构建 Mac App](https://developer.apple.com/documentation/uikit/mac_catalyst/choosing_a_user_interface_idiom_for_your_mac_app?language=objc)。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624375042511-11599b06-4743-4b64-b6bb-283bc3295078.png#clientId=u279b7986-353f-4&from=paste&height=347&id=fKQX8&margin=%5Bobject%20Object%5D&name=image.png&originHeight=347&originWidth=926&originalType=binary&ratio=1&size=50468&status=done&style=shadow&taskId=u28dbd4d5-69ae-45d7-b1ab-3c257742cd8&width=926)

- Scale Interface to Match iPad（iPad Idiom）。当你勾选 Mac Catalyst 复选框时，默认会选择该方案，它将保持 iOS 原生的 UI，仅通过对你的 iPad App 进行缩放来构建 Mac Catalyst App，这样你就不需要在界面适配上下太大功夫，可以快速将 iPad App 带到 Mac 上。但视图和文本在 macOS 上会被缩放为原来的 77%，这样会导致视图和文本有些模糊。
- Optimize Interface for Mac（Mac Idiom）。这是 Apple 在 WWDC20 上发布的新方案，UI 将会被自动转换为 macOS 原生的 UI，视图和文字大小将 100% 呈现而不是缩放，让你的 App 更像一个 masOS App 从而带来更好的体验。当然付出的代价就是需要花费更多时间和精力去做适配。

笔者前段时间负责了[《宝宝巴士》](https://apps.apple.com/cn/app/%E5%AE%9D%E5%AE%9D%E5%B7%B4%E5%A3%AB-%E5%AE%9D%E5%AE%9D%E6%80%9D%E7%BB%B4%E7%9B%8A%E6%99%BA%E5%90%AF%E8%92%99%E5%A4%A7%E5%85%A8/id1031121586)App 的 Mac Catalyst 版构建，已上架 App Store。我们为节约成本选择了 Scale Interface to Match iPad 方案，并且我们禁止了窗口自由缩放功能，同时将 App 窗口大小固定按 iPad 9.7 寸的比例进行缩放，以极小的成本完成了 UI 的适配。当然将来如果要做 Mac 版优化的话，Optimize Interface for Mac 是首选方案。你可以根据自己的需求来选择其中一种方案。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624802078872-67e3ffab-e8c8-49a9-8a0f-70f94cadc7f9.png#clientId=u8e681558-0371-4&from=paste&height=1965&id=u76bc6f4b&margin=%5Bobject%20Object%5D&name=%E5%AE%9D%E5%AE%9D%E5%B7%B4%E5%A3%AB.png&originHeight=1965&originWidth=4247&originalType=binary&ratio=1&size=5778646&status=done&style=shadow&taskId=u8a70144e-c865-4fe9-b76e-23e1a2efffd&width=4247)

好的，接下来我们正式进入 Mac Catalyst 今年带来的第一个更新 —— 更多的按钮样式。

在 macOS Big Sur（iOS 14）中，Apple 给 UIControl 添加了 [showsMenuAsPrimaryAction](https://developer.apple.com/documentation/uikit/uicontrol/3601223-showsmenuasprimaryaction/) 属性，这允许你将 Pull-down menu 附加到 UIButton 上。

> 由于 Mac Catalyst 是 UIKit for Mac，所以其 macOS 版本与 iOS 版本有着对应关系。

现在，在 macOS Monterey（iOS 15）中，Apple 给 UIButton 添加了一个新属性 [changesSelectionAsPrimaryAction](https://developer.apple.com/documentation/uikit/uibutton/3752184-changesselectionasprimaryaction/)，该属性使按钮的标题可以跟随 menu 中的选择而改变。

有了这两个属性，现在我们有四种方法可以配置按钮。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624201509540-dd469e51-35c6-4794-8c9b-a67989ba9b3b.png#clientId=u23cdbea8-a944-4&from=paste&height=733&id=u15f797d5&margin=%5Bobject%20Object%5D&name=%E6%88%AA%E5%B1%8F2021-06-20%20%E4%B8%8B%E5%8D%8811.03.15.png&originHeight=733&originWidth=1298&originalType=binary&ratio=1&size=119494&status=done&style=shadow&taskId=ub790dfbe-3d36-4a52-acb8-2b81dd2e811&width=1298)

1. Push Button。当这两个属性都设置为 false 时，也就是默认情况下，我们将得到一个标准按钮。但如果在单击或长按按钮时设置了 menu，那么仍可以显示该 menu。

```swift
let button = UIButton(type: .system)
```

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624379986054-eca0057a-1f55-4c86-b614-f13c66b1a8ff.png#clientId=u279b7986-353f-4&from=paste&height=360&id=u07486c02&margin=%5Bobject%20Object%5D&name=Push%20Button.png&originHeight=360&originWidth=2048&originalType=binary&ratio=1&size=78009&status=done&style=shadow&taskId=u8f7d806f-82af-46ca-bcce-85556cb2e9f&width=2048)

可以看到，在 iPad Idiom 下，Mac App 中的 button 被缩小至了 77% 大小，字体会显得有些模糊。而在 Mac Idiom 下，button 被自动转换为了 macOS 原生的 UI。

2. Toggle Button。如果仅 changesSelectionAsPrimaryAction 设置为 true，那么会得到一个 Toggle Button，单击该按钮将改变 selected 的值并切换呈现打开或关闭状态。经测试，Toggle Button 仅在 Mac Idiom 下有效，在 iPad Idiom 下其等同于 Push Button。

```swift
let button = UIButton(type: .system)
button.changesSelectionAsPrimaryAction = true
```

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624384742892-c7a7c9ea-935c-463b-8a83-c341f842bf2e.png#clientId=u279b7986-353f-4&from=paste&height=360&id=u9b8b9b6a&margin=%5Bobject%20Object%5D&name=Toggle%20Button.png&originHeight=360&originWidth=2048&originalType=binary&ratio=1&size=93588&status=done&style=shadow&taskId=u0ce8e70c-ca77-4904-b167-6363f9bc2d8&width=2048)

如果我们在 Toggle Button 的基础上设置了 menu，但 showsMenuAsPrimaryAction 值为 false，那么当我们长按按钮时将显示该 menu。

3. Pull-down menu Button。如果仅 showsMenuAsPrimaryAction 设置为 true，我们将得到一个附带 Pull-down menu 的按钮，在点击按钮时出现 menu。

```swift
let button = UIButton(type: .system)
button.menu = UIMenu()
button.showsMenuAsPrimaryAction = true
```

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624381296832-913afa03-f097-4c62-be3d-701d5bfbc8a2.png#clientId=u279b7986-353f-4&from=paste&height=840&id=uc6d21759&margin=%5Bobject%20Object%5D&name=Pull%20Down.png&originHeight=840&originWidth=2648&originalType=binary&ratio=1&size=257631&status=done&style=shadow&taskId=u0f3c6a03-f8d5-4da6-bbdf-d0a06db580f&width=2648)

4. Pop-up menu Button。如果这两个属性都设置为 true，我们将得到一个附带 Pop-up menu 的按钮，在点击按钮时出现 menu，且按钮的标题可以跟随 menu 中的选择而改变。虽然我们使用 Pull-down Button 也可以实现按钮标题跟随改变的效果，但 Pop-up Button 使之更方便，而且在按钮样式和 menu 样式上也存在差异。

```swift
let button = UIButton(type: .system)
button.menu = UIMenu()
button.showsMenuAsPrimaryAction = true
button.changesSelectionAsPrimaryAction = true
```

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624381928472-a75eb7d5-2ee1-477c-85b1-77504bb7edc4.png#clientId=u279b7986-353f-4&from=paste&height=840&id=u96cc36ba&margin=%5Bobject%20Object%5D&name=Pop%20Up.png&originHeight=840&originWidth=3448&originalType=binary&ratio=1&size=316116&status=done&style=shadow&taskId=ue6ed4032-6405-4938-a921-f24e1489784&width=3448)<br />现在，对于设置了 showsMenuAsPrimaryAction 为 true 的按钮，在 macOS 上通过鼠标左键点击按钮时，将立即显示 menu。对于未设置 showsMenuAsPrimaryAction 为 true 但设置了 menu 的按钮，其在 macOS 上的行为取决于 App 的 idiom：

- 在 iPad Idiom 中，通过鼠标右键点击按钮来显示 menu。
- 在 Mac Idiom 中，通过长按按钮来显示 menu。

你可以在 “[Meet the UIKit button system](https://developer.apple.com/videos/play/wwdc2021/10064/)” 视频中了解更多有关 UIButton 的更新，例如配合 iOS 15 的 [UIButtonConfiguration](https://developer.apple.com/documentation/uikit/uibuttonconfiguration/) API 配置更多样的按钮。

### ToolTips

在 macOS Monterey（iOS 15）中，Apple 为 Mac Catalyst App 提供了 ToolTips。ToolTips 是一个小的浮动窗口，当光标悬停在 App 中的某个视图上一小段时间后会出现，它可以用来解释这个视图的用途或其他细节。

我们可以创建一个带有文本的 [UIToolTipInteraction](https://developer.apple.com/documentation/uikit/uitooltipinteraction/) 实例，然后将其添加到视图上。

```swift
let toolTipInteraction = UIToolTipInteraction(defaultToolTip: string)
view.addInteraction(tooltipInteraction)
```

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1625502591073-2f27ff07-61c8-4539-9e0b-88797400452a.png?x-oss-process=image%2Fresize%2Cw_748)


ToolTips 的一个常见用法是阐明 UI 控件的行为，所以 Apple 为 UIControl 添加了一个 [toolTip](https://developer.apple.com/documentation/uikit/uicontrol/3750832-tooltip/) 属性，可以快速设置 toolTip，相当于是使用 UIToolTipInteraction 的快捷方式吧。

```swift
control.toolTip = "Enable updates"
```

在 iOS 中当 UILabel 的文本过长而无法容纳时，超出的文本会被截断。Apple 给 UILabel 添加了 [showsExpansionTextWhenTruncated](https://developer.apple.com/documentation/uikit/uilabel/3750862-showsexpansiontextwhentruncated/) 属性，来添加 UILabel 对 ToolTips 的支持，当我们将鼠标悬停在 label 上时，它将显示完整的文本。

```swift
label.showsExpansionTextWhenTruncated = true
```

### 新增 “Print” 和 “Export as PDF” 菜单项

在 macOS Monterey 中，Mac Catalyst App 的打印功能将变得更方便。

Apple 在 iOS 15 中新增了一个名为 [UIApplicationSupportsPrintCommand](https://developer.apple.com/documentation/bundleresources/information_property_list/uiapplicationsupportsprintcommand/) 的 Info.plist 的 key。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624719422356-81a1a3af-1f55-4bc8-9f48-6f9f335142a7.png#clientId=u8e681558-0371-4&from=paste&height=1040&id=XrPP6&margin=%5Bobject%20Object%5D&name=WX20210626-225629.png&originHeight=1040&originWidth=1849&originalType=binary&ratio=1&size=330606&status=done&style=shadow&taskId=uada7b7d4-3e7f-4b6f-a565-9eb02886047&width=1849)

如果我们将其添加到 Info.plist 中并设置为 true，那么当我们启动 Mac Catalyst App 时将会自动拥有 “Print” 和 “Export as PDF” 两个菜单项（归类在 “File” 菜单下）。我们可以使用快捷键 Command-P（⌘P）来使用 “Print” 功能。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1625503881714-bf9a440b-d7f0-4f58-8485-491c30af7c7b.png?x-oss-process=image%2Fresize%2Cw_748)

我们也可以将该 plist key 添加到 iOS App 中，它会在 [iPadOS 键盘快捷键](https://support.apple.com/zh-cn/HT211096) 上显示 “Print” 选项，其快捷键也为 ⌘P。当该 iOS App 在 M1 Mac 上运行时，将自动拥有 “Print” 和 “Export as PDF” 两个菜单项，同 Mac Catalyst App。


![](https://cdn.nlark.com/yuque/0/2021/jpeg/12376889/1625505895123-c5f094cc-8cde-4028-ad6e-f05d4a32d436.jpeg?x-oss-process=image%2Fresize%2Cw_748)

虽然这个 plist 键告诉操作系统我们需要 “Print” 和 “Export as PDF” 菜单项，但是我们还需要在代码中实现 “Print” 和 “Export as PDF” 功能。否则这个两个菜单项呈灰显状态且不可点击。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1625503564111-8179e0bd-638a-44df-a56c-be2450444d58.png?x-oss-process=image%2Fresize%2Cw_748)

为此，Apple 给 UIResponder 添加了一个 [printContent](https://developer.apple.com/documentation/uikit/uiresponderstandardeditactions/3750910-printcontent/) 方法，我们可以在任何一个 UIResponder 里实现这个方法来处理打印功能。当我们点击 “Print” 或 “Export as PDF” 菜单项时，UIKit 会在响应者链中查找可以执行此操作的对象来处理打印。比如，我们在一个 viewController 中实现了 printContent 方法，当 App 焦点和第一响应者状态转移到这个 viewController 时，点击 “Print” 菜单项或者使用快捷键 ⌘P，该 viewController 的 printContent 方法就会被调用，我们只需在该方法中像平常一样设置和使用 [UIPrintInteractionController](https://developer.apple.com/documentation/uikit/uiprintinteractioncontroller/) 就行。

```swift
override func printContent(_ sender: Any?) {
    let printInteractionController = UIPrintInteractionController.shared
    ...
}
```

![](https://cdn.nlark.com/yuque/0/2021/jpeg/12376889/1625503734322-c1194dda-7d10-476d-a77d-f4e17de51a90.jpeg?x-oss-process=image%2Fresize%2Cw_748)

在某些情况下，当前的响应者可能不想执行打印，需要由业务逻辑决定。这时候我们可以重写 [canPerformAction](https://developer.apple.com/documentation/uikit/uiresponder/1621105-canperformaction/) 方法来自定义逻辑。canPerformAction 方法会在 printContent 方法之前被调用，如果返回 false，当前的响应者将被跳过，UIKit 会在响应者链中查找下一个可以执行打印操作的响应者。

```swift
override func canPerformAction(_ action: Selector, withSender sender: Any?) -> Bool {
    if action == #selector(self.printContent(_:)) {
        ...
    } else {
        return super.canPerformAction(action, withSender: sender)
    }
}
```

我们可以在接近响应者链顶部的响应者（比如 rootViewController）中实现 [targetForAction:withSender:](https://developer.apple.com/documentation/uikit/uiresponder/1621146-targetforaction/) 方法来处理打印操作，这在我们想要打印某些页面集，但实现打印的对象又不在响应者链中时非常有用。这种处理方式比我们通过给需要打印的对象发送 [becomeFirstResponder](https://developer.apple.com/documentation/uikit/uiresponder/1621113-becomefirstresponder/) 消息来强制该对象进入响应者链的方式要好得多，而且在这种情况下使用 becomeFirstResponder 将是一种反模式。

```swift
override func target(forAction action: Selector, withSender sender: Any?) -> Any? {
    switch action {
    case #selector(UIResponder.printContent(_:)):
        ...
    default:
        return super.target(forAction: action, withSender: sender)
    }
}
```

### 支持在 Mac Idiom 下保持自定义的 button 和 slider 样式

Apple 不建议我们使用 iPad Idiom 来构建 Mac Catalyst App，而是建议我们使用 macOS 原生的 UI。但有些情况下，可能我们的某些 button 或 slider 需要适配 App 的风格主题，因此想要它们保持 iPad Idiom，于是 Apple 今年做了支持。

UIButton 和 UISlider 各有两个新属性 [behavioralStyle](https://developer.apple.com/documentation/uikit/uibutton/3752180-behavioralstyle/) 和 [preferredBehavioralStyle](https://developer.apple.com/documentation/uikit/uibutton/3752181-preferredbehavioralstyle/)。我们可以通过给 button 和 slider 设置 preferredBehavioralStyle 值来决定其采用 iPad Idiom 还是 Mac Idiom，并且它不会影响到我们的 iOS App，这样我们就可以在不影响 iOS App 的情况下适配 Mac Catalyst App。

### 支持禁用自由组合窗口功能

之前，你可以在 Info.plist 中，将 [UIApplicationSceneManifest](https://developer.apple.com/documentation/bundleresources/information_property_list/uiapplicationscenemanifest/) 中的 [UIApplicationSupportsMultipleScenes](https://developer.apple.com/documentation/bundleresources/information_property_list/uiapplicationscenemanifest/uiapplicationsupportsmultiplescenes/) 值设置为 true 来让你的 Mac Catalyst App 支持创建多窗口。默认情况下，支持多窗口的 Mac Catalyst App 的 [tabbingMode](https://developer.apple.com/documentation/appkit/nswindow/1644729-tabbingmode/) 是 [NSWindow.TabbingMode.automatic](https://developer.apple.com/documentation/appkit/nswindow/tabbingmode/automatic)。这会为你的 App 添加 Show/Hide Tar Bar、Show All Tabs/Exit Tab Overview 菜单项（归类在 “View” 菜单下），且允许你的 App 自由组合窗口。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1625504024105-d88ee2a9-6122-46e2-9ba3-522be7acaeb1.png?x-oss-process=image%2Fresize%2Cw_748)

点击 Show Tab Bar，然后你就可以通过 “+” 按钮创建新的 tab。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1625504269952-7259db8a-1354-4631-960d-c4b6b295a5ea.png?x-oss-process=image%2Fresize%2Cw_748)

你可以长按 tab（这里指视图，而非 tab 键）将它拖出来形成新的窗口，同时也支持你再拖回去。

![](https://cdn.nlark.com/yuque/0/2021/jpeg/12376889/1625504396920-dc4fd996-0d71-43bd-b662-a82a97765887.jpeg?x-oss-process=image%2Fresize%2Cw_748)

你也可以点击 Show All Tabs 进入 Tab Overview 界面，可以在该界面选择 tab 或者创建新 tab。

![](https://cdn.nlark.com/yuque/0/2021/jpeg/12376889/1625504452520-03aebeb7-61c2-491f-8982-802e8012f2a5.jpeg)

但是，也许你希望你的 Mac Catalyst App 支持多窗口，但不想拥有自由组合窗口的功能，这在之前是没办法做到的。Apple 在 macOS Monterey 中做了支持。你可以通过在 Info.plist 中的 UIApplicationSceneManifest 中添加值为 false 的 [UIApplicationSupportsTabbedSceneCollection](https://developer.apple.com/documentation/bundleresources/information_property_list/uiapplicationscenemanifest/uiapplicationsupportstabbedscenecollection/) 键来禁用这个功能。

![](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624807024581-a4510319-438e-438b-8119-3544700f8302.png#clientId=u8e681558-0371-4&from=paste&height=482&id=zFFAw&margin=%5Bobject%20Object%5D&name=image.png&originHeight=482&originWidth=1610&originalType=binary&ratio=1&size=177809&status=done&style=shadow&taskId=uf96f6ee9-b60d-4c71-86ec-3469f72dd89&width=1610)

### 光标的更新

一、Apple 现在支持 [UIPointerLockState](https://developer.apple.com/documentation/uikit/uipointerlockstate/) 来隐藏并锁定光标，这样它就不会移动到 Mac App 的窗口之外。这对于游戏 App 来说非常有用，它能防止用户不小心在游戏 App 窗口外点击并将另一个 App 带到前台。当你切换 App 或窗口时，它会暂时解锁窗口；当你点击窗口时，它会重新锁定窗口。

你可以在[【WWDC20 10094】Handle trackpad and mouse input](https://developer.apple.com/videos/play/wwdc2020/10094/) 中了解更多相关内容。

二、[UIPointerShape](https://developer.apple.com/documentation/uikit/uipointershape/) 现在支持水平或垂直的 iBeam 光标样式。

```swift
UIPointerShape.verticalBeam(length: 0) // API 应该是做了修改
UIPointerShape.horizontalBeam(length: 0)
```

![](https://cdn.nlark.com/yuque/0/2021/jpeg/12376889/1624748369893-eeadb68f-8e7a-428d-b089-adddc64979d1.jpeg#clientId=u8e681558-0371-4&from=paste&height=339&id=mck0g&margin=%5Bobject%20Object%5D&name=20210624002801.jpg&originHeight=339&originWidth=1414&originalType=binary&ratio=1&size=55956&status=done&style=shadow&taskId=u24eee124-5f7d-435f-adb5-c6f9fd20c81&width=1414)


三、Apple 现在支持 [hidden](https://developer.apple.com/documentation/uikit/uipointerstyle/3539025-hidden) 的光标样式，可以让我们在必要时在 App 中隐藏光标。

```swift
UIPointerStyle.hidden
```
