## Mac Catalyst 是什么

<br />Apple 在 WWDC19 上宣布了 [Mac Catalyst](https://developer.apple.com/documentation/uikit/mac_catalyst?language=objc) 技术，它可以让我们为现有的 iPad App 创建 Mac 版本，并且可以充分利用 Mac 的大显示屏、键盘、鼠标或触控板，同时我们只需要维护一份工程代码。在 Xcode 11 及更高版本中，我们只需要勾选一个复选框即可 [开启 Mac Catalyst 技术](https://developer.apple.com/cn/documentation/xcode/creating_a_mac_version_of_your_ipad_app/)。如果你的 App 只有 iPad 版本而没有 Mac 版本，而且主要功能不是 macOS 不支持的（比如 ARKit），或者你不想继续维护你的 Mac App，仅想通过一份工程代码构建 iPad 和 Mac App，那么 Mac Catalyst 非常适合你，你可以用极少的成本来获得一个 Mac App。<br />​

Mac Catalyst 是将 iPad App 带到 Mac 上的一种方式，它既支持 Intel Mac 也支持 ARM Mac（M1 Mac）。还有一种方式是 [iPhone and iPad Apps on Apple Silicon Macs](https://developer.apple.com/macos/iphone-and-ipad-apps/)，它只支持 ARM Mac，你可以在[《WWDC20 内参》- WWDC20 10114 - Apple Silicon Mac 上的 iPad 与 iPhone App 运行](https://xiaozhuanlan.com/topic/1845067932) 中了解更多。<br />​

下面我将开始讲解今年 WWDC 中 Mac Catalyst 的更新。如果你还没有接触过 Mac Catalyst 但对此感兴趣的话，可以在文末获取相关学习资料。<br />

## Mac Catalyst 有哪些更新


- 更多的按钮样式
- ToolTips
- 新增 “Print” 和 “Export as PDF” 菜单项
- 支持为窗口添加副标题
- 支持快捷指令
- 支持在 Mac Idiom 下保持自定义的 button 和 slider 样式
- 支持禁用自由组合窗口功能
- 光标的更新

​<br />
### 更多的按钮样式

<br />我们先来看一个 Mac Catalyst 的配置选项。当你勾选 Mac Catalyst 复选框时，你可以 [选择以 iPad 或是以 Mac 风格的用户界面来构建 Mac App](https://developer.apple.com/documentation/uikit/mac_catalyst/choosing_a_user_interface_idiom_for_your_mac_app?language=objc)。<br />![image.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624375042511-11599b06-4743-4b64-b6bb-283bc3295078.png#clientId=u279b7986-353f-4&from=paste&height=347&id=fKQX8&margin=%5Bobject%20Object%5D&name=image.png&originHeight=347&originWidth=926&originalType=binary&ratio=1&size=50468&status=done&style=shadow&taskId=u28dbd4d5-69ae-45d7-b1ab-3c257742cd8&width=926)

- Scale Interface to Match iPad（iPad Idiom）。当你勾选 Mac Catalyst 复选框时，默认会选择该方案，它将保持 iOS 原生的 UI，仅通过对你的 iPad App 进行缩放来构建 Mac Catalyst App，这样你就不需要在界面适配上下太大功夫，可以快速将 iPad App 带到 Mac 上。但视图和文本在 macOS 上会被缩放为原来的 77%，这样会导致视图和文本会有些模糊。
- Optimize Interface for Mac（Mac Idiom）。这是 Apple 在 WWDC20 上发布的新方案，UI 将会被自动转换为 macOS 原生的 UI，视图和文字大小将 100% 呈现而不是缩放，让你的 App 更像一个 masOS App 从而带来更好的体验。当然付出的代价就是需要花费更多时间和精力去做适配。


<br />笔者前段时间负责了《宝宝巴士》App 的 Mac Catalyst 版构建，已上架 App Store。考虑到成本和收益，我们为节约成本（包括开发和设计等成本）选择了 Scale Interface to Match iPad 方案，并且我们禁止了窗口自由缩放功能，同时将 App 窗口大小固定按 iPad 9.7 寸的比例进行缩放，以极小的成本完成了 UI 的适配。当然将来如果要做 Mac 版优化的话，Optimize Interface for Mac 是首选方案。你可以根据自己的需求来选择其中一种方案。<br />![宝宝巴士.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624802078872-67e3ffab-e8c8-49a9-8a0f-70f94cadc7f9.png#clientId=u8e681558-0371-4&from=paste&height=1965&id=u76bc6f4b&margin=%5Bobject%20Object%5D&name=%E5%AE%9D%E5%AE%9D%E5%B7%B4%E5%A3%AB.png&originHeight=1965&originWidth=4247&originalType=binary&ratio=1&size=5778646&status=done&style=shadow&taskId=u8a70144e-c865-4fe9-b76e-23e1a2efffd&width=4247)<br />好的，接下来我们正式进入 Mac Catalyst 今年带来的第一个更新 —— 更多的按钮样式。<br />​

在 macOS Big Sur（iOS 14）中，Apple 给 UIControl 添加了 [showsMenuAsPrimaryAction](https://developer.apple.com/documentation/uikit/uicontrol/3601223-showsmenuasprimaryaction/) 属性，这允许你将 Pull-down menu 附加到 UIButton 上。
> 由于 Mac Catalyst 是 UIKit for Mac，所以其 macOS 版本与 iOS 版本有着对应关系。

现在，在 macOS Monterey（iOS 15）中，Apple 给 UIButton 添加了一个新属性 [changesSelectionAsPrimaryAction](https://developer.apple.com/documentation/uikit/uibutton/3752184-changesselectionasprimaryaction/)，该属性使按钮的标题可以跟随 menu 中的选择而改变。<br />​

有了这两个属性，现在我们有四种方法可以配置按钮。<br />![截屏2021-06-20 下午11.03.15.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624201509540-dd469e51-35c6-4794-8c9b-a67989ba9b3b.png#clientId=u23cdbea8-a944-4&from=paste&height=733&id=u15f797d5&margin=%5Bobject%20Object%5D&name=%E6%88%AA%E5%B1%8F2021-06-20%20%E4%B8%8B%E5%8D%8811.03.15.png&originHeight=733&originWidth=1298&originalType=binary&ratio=1&size=119494&status=done&style=shadow&taskId=ub790dfbe-3d36-4a52-acb8-2b81dd2e811&width=1298)

1. Push Button。当这两个属性都设置为 false 时，也就是默认情况下，我们将得到一个标准按钮。但如果在单击或长按按钮时设置了 menu，那么仍可以显示该 menu。
```swift
let button = UIButton(type: .system)
```
![Push Button.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624379986054-eca0057a-1f55-4c86-b614-f13c66b1a8ff.png#clientId=u279b7986-353f-4&from=paste&height=360&id=u07486c02&margin=%5Bobject%20Object%5D&name=Push%20Button.png&originHeight=360&originWidth=2048&originalType=binary&ratio=1&size=78009&status=done&style=shadow&taskId=u8f7d806f-82af-46ca-bcce-85556cb2e9f&width=2048)<br />可以看到，在 iPad Idiom 下，Mac App 中的 button 被缩小至了 77% 大小，字体会显得有些模糊。而在 Mac Idiom 下，button 被自动转换为了 macOS 原生的 UI。<br />​<br />

2. Toggle Button。如果仅 changesSelectionAsPrimaryAction 设置为 true，那么会得到一个 Toggle Button，单击该按钮将改变 selected 的值并切换呈现打开或关闭状态。经测试，Toggle Button 仅在 Mac Idiom 下有效，在 iPad Idiom 下其等同于 Push Button。
```swift
let button = UIButton(type: .system)
button.changesSelectionAsPrimaryAction = true
```
![Toggle Button.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624384742892-c7a7c9ea-935c-463b-8a83-c341f842bf2e.png#clientId=u279b7986-353f-4&from=paste&height=360&id=u9b8b9b6a&margin=%5Bobject%20Object%5D&name=Toggle%20Button.png&originHeight=360&originWidth=2048&originalType=binary&ratio=1&size=93588&status=done&style=shadow&taskId=u0ce8e70c-ca77-4904-b167-6363f9bc2d8&width=2048)<br />如果我们在 Toggle Button 的基础上设置了 menu，但 showsMenuAsPrimaryAction 值为 false，那么当我们长按按钮时将显示该 menu。<br />​<br />

3. Pull-down menu Button。如果仅 showsMenuAsPrimaryAction 设置为 true，我们将得到一个附带 Pull-down menu 的按钮，在点击按钮时出现 menu。
```swift
let button = UIButton(type: .system)
button.menu = UIMenu()
button.showsMenuAsPrimaryAction = true
```
![Pull Down.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624381296832-913afa03-f097-4c62-be3d-701d5bfbc8a2.png#clientId=u279b7986-353f-4&from=paste&height=840&id=uc6d21759&margin=%5Bobject%20Object%5D&name=Pull%20Down.png&originHeight=840&originWidth=2648&originalType=binary&ratio=1&size=257631&status=done&style=shadow&taskId=u0f3c6a03-f8d5-4da6-bbdf-d0a06db580f&width=2648)

4. Pop-up menu Button。如果这两个属性都设置为 true，我们将得到一个附带 Pop-up menu 的按钮，在点击按钮时出现 menu，且按钮的标题可以跟随 menu 中的选择而改变。虽然我们使用 Pull-down Button 也可以实现按钮标题跟随改变的效果，但 Pop-up Button 使之更方便，而且在按钮样式和 menu 样式上也存在差异。
```swift
let button = UIButton(type: .system)
button.menu = UIMenu()
button.showsMenuAsPrimaryAction = true
button.changesSelectionAsPrimaryAction = true
```
![Pop Up.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624381928472-a75eb7d5-2ee1-477c-85b1-77504bb7edc4.png#clientId=u279b7986-353f-4&from=paste&height=840&id=u96cc36ba&margin=%5Bobject%20Object%5D&name=Pop%20Up.png&originHeight=840&originWidth=3448&originalType=binary&ratio=1&size=316116&status=done&style=shadow&taskId=ue6ed4032-6405-4938-a921-f24e1489784&width=3448)<br />现在，对于设置了 showsMenuAsPrimaryAction 为 true 的按钮，在 macOS 上通过鼠标左键点击按钮时，将立即显示 menu。对于未设置 showsMenuAsPrimaryAction 为 true 但设置了 menu 的按钮，其在 macOS 上的行为取决于 App 的 idiom：

- 在 iPad Idiom 中，通过鼠标右键点击按钮来显示 menu。
- 在 Mac Idiom 中，通过长按按钮来显示 menu。


<br />你可以在 “[Meet the UIKit button system](https://developer.apple.com/videos/play/wwdc2021/10064/)” 视频中了解更多有关 UIButton 的更新，例如配合 iOS 15 的 [UIButtonConfiguration](https://developer.apple.com/documentation/uikit/uibuttonconfiguration/) API 配置更多样的按钮。

### ToolTips

<br />在 macOS Monterey（iOS 15）中，Apple 为 Mac Catalyst App 提供了 ToolTips。ToolTips 是一个小的浮动窗口，当光标悬停在 App 中的某个视图上一小段时间后会出现，它可以用来解释这个视图的用途或其他细节。<br />​

我们可以创建一个带有文本的 [UIToolTipInteraction](https://developer.apple.com/documentation/uikit/uitooltipinteraction/) 实例，然后将其添加到视图上。
```swift
let toolTipInteraction = UIToolTipInteraction(defaultToolTip: string)
view.addInteraction(tooltipInteraction)
```

```swift
control.toolTip = "Enable updates"
```
![截屏2021-06-23 上午1.24.00.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624382714917-ef0bff69-d1d5-4233-88e9-9eab58ec219e.png#clientId=u279b7986-353f-4&from=paste&height=143&id=u20a4e901&margin=%5Bobject%20Object%5D&name=%E6%88%AA%E5%B1%8F2021-06-23%20%E4%B8%8A%E5%8D%881.24.00.png&originHeight=143&originWidth=486&originalType=binary&ratio=1&size=6948&status=done&style=shadow&taskId=u4a99edb6-3b94-420b-9da1-f56a7e90eaa&width=486)<br />在 iOS 中当 UILabel 的文本过长而无法容纳时，超出的文本会被截断。Apple 给 UILabel 添加了 [showsExpansionTextWhenTruncated](https://developer.apple.com/documentation/uikit/uilabel/3750862-showsexpansiontextwhentruncated/) 属性，来添加 UILabel 对 ToolTips 的支持，当我们将鼠标悬停在 label 上时，它将显示完整的文本。
```swift
label.showsExpansionTextWhenTruncated = true
```

### 新增 “Print” 和 “Export as PDF” 菜单项

<br />在 macOS Monterey 中，Mac Catalyst App 的打印功能将变得更方便。<br />​

Apple 在 iOS 15 中新增了一个名为 [UIApplicationSupportsPrintCommand](https://developer.apple.com/documentation/bundleresources/information_property_list/uiapplicationsupportsprintcommand/) 的 Info.plist 的 key。<br />![WX20210626-225629.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624719422356-81a1a3af-1f55-4bc8-9f48-6f9f335142a7.png#clientId=u8e681558-0371-4&from=paste&height=1040&id=XrPP6&margin=%5Bobject%20Object%5D&name=WX20210626-225629.png&originHeight=1040&originWidth=1849&originalType=binary&ratio=1&size=330606&status=done&style=shadow&taskId=uada7b7d4-3e7f-4b6f-a565-9eb02886047&width=1849)如果我们将其添加到 Info.plist 中并设置为 true，那么当我们启动 Mac Catalyst App 时将会自动拥有 “Print” 和 “Export as PDF” 两个菜单项（归类在 “File” 菜单下）。我们可以使用快捷键 Command-P（⌘P）来使用 “Print” 功能。<br />![image.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624720125654-6dd86761-3b0e-4ac7-aef9-18a5a78e0c78.png#clientId=u8e681558-0371-4&from=paste&height=1040&id=ubcc61a62&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1040&originWidth=1849&originalType=binary&ratio=1&size=2451493&status=done&style=shadow&taskId=u355b0318-9c10-4d2d-afec-550900e60b1&width=1849)<br />我们也可以将该 plist key 添加到 iOS App 中，它会在 [iPadOS 键盘快捷键]() 上显示 “Print” 选项，其快捷键也为 ⌘P。当该 iOS App 在 M1 Mac 上运行时，将自动拥有 “Print” 和 “Export as PDF” 两个菜单项，同 Mac Catalyst App。<br />![image.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624719923101-7b2fa930-219f-47fc-80a5-221da001d72e.png#clientId=u8e681558-0371-4&from=paste&height=806&id=u05de0e59&margin=%5Bobject%20Object%5D&name=image.png&originHeight=806&originWidth=1005&originalType=binary&ratio=1&size=45966&status=done&style=shadow&taskId=ucf1a55b2-f0f7-47c0-9419-9a3c644125f&width=1005)<br />虽然这个 plist 键告诉操作系统我们需要 “Print” 和 “Export as PDF” 菜单项，但是我们还需要在代码中实现 “Print” 和 “Export as PDF” 功能。否则这个两个菜单项呈灰显状态且不可点击。<br />![截屏2021-06-28 下午10.34.28.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624890908365-f476aa80-3e0a-4bb4-8759-b62fbb4af0e3.png#clientId=u8e681558-0371-4&from=paste&height=147&id=ub8219c11&margin=%5Bobject%20Object%5D&name=%E6%88%AA%E5%B1%8F2021-06-28%20%E4%B8%8B%E5%8D%8810.34.28.png&originHeight=147&originWidth=732&originalType=binary&ratio=1&size=120690&status=done&style=shadow&taskId=uc87ea945-29c7-4780-a0df-55f627ee0f8&width=732)<br />为此，Apple 给 UIResponder 添加了一个 [printContent](https://developer.apple.com/documentation/uikit/uiresponderstandardeditactions/3750910-printcontent/) 方法，我们可以在任何一个 UIResponder 里实现这个方法来处理打印功能。当我们点击 “Print” 或 “Export as PDF” 菜单项时，UIKit 会在响应者链中查找可以执行此操作的对象来处理打印。比如，我们在一个 viewController 中实现了 printContent 方法，当 App 焦点和第一响应者状态转移到这个 viewController 时，点击 “Print” 菜单项或者使用快捷键 ⌘P，该 viewController 的 printContent 方法就会被调用，我们只需在该方法中像平常一样设置和使用 [UIPrintInteractionController](https://developer.apple.com/documentation/uikit/uiprintinteractioncontroller/) 就行。
```swift
override func printContent(_ sender: Any?) {
    let printInteractionController = UIPrintInteractionController.shared
    ...
}
```
![image.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624731391433-f741be09-c146-44e6-8d5d-8eb7fcfa83de.png#clientId=u8e681558-0371-4&from=paste&height=1040&id=ufe740971&margin=%5Bobject%20Object%5D&name=image.png&originHeight=1040&originWidth=1850&originalType=binary&ratio=1&size=1787009&status=done&style=shadow&taskId=u9ba45c95-4de5-4e4d-90af-c2920ff9b85&width=1850)<br />在某些情况下，当前的响应者可能不想执行打印，需要由业务逻辑决定。这时候我们可以重写 [canPerformAction](https://developer.apple.com/documentation/uikit/uiresponder/1621105-canperformaction/) 方法来自定义逻辑。canPerformAction 方法会在 printContent 方法之前被调用，如果返回 false，当前的响应者将被跳过，UIKit 会在响应者链中查找下一个可以执行打印操作的响应者。
```swift
override func canPerformAction(_ action: Selector, withSender sender: Any?) -> Bool {
    if action == #selector(self.printContent(_:)) {
        ...
    } else {
        return super.canPerformAction(action, withSender: sender)
    }
}
```
我们可以在接近响应者链顶部的响应者（比如 rootViewController）中实现 [targetForAction:withSender:](https://developer.apple.com/documentation/uikit/uiresponder/1621146-targetforaction/) 方法来处理打印操作，这在我们想要打印某些页面集，但实现打印的对象又不在响应者链中时非常有用。这种处理方式比我们通过给需要打印的对象发送 [becomeFirstResponder](https://developer.apple.com/documentation/uikit/uiresponder/1621113-becomefirstresponder/) 消息来强制该对象进入响应者链的方式要好得多，而且在这种情况下使用 becomeFirstResponder 将是一种反模式。
```swift
override func target(forAction action: Selector, withSender sender: Any?) -> Any? {
    switch action {
    case #selector(UIResponder.printContent(_:)):
        ...
    default:
        return super.target(forAction: action, withSender: sender)
    }
}
```

### 支持在 Mac Idiom 下保持自定义的 button 和 slider 样式

<br />Apple 不建议我们使用 iPad Idiom 来构建 Mac Catalyst App，而是建议我们使用 macOS 原生的 UI。但有些情况下，可能我们的某些 button 或 slider 需要适配 App 的风格主题，因此想要它们保持 iPad Idiom，于是 Apple 今年做了支持。<br />​

UIButton 和 UISlider 各有两个新属性 [behavioralStyle](https://developer.apple.com/documentation/uikit/uibutton/3752180-behavioralstyle/) 和 [preferredBehavioralStyle](https://developer.apple.com/documentation/uikit/uibutton/3752181-preferredbehavioralstyle/)。我们可以通过给 button 和 slider 设置 preferredBehavioralStyle 值来决定其采用 iPad Idiom 还是 Mac Idiom，并且它不会影响到我们的 iOS App，这样我们就可以在不影响 iOS App 的情况下适配 Mac Catalyst App。<br />![image.png](https://cdn.nlark.com/yuque/0/2021/png/12376889/1624807084576-56433511-47bd-4152-bc25-fdfd5e9c72c8.png#clientId=u8e681558-0371-4&from=paste&height=309&id=ub73d01ef&margin=%5Bobject%20Object%5D&name=image.png&originHeight=309&originWidth=1612&originalType=binary&ratio=1&size=105249&status=done&style=none&taskId=u6d2abb61-eb0a-41a6-b521-0f21766056d&width=1612)
```swift
API_AVAILABLE(ios(15.0), macCatalyst(15.0)) API_UNAVAILABLE(watchos, tvos)
typedef NS_ENUM(NSUInteger, UIBehavioralStyle) {
    UIBehavioralStyleAutomatic = 0,
    UIBehavioralStylePad,
    UIBehavioralStyleMac,
};

API_AVAILABLE(ios(15.0), macCatalyst(15.0)) API_UNAVAILABLE(watchos, tvos)
@interface UIButton (UIBehavioralStyle)

@property (nonatomic, readonly) UIBehavioralStyle behavioralStyle API_AVAILABLE(ios(15.0), macCatalyst(15.0)) API_UNAVAILABLE(watchos, tvos);
@property (nonatomic) UIBehavioralStyle preferredBehavioralStyle API_AVAILABLE(ios(15.0), macCatalyst(15.0)) API_UNAVAILABLE(watchos, tvos);

@end
```

### 光标的更新

<br />Apple 现在支持 [UIPointerLockState](https://developer.apple.com/documentation/uikit/uipointerlockstate/) 来隐藏并锁定光标，这样它就不会移动到 Mac 窗口之外。这对于游戏 App 来说非常有用，它能防止用户不小心在游戏 App 窗口外点击并将另一个 App 带到前台。当你切换 App 或窗口时，它会暂时解锁窗口，当你点击窗口时，它会重新锁定窗口。<br />​

Apple 现在支持了可以用来获取水平或垂直的 iBeam 样式光标的 [UIPointerShape](https://developer.apple.com/documentation/uikit/uipointershape/)。
```swift
UIPointerShape.verticalBeam(length: 0) // API 应该是做了修改
UIPointerShape.horizontalBeam(length: 0)
```
