# 认识 Swift 中的异步与并发

## 认识 Swift 中的 async / await

异步编程在 iOS 开发里是一个常见的操作，例如我们经常需要在网络请求回来之后更新数据模型和视图。但是当异步操作嵌套时，不出错实在是太难了。

### `completion` 回调

我们用获取缩略图来举个例子，十秒钟浏览以下代码，这段代码有问题吗？如果有，有几处呢？

```swift
func fetchThumbnail(for id: String, completion: @escaping (UIImage?, Error?) -> Void) {
    let request = thumbnailURLRequest(for: id)
    let task = URLSession.shared.dataTask(with: request) { data, response, error in
        if let error = error {
            completion(nil, error)
        } else if (response as? HTTPURLResponse)?.statusCode != 200 {
            completion(nil, FetchError.badID)
        } else {
            guard let image = UIImage(data: data!) else {
                return
            }
            image.prepareThumbnail(of: CGSize(width: 40, height: 40)) { thumbnail in
                guard let thumbnail = thumbnail else {
                    return
                }
                completion(thumbnail, nil)
            }
        }
    }
    task.resume()
}
```

十、九、八……三、二、一，好了，时间到，揭晓答案。以上代码有两处错误，都是在 `guard` 语句之后直接退出，没有调用 `completion`，这样外部调用就无法处理所有可能的情况。而且不调用 `completion` 是一个合法（但不符合预期）的行为，编译器不会产生错误。

通常我们在请求网络数据时，都会使用菊花转圈作为 UI 等待提示，假如无法从 `Data` 转换到 `UIImage`，又或者无法生成缩略图，由于退出异步函数之前没有调用 `completion`，调用方无法知晓函数已经结束，菊花转圈会一直存在无法关闭。你可能会想到引入超时时间来关闭菊花转圈，但是上面的代码有两处异步调用，网络请求的超时时间容易计算，获取 `timeoutIntervalForRequest` 即可，但是生成缩略图的超时时间可不好计算，需要根据多次实验结果来估算合理范围。即使超时时间机制有效，这样也会引入额外的逻辑来处理，使问题变得更加复杂。

### 正确的 `completion` 回调

把遗留的两个 `completion` 补全后，代码如下。

```swift
func fetchThumbnail(for id: String, completion: @escaping (UIImage?, Error?) -> Void) {
    let request = thumbnailURLRequest(for: id)
    let task = URLSession.shared.dataTask(with: request) { data, response, error in
        if let error = error {
            completion(nil, error)
        } else if (response as? HTTPURLResponse)?.statusCode != 200 {
            completion(nil, FetchError.badID)
        } else {
            guard let image = UIImage(data: data!) else {
                completion(nil, FetchError.badImage)
                return
            }
            image.prepareThumbnail(of: CGSize(width: 40, height: 40)) { thumbnail in
                guard let thumbnail = thumbnail else {
                    completion(nil, FetchError.badImage)
                    return
                }
                completion(thumbnail, nil)
            }
        }
    }
    task.resume()
}
```

一个简单的获取缩略图的函数，多达五个 `completion` 来通知调用方需要处理的情况，并且调用方需要依次判断 `UIImage?` 和 `Error?` 的组合情况，高达 4 种，而实际上有效的只有 2 种。

### 使用 `Result` 重构 `completion` 回调

我们可以使用 Swift 标准库里面的 `Result` 来解决这个问题，代码如下。

```swift
func fetchThumbnail(for id: String, completion: @escaping (Result<UIImage, Error>) -> Void) {
    let request = thumbnailURLRequest(for: id)
    let task = URLSession.shared.dataTask(with: request) { data, response, error in
        if let error = error {
            completion(.failure(error))
        } else if (response as? HTTPURLResponse)?.statusCode != 200 {
            completion(.failure(FetchError.badID))
        } else {
            guard let image = UIImage(data: data!) else {
                completion(.failure(FetchError.badImage))
                return
            }
            image.prepareThumbnail(of: CGSize(width: 40, height: 40)) { thumbnail in
                guard let thumbnail = thumbnail else {
                    completion(.failure(FetchError.badImage))
                    return
                }
                completion(.success(thumbnail))
            }
        }
    }
    task.resume()
}
```

只要调用方可以知道结果，那么结果一定只有 2 种，要么成功，要么失败，不需要判断其他情况，更加符合使用习惯，但是这样仍然无法避免 `completion` 被忽略调用的情况。

### 使用 `async` 和 `await` 重构函数

我们使用 Swift 5.5 新增的 `async` 和 `await` 来解决这个问题，代码如下。

```swift
func fetchThumbnail(for id: String) async throws -> UIImage {
    let request = thumbnailURLRequest(for: id)
    let (data, response) = try await URLSession.shared.data(for: request)
    guard (response as? HTTPURLResponse)?.statusCode == 200 else { throw FetchError.badID }
    let maybeImage = UIImage(data: data)
    guard let thumbnail = await maybeImage?.thumbnail else { throw FetchError.badImage }
    return thumbnail
}
```

步骤非常简单：

1. 生成 `request` 请求，这是一个同步操作；
2. 请求数据，`try` 标记可能抛出错误，`await` 标记可能暂停，这是一个异步操作；
3. 判断返回状态码是否 200，不是则抛出错误；
4. 生成原始图片，这是一个同步操作；
5. 生成缩略图，`await` 标记可能暂停，无法生成则抛出错误，这是一个异步操作；
6. 返回成功生成的缩略图。

相较于前面 `completion` 版本的 23 行，这里 `async` 版本仅需要 8 行，没有函数的层层嵌套，都是一步接一步线性的，易读性大大提升，同时编译器也可以检查错误。

### 它们是如何工作的

![wwdc2021-10132.hd-0002.png](wwdc2021-10132.hd-0002.png)

上图是一个普通函数调用，当 `thumbnailURLRequest` 完成之后，返回 `fetchThumbnail` 继续执行后续任务，如果这个函数是一个耗时任务，那么当前线程就会持续等待，直到完成。

![wwdc2021-10132.hd-0003.png](wwdc2021-10132.hd-0003.png)

上图是一个异步函数调用，当 `data(for:)` 调用后，函数被挂起，当任务完成后，系统通知 `data(for:)` 继续执行后续任务。

![wwdc2021-10132.hd-0004.png](wwdc2021-10132.hd-0004.png)

`async` / `await` 实际上的意思：

1. `async` 允许一个函数被挂起；
2. `await` 标记一个异步函数可能暂停执行；
3. 在挂起期间可能会进行其他工作；
4. 一旦等待的异步调用完成，在 `await` 之后恢复执行。

## 探索 Swift 中的结构化并发

### `completion` 回调处理多个缩略图

上个章节我们介绍了如何获取单张图片的缩略图，打怪升级，这次我们来处理获取多张缩略图。

```swift
func fetchThumbnails(
    for ids: [String],
    completion handler: @escaping ([String: UIImage]?, Error?) -> Void
) {
    guard let id = ids.first else { return handler([:], nil) }
    let request = thumbnailURLRequest(for: id)
    let dataTask = URLSession.shared.dataTask(with: request) { data, response, error in
        guard let response = response,
              let data = data
        else {
            return handler(nil, error)
        }
        // ... check response ...
        UIImage(data: data)?.prepareThumbnail(of: thumbSize) { image in
            guard let image = image else {
                return handler(nil, ThumbnailFailedError())
            }
            fetchThumbnails(for: Array(ids.dropFirst())) { thumbnails, error in
                // ... add image to thumbnails ...
            }
        }
    }
    dataTask.resume()
}
```

这代码，是不是看着就血压高了？递归调用依次获取缩略图，并且代码里还没给出如何合并返回结果。

### 使用 `async` 和 `await` 处理多个缩略图

我们使用 Swift 5.5 新增的 `async` 和 `await` 进行重构，代码如下。

```swift
func fetchThumbnails(for ids: [String]) async throws -> [String: UIImage] {
    var thumbnails: [String: UIImage] = [:]
    for id in ids {
        let request = thumbnailURLRequest(for: id)
        let (data, response) = try await URLSession.shared.data(for: request)
        try validateResponse(response)
        guard let image = await UIImage(data: data)?.byPreparingThumbnail(ofSize: thumbSize) else {
            throw ThumbnailFailedError()
        }
        thumbnails[id] = image
    }
    return thumbnails
}
```

1. 声明 `thumbnails` 保存返回值；
2. 使用 `for-in` 循环创建任务；
   1. 创建请求；
   2. 下载图片；
   3. 检查资源合法性；
   4. 生成缩略图；
   5. 保存缩略图。
3. 返回 `thumbnails`。

有没有发现，在 `for-in` 循环中执行的代码，和上一个章节中单个缩略图的处理逻辑是一样的！这里仅仅是增加了一个循环，就支持了多个任务的处理。

### 使用 `async-let` 处理结构化并发

上面例子的 `thumbSize` 是本地提供的，处理起来比较方便。假设 `thumbSize` 也要从网络接口获取，那么在生成缩略图的过程中会有两个网络请求，代码如下。

```swift
func fetchOneThumbnail(withID id: String) async throws -> UIImage {
    let imageReq = imageRequest(for: id), metadataReq = metadataRequest(for: id)
    let (data, _) = try await URLSession.shared.data(for: imageReq)
    let (metadata, _) = try await URLSession.shared.data(for: metadataReq)
    guard let size = parseSize(from: metadata),
          let image = await UIImage(data: data)?.byPreparingThumbnail(ofSize: size)
    else {
        throw ThumbnailFailedError()
    }
    return image
}
```

可以看到，在 `imageReq` 请求完成之后，才发起 `metadataReq` 请求，只有两个请求都完成之后，才能进行之后的任务。如果在两个请求任务和使用他们的任务之间有其他的无关任务，无关任务的执行必须等到两个请求任务完成之后，这无疑是一种浪费。

我们可以用 `async-let` 来解决这个问题，代码如下。

```swift
func fetchOneThumbnail(withID id: String) async throws -> UIImage {
    let imageReq = imageRequest(for: id), metadataReq = metadataRequest(for: id)
    async let (data, _) = URLSession.shared.data(for: imageReq)
    async let (metadata, _) = URLSession.shared.data(for: metadataReq)
    guard let size = parseSize(from: try await metadata),
          let image = try await UIImage(data: data)?.byPreparingThumbnail(ofSize: size)
    else {
        throw ThumbnailFailedError()
    }
    return image
}
```

使用 `async let` 标记 `data` 和 `metadata`，使用 `try await` 进行访问。系统会在执行网络请求并发任务的同时，执行后续代码，即上文提到的无关任务。当进行到访问网络请求的结果的时候，系统会进行等待直到完成或者抛出错误。这样便能更高效地完成整个任务。

### 调用异步函数处理多个缩略图

既然 `for-in` 内部的逻辑就是处理单个缩略图的逻辑，自然而然我们就会想到直接调用封装好的异步函数，代码如下。

```swift
func fetchThumbnails(for ids: [String]) async throws -> [String: UIImage] {
    var thumbnails: [String: UIImage] = [:]
    for id in ids {
        try Task.checkCancellation()
        thumbnails[id] = try await fetchOneThumbnail(withID: id)
    }
    return thumbnails
}
```

你应该发现了，在获取缩略图之前，调用了 `checkCancellation` 来进行任务检查。我们把 `fetchThumbnails` 称为父任务，把 `fetchOneThumbnail` 称为子任务。

如果希望任务取消后返回之前已经处理的缩略图，可以使用 `isCancelled` 进行判断，代码如下。

```swift
func fetchThumbnails(for ids: [String]) async throws -> [String: UIImage] {
    var thumbnails: [String: UIImage] = [:]
    for id in ids {
        if Task.isCancelled { break }
        thumbnails[id] = try await fetchOneThumbnail(withID: id)
    }
    return thumbnails
}
```

### 使用任务组进行处理

上文提到的两个函数，在获取缩略图的时候，只能一个一个地获取，如果我们想要同时获取多个，就需要引入任务组进行处理，代码如下。

```swift
func fetchThumbnails(for ids: [String]) async throws -> [String: UIImage] {
    var thumbnails: [String: UIImage] = [:]
    try await withThrowingTaskGroup(of: (String, UIImage).self) { group in
        for id in ids {
            group.async {
                return (id, try await fetchOneThumbnail(withID: id))
            }
        }
        // Obtain results from the child tasks, sequentially, in order of completion.
        for try await (id, thumbnail) in group {
            thumbnails[id] = thumbnail
        }
    }
    return thumbnails
}
```

1. 声明 `thumbnails` 保存返回值；
2. 调用 `withThrowingTaskGroup` 创建任务组；
   1. 使用 `for-in` 循环依次把任务放到任务组，任务被添加时会立刻执行，到达最大并发数量后进行等待；
   2. 使用 `for-await-in` 循环依次取回已处理的缩略图。
3. 返回 `thumbnails`。

### 非结构化任务

在实际编码中，我们遇到的并不总是结构化任务，相当一部分是非结构化任务。

![wwdc2021-10134.hd-0012.png](wwdc2021-10134.hd-0012.png)

1. 一些任务需要从非异步上下文启动；
2. 一些任务超出了单一范围的限制。

例如我们在将要展示 `UICollectionViewCell` 前获取缩略图并进行展示，获取是一个异步任务，而展示是一个同步任务，代码如下。

```swift
@MainActor
class MyDelegate: UICollectionViewDelegate {
    func collectionView(_ view: UICollectionView, willDisplay cell: UICollectionViewCell, forItemAt item: IndexPath) {
        let ids = getThumbnailIDs(for: item)
        let thumbnails = await fetchThumbnails(for: ids)
        display(thumbnails, in: cell)
    }
}
```

以上代码将会产生编译错误，因为异步函数在一个非异步上下文中启动，解决的方法也很简单，使用 `async()` 全局函数包住即可，代码如下。

```swift
@MainActor
class MyDelegate: UICollectionViewDelegate {
    func collectionView(_ view: UICollectionView, willDisplay cell: UICollectionViewCell, forItemAt item: IndexPath) {
        let ids = getThumbnailIDs(for: item)
        async {
            let thumbnails = await fetchThumbnails(for: ids)
            display(thumbnails, in: cell)
        }
    }
}
```

但是上面的代码还是有问题，当用户快速滑动时，将会产生多个不必要的任务，我们需要在适当的时机取消任务，代码如下。

```swift
@MainActor
class MyDelegate: UICollectionViewDelegate {
    var thumbnailTasks: [IndexPath: Task.Handle<Void, Never>] = [:]

    func collectionView(_ view: UICollectionView, willDisplay cell: UICollectionViewCell, forItemAt item: IndexPath) {
        let ids = getThumbnailIDs(for: item)
        thumbnailTasks[item] = async {
            defer { thumbnailTasks[item] = nil }
            let thumbnails = await fetchThumbnails(for: ids)
            display(thumbnails, in: cell)
        }
    }

    func collectionView(_ view: UICollectionView, didEndDisplay cell: UICollectionViewCell, forItemAt item: IndexPath) {
        thumbnailTasks[item]?.cancel()
    }
}
```

1. 记录 `async()` 函数的返回值；
2. 当任务完成时，移除记录；
3. 当任务不再需要时，取消任务。

### 分离的任务

![wwdc2021-10134.hd-0014.png](wwdc2021-10134.hd-0014.png)

1. 声明周期不定，手动取消并等待；
2. 不从它们的起始上下文继承任何东西；
3. 可选参数控制优先级和其他特征。

假如我们需要对上面获取的缩略图进行缓存，需要使用分离的任务，代码如下。

```swift
@MainActor
class MyDelegate: UICollectionViewDelegate {
    var thumbnailTasks: [IndexPath: Task.Handle<Void, Never>] = [:]

    func collectionView(_ view: UICollectionView, willDisplay cell: UICollectionViewCell, forItemAt item: IndexPath) {
        let ids = getThumbnailIDs(for: item)
        thumbnailTasks[item] = async {
            defer { thumbnailTasks[item] = nil }
            let thumbnails = await fetchThumbnails(for: ids)
            asyncDetached(priority: .background) {
                withTaskGroup(of: Void.self) { g in
                    g.async { writeToLocalCache(thumbnails) }
                    g.async { log(thumbnails) }
                    g.async { ... }
                }
            }
            display(thumbnails, in: cell)
        }
    }
}
```

1. 使用 `asyncDetached(priority: .background)` 产生一个后台的分离任务；
2. 任务中使用 `withTaskGroup(of:)` 创建任务组，进而创建多个子任务。
