# WWDC21 - #Discover Metal debugging, profiling, and asset creation tools/Metal 调试、分析和资源创建工具.md

## 一、概览
新版Xcode针对Metal调试增加了很多特性，此session将帮助你将 Metal 调试、分析和资产创建工作流程提升到一个新的水平。包含：光线追踪和 GPU 分析的最新工具，了解 Metal Debugger 工作流程，以及展示展示如何使用纹理转换器工具，该工具支持所有现代 GPU 纹理格式，它并且可以轻松集成到你的多平台资产创建管道中。

- Mac版本：11.4
- Xcode版本：Xcode13-beta
- iOS版本：iOS15-beta

主要从以下四个内容介绍：
1. Ray tracing support(光线追踪支持)
2. New profiling workflows(全新的分析工作流)
3. Debugging improvements(调试改进)
4. Advances in texture compression(纹理压缩的进展)

 ### GPU performance state
现在我已经向你展示了一种新的方式来分析你的应用程序，重要的是要了解它的性能取决于几个因素。当我们谈论 Metal 时，GPU 性能状态是一个非常重要的因素。

![009.png](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f80bf33328b6422da6e5b743b5f59b6e~tplv-k3u1fbpfcp-watermark.image)
它由操作系统管理，它会根据设备散热、系统设置、GPU 利用率和其他参数降低或提高状态。这些状态更改可能会影响你看到的分析结果。

今年，我们推出了新方法，让你可以通过更一致的结果来分析你的应用。我们添加了一些方法让你在我们的整套 Metal 工具中查看和更改 GPU 性能状态。

首先，利用Instruments 和 Metal system trace 用于实时性能监控，然后，利用Metal Debugger用于分析 GPU 跟踪，最后使用Xcode 中的 Device conditions用于一般用例。
 
![010.png](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b44b6b62ef34401c85e50f034e0fe53a~tplv-k3u1fbpfcp-watermark.image)
#### Instruments
今年，我们在 Metal 系统跟踪中添加了 GPU 性能状态跟踪，将它与其他轨道结合使用，将你的应用程序的性能与设备的性能状态相关联。但是请记住，能够看到性能状态只是等式的一部分。为了获得一致且可重现的分析结果，你还需要一种在设备上设置 GPU 性能状态的方法。

今年的新功能是在 Instruments 中记录跟踪时诱导特定 GPU 性能状态的能力。只需转到“录制选项”，然后在录制开始前选择一个演奏状态即可。之后，你可以照常记录性能跟踪，如果设备可以维持状态，Instruments将在跟踪期间诱导你选择的状态。有时，你可能需要检查现有的 Instruments 跟踪在记录期间是否具有 GPU 性能状态。你可以在信息弹出窗口的“录制设置”部分中找到此信息。

现在，你知道如何从 Instruments 查看和诱导 GPU 性能状态。利用一致的 GPU 性能状态的第二种方法是使用 Metal Debugger。默认情况下，当你捕获应用程序的 GPU 跟踪时，Xcode 将为你分析跟踪。它将使用设备在捕获时所处的相同性能状态来执行此操作。根据我们之前提到的因素，该状态可能会有所波动。相反，如果你想自己选择某个性能状态，请使用调试栏中的 Stopwatch 按钮。
#### Metal Debugger
做出选择后，Metal Debugger 将再次分析你的 GPU 跟踪。完成后，该按钮会突出显示以反映已达到一致的性能状态。此外，摘要页面上的“性能”部分现在可以一目了然地显示新的性能数据以及所选的性能状态。 这两种方法与 Metal 工具套件相关。
#### Device conditions
有时，你可能希望在分析工作流之外引入一致的性能状态。设置 GPU 性能状态的第三种方法是通过设备条件。
如果你想测试你的应用在不同 GPU 性能状态下的表现，这是你的最佳选择。在 Xcode 13 中，我们添加了 GPU 性能状态设备条件。它强制操作系统使用设备上的指定状态，只要它可以维持它并且它与 Xcode 保持连接。你可以从 Xcode 添加此条件，如果你转到窗口、设备和模拟器，在那里选择你的设备，然后滚动到“设备条件”部分，并添加具有所需级别的“GPU 性能状态”条件。当你想在设备上应用 GPU 性能状态更改时，请按开始。然后，完成后，按停止。

这些直接通过我们的工具查看和更改 GPU 性能状态的新方法应该可以帮助你分析和测试你的应用程序。而且我认为你会喜欢我们对分析工作流的最新添加和改进，我希望它们能帮助你使你的应用程序变得更好。

### Separate debug information
现在，如果你想使用这些功能，你有两个选择。第一个选项是在应用程序运行时从源代码编译你的库。第二个更好的选择是构建带有离线嵌入源的 Metallib 文件，然后在运行时加载它们。但是，App Store 规则不允许你使用这些调试 Metallib 发布你的应用程序。所有这一切意味着，如果你离线编译库并且希望能够调试着色器，则必须编译它们两次：一次嵌入源代码，用于开发过程中，一次没有源代码，用于分发。
 
![018.png](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b6eeee20896446bfb7084a077ce58e7e~tplv-k3u1fbpfcp-watermark.image)
今年，我们正在改变这种状况。你现在可以在编译 Metallib 时生成带有源代码和其他调试信息的单独文件。
 
这些文件具有 Metallibsym 扩展名，它们允许你调试和分析着色器，而无需在库本身中嵌入附加信息。将它们分开的最重要的好处是，现在你不需要拥有同一个 Metallib 的两个版本。
 
![019.png](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1ef04d9ade1345c7a305dc01da65a940~tplv-k3u1fbpfcp-watermark.image)
另一个好处是，使用这些 Metallibsym 文件，你现在甚至可以在应用程序的发布版本中调试着色器，而不必危及着色器源。
 
我将向你展示一个示例，说明如何将着色器源文件编译为带有 Metallibsym 文件的 Metallib。我将从 xcrun 终端命令开始，它可以正常编译 Metallib。
```
xcrun -sdk iphoneos metal Shaders.metal
```
要生成 Metallibsym 文件，我只需添加带有“flat”选项的“record-sources”标志，然后运行编译器。
```
xcrun -sdk iphoneos metal -frecord-source=flat Shaders.metal
```
现在，当我尝试调试使用单独的调试信息文件编译的着色器时，系统会提示我导入它。
 
![021.png](https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c1d34926a7234374b4863ab508c9f5de~tplv-k3u1fbpfcp-watermark.image)
单击“导入源”会打开一个对话框，其中列出了所有库以及它们是否导入了源文件。从这里，我可以导入任何 Metallibsym 文件，一旦导入，库及其源将自动匹配。完成导入后，我可以关闭对话框，现在我可以查看着色器的源代码并对其进行调试。

### Selective shader debugging
我想向你展示最后一项调试改进。这称为“选择性着色器调试”， 如果你的应用程序使用大型着色器，你可能已经注意到着色器调试可能需要一段时间才能启动。为了在这种情况下提供帮助，今年我们引入了选择性着色器调试。它可以帮助你缩小调试范围，从而更快地调试着色器。

让我们用其中一个这样的大型着色器来看看它的作用。我想调试这个 GPU ASTCDecoder。我知道如果我尝试调试整个内核，Shader Debugger 将需要很长时间才能启动。我不想等那么久，所以相反，我可以将调试范围缩小到只有这个函数 decodeIntegerSequence。

## 五、Advances in texture compression(纹理压缩的进展)
在深入研究这些工具之前，我将简要讨论 Apple 平台上纹理压缩的基础知识。

### Apple 平台上纹理压缩的基础知识
#### fixed-rate
在这种情况下，纹理压缩是纹理数据的固定速率、有损压缩。这主要用于静态纹理数据的离线压缩，例如贴花或法线贴图。虽然你可以在运行时压缩动态纹理数据，但这不是我今天要讨论的内容。
#### Block-based encoding
大多数纹理压缩的工作原理是将纹理分成块并将每个块压缩为一对颜色。
#### Muttiple formats to suit any kind of texture data
这对定义了一个本地化的调色板，包括从这些端点插入的其他颜色，以及从这个调色板中选择的每像素索引。每种格式都有适合不同类型纹理数据的不同强度。Apple GPU 还支持从我们的 A12 设备开始的无损帧缓冲压缩，非常适合优化带宽。

查看去年的会议“使用 GPU 计数器优化 Metal 应用程序和游戏”，了解有关测量 GPU 用于你的应用程序的内存带宽的更多信息。

### lossless compression
另一种选择是在我在本演示文稿中介绍的 GPU 纹理压缩之上执行纹理文件的无损压缩。这可以进一步减少应用程序下载的大小。既然我已经为本次演讲定义了纹理压缩，我将讨论纹理压缩可以为你的应用程序带来的好处。纹理压缩是开发应用程序的重要一步。

一般来说，游戏的大部分内存占用由纹理组成。使用纹理压缩可以将更多纹理加载到内存中，并使用更详细的纹理来创建更具视觉吸引力的游戏。压缩还可以让你减少应用程序的大小和内存占用。
 
### TextureConverter
 
iOS SDK 中现有的 TextureTool 有一个相对简单的管道。TextureTool 读取输入图像，根据需要生成 mipmap，逐块压缩纹理，然后将结果写入新的输出文件。

但随着图形算法复杂性的增加，纹理需要更高级的处理。这些过程的核心是在正确的色彩空间中执行操作，同时最大限度地减少数字精度之间转换的舍入。了解这一点，我们设计了一个名为 TextureConverter 的新压缩工具来处理纹理处理复杂性的必要增加，并为你提供大量新选项。

#### gamma correction
让我们逐步完成扩展管道的每个阶段，从 gamma 开始。Gamma 校正是一种非线性操作，用于对图像中的亮度进行编码和解码。
 
![032.png](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7930fac7e2774a7f8889c888a151ca3e~tplv-k3u1fbpfcp-watermark.image)
纹理可以在许多伽马空间中编码。最佳选择取决于纹理表示的数据类型。大多数视觉数据（例如贴花或光照贴图）在非线性空间（例如 sRGB）中编码时效果最佳。非视觉数据，如法线贴图，应该在线性空间中编码。此选择可让你在需要的黑暗区域获得更高的准确性。 非视觉数据，如法线贴图，应该在线性空间中编码。压缩应在你的目标色彩空间中执行，使用“gamma_in”和“gamma_out”选项指定。

###### RGBM
RGBM 编码是一种通过在 alpha 通道中存储乘数并通过该乘数缩放 RGB 通道来压缩 LDR 格式的 HDR 数据的技术。这是教室的 HDR 图像示例。这是同样的教室图像，乘数存储在灰度可见的 alpha 通道中。我将通过代码示例向你展示如何计算要编码为 RGBM 的乘数。
 ```
 float4 EncodeRGBM(float3 in)
 {
     float4 rgbm;
     rbgm.a = max3(in.r, in.g, in.b) / RGBM_Range;
     rbgm.rgb = in / (rgbm.a * RGBM_Range);
     return rgbm;
 }
 ```
EncodeRGBM 是一个简化的伪代码函数，我将引导你完成以帮助你了解编码为 RGBM 的机制。此代码段包括使用 RGBM_Range，这是用于设置 RGBM 范围的全新参数，默认为 6.0,为了计算 RGBM alpha 值，乘数，首先，我将确定输入纹理的红色、绿色和蓝色通道的最大值。这是通过 Metal 的 max3 函数完成的。然后将此最大值除以 RGBM_Range。

为了计算编码的 RGBM 的红色、绿色和蓝色通道值，首先，将先前计算的乘数乘回 RGBM_Range，用于缩放值以存储在 alpha 通道中。然后，输入纹理除以最终的乘数值。

为了计算编码的 RGBM 的红色、绿色和蓝色通道值，首先，将先前计算的乘数乘回 RGBM_Range，用于缩放值以存储在 alpha 通道中。然后，输入纹理除以最终的乘数值。

###### Normal map encoding
既然我已经向你介绍了 RGBM 编码，我将继续介绍法线贴图编码。在大多数情况下，当提到法线贴图时，我们特指对象空间法线贴图。

X 分量是从红色或 alpha 通道中读取的，而 Y 分量则来自 alpha 或绿色通道，具体取决于压缩格式。

##### Texture compression families
现在，我将介绍可供你选择的纹理压缩格式。以下是 Apple 平台支持的纹理压缩格式系列的概述。

![056.png](https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bf3acdc9dc1e4cb6ab9c8fd7eb9248b3~tplv-k3u1fbpfcp-watermark.image)
 iOS 和 Apple Silicon 平台支持 ASTC 和 PVRTC 系列，所有 macOS 平台都支持 BCn 系列。
 我将更详细地介绍这些格式系列中的每一个，并为你提供一些指南，以帮助你选择最适合你需求的格式。
###### BCN
 
![057.png](https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/15433678ab864d51bcfc3abd6ef574fc~tplv-k3u1fbpfcp-watermark.image)
 我将从 BCN 格式开始。BCn 是一组七种格式，它们都使用 4×4 像素块进行操作，并且每个像素使用 4 位或 8 位。每种压缩格式都适用于不同的数据格式。BC1 和 BC3 通常用于 RGB 和 RGBA 压缩，BC6 非常适合 HDR 图像，而 BC5 具有双独立通道，非常适合法线贴图编码。
###### ASTC
接下来是 ASTC，这是 LDR、sRGB 和 HDR 色彩空间中的 RGBA 格式系列。ASTC 格式系列允许所有尺寸的最高质量，因此更普遍地推荐使用 PVRTC。每种格式都有一个每像素位数与质量的范围。
使用ASTC，无论格式如何，每个块的字节大小都是相同的，而它所代表的纹素数量则有所不同。这使你可以在 4×4 块大小的最高质量压缩但最低压缩率与 12×12 块大小的最低压缩质量但最高压缩率之间保持连续。

LDR、sRGB 和 HDR 变体描述了压缩 ASTC 纹理的颜色范围。LDR 和 sRGB 都在零到一范围内，在线性或 sRGB 空间中，而 HDR 变体用于零到一范围之外的数据。
###### PVRTC
最后，PVRTC 格式以 2 位或 4 位模式提供 RGB 和 RGBA。
这种格式的数据块总是占用8个字节，所以在2位模式下，每8×4像素有一个块，而在4位模式下，每4×4像素有一个块.
###### Guidelines
 
![058.png](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/afa48fc08287416489eb7e1d424c35c2~tplv-k3u1fbpfcp-watermark.image)
在 iOS 设备上，默认情况下你应该始终使用 ASTC 压缩，仅当你支持 A7 GPU 和更早版本时才添加 PVRTC 压缩和每设备细化。如果你有任何 HDR 纹理，则可以在 A13 和更高版本的 GPU 上利用 ASTC HDR。对于 macOS，BCn 可全面使用。

在 Apple Silicon Mac 上，你还可以选择使用 ASTC，如果你还针对 iOS 设备，则应考虑使用此选项。虽然 PVRTC 在 Apple Silicon 上可用，但我们不推荐此选项，它仅用于 iOS 旧版支持。

由于每个压缩格式系列中有许多不同的格式可供选择，因此为你的应用选择最有效的纹理压缩格式的准则是尽可能选择每个纹理和每个目标。除非你的所有纹理都是 RGB 或 RGBA 数据，否则你应该根据要压缩的数据类型选择压缩格式，例如选择一种格式，允许将压缩作为普通数据的两个独立通道。

压缩为 ASTC 格式时，你可能需要选择格式的子集。考虑将那些需要最高质量的纹理与那些在更高压缩率下可接受的纹理进行分桶。

### How to get it(相关资源)

TextureConverter 可以在 Xcode 13 中实用，目前可以下载beta版本。

在 Windows 上，想实用TextureConverter 需要下载 Metal Developer Tools for Windows 2.0 包的一部分提供，可从 developer.apple.com 下载。请注意，在 Windows 中，不支持压缩为 PVRTC 格式，因为 PVRTC 在 macOS 中可用于支持旧版 iOS 平台。

Metal Developer Tools for Windows 的另一个重要部分是 Metal Compiler for Windows。适用于 Windows 的 Metal Compiler 于去年推出，支持 Metal Shading Language 2.3 版。

全年的更新反映了 Xcode 中发布的 Metal 编译器的更新。最新发布版本是 1.2，其中包括对 Apple Silicon Mac 上的 Metal Shading Language 的支持。2.0 版的现已推出，支持 Metal Shading Language 2.4 中的所有重要新功能。
